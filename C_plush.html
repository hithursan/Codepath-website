About This Curriculum

This is a 16-week, project-driven C++ curriculum designed to take a complete beginner to a professional-level C++ developer. The roadmap moves from compilation, types, operators, and strings through control flow, functions, OOP, and the STL, into templates, smart pointers, file I/O, and exception handling, all the way to lambdas, design patterns, C++20 coroutines, concurrency, and three complete capstone projects.

The curriculum is organized into four phases: Phase 1 covers absolute basics and modern C++ syntax, Phase 2 covers control flow, functions, and data structures, Phase 3 covers OOP, file handling, error handling, templates, and the STL, and Phase 4 covers master-level patterns, concurrency, and full production-grade applications.

By Week 16, you will have built three complete, portfolio-ready applications and be confident writing production-grade modern C++ from scratch.

Level: Beginner → Master  |  Duration: 16 weeks  |  Standard: C++17/C++20  |  Prerequisites: Basic programming logic (no C++ experience needed)

Phase 1 — Absolute Basics

Weeks 1–4: Toolchain, types, operators, input/output, and the modern C++ syntax foundation.

Week 1 — Setting Up & Your First Program

Understand the C++ compilation pipeline and write your very first working program.

Core Focus

Toolchain setup, compilation, and basic program anatomy.

Topics & Subtopics

  1. Compilation & Toolchain — C++ source code is compiled by a compiler (like g++ or clang++) into a binary executable through preprocessing, compiling, and linking stages. Understanding this helps you diagnose build errors early.
  2. Program Structure (main, #include) — Every C++ program begins execution in main(), and #include directives bring in standard library functionality like input/output. This skeleton is the same across all C++ programs.
  3. std::cout and Basic Output — std::cout is the standard output stream object used with the << insertion operator to print text and values. It is the idiomatic C++ replacement for C's printf.

Why This Week Matters: The compilation model is fundamental to understanding C++ — unlike interpreted languages, every error, warning, and optimization happens at compile time. Getting the toolchain right from Day 1 means you spend time learning C++, not fighting your environment.

Code Example

#include <iostream>     // Provides std::cout

int main() {
    std::cout << "Hello, World!" << std::endl; // endl flushes + newline
    return 0;
}

Week 2 — Variables, Data Types & auto

Store and label data in memory the modern C++ way using type inference.

Core Focus

Storing and labeling data in memory the modern C++ way.

Topics & Subtopics

  1. Variables & Declaration — A variable is a named, typed storage location in memory, and C++ requires you to declare its type before use. Initialization at declaration is strongly encouraged to avoid undefined values.
  2. Fundamental Types (int, double, char, bool) — C++ provides built-in types for integers, decimals, characters, and true/false logic. Each type has a fixed size and range that affects memory usage and behavior.
  3. Type Inference with auto — The auto keyword lets the compiler deduce a variable's type from its initializer, reducing verbosity. It is especially useful with complex types like iterators that you will meet later.

Why This Week Matters: C++ is a strongly typed language — every variable has a type the compiler enforces. Understanding types prevents an entire class of bugs (integer overflow, precision loss) that crash programs silently. auto is the modern, preferred style in C++11 and beyond.

Code Example

#include <iostream>

int main() {
    auto age    = 25;        // Compiler infers int
    double price = 9.99;
    bool isStudent = true;
    std::cout << age << " " << price << " " << isStudent << "\n";
    return 0;
}

Week 3 — Operators & Type Casting

Perform computations and safely convert between types using modern C++ casts.

Core Focus

Performing computations and safely converting between types.

Topics & Subtopics

  1. Arithmetic & Compound Assignment — Arithmetic operators (+ - * / %) perform calculations, while compound assignment (+= -=) updates a variable in place. These are the building blocks of all numeric logic in any C++ program.
  2. Relational & Logical Operators — Relational operators (== != < >) compare values to produce a bool, and logical operators (&& || !) combine conditions. Together they drive every decision your program makes.
  3. C++-Style Casting (static_cast) — static_cast<Type>(value) explicitly and safely converts a value's type, replacing C-style casts in modern code. It clearly signals intent and catches more errors at compile time.

Why This Week Matters: Integer division is one of the most common beginner bugs — 7 / 2 produces 3, not 3.5. static_cast is the explicit, readable, and compiler-checked solution. Understanding operator precedence prevents subtle logic bugs in every program you will ever write.

Code Example

#include <iostream>

int main() {
    int a = 7, b = 2;
    double result = static_cast<double>(a) / b; // Safe explicit cast
    std::cout << "7 / 2 = " << result << "\n";  // Prints 3.5
    return 0;
}

Week 4 — Input Handling, Constants & Strings

Build interactive programs and work with text using the safe std::string class.

Core Focus

Interactive programs and the std::string class.

Topics & Subtopics

  1. std::cin for Input — std::cin reads formatted input from the keyboard using the >> extraction operator. It works seamlessly with built-in types and std::string, and std::getline is used for full-line input including spaces.
  2. Constants (const, constexpr) — const marks a variable as unmodifiable after initialization, while constexpr guarantees evaluation at compile time. Both improve safety and enable compiler optimizations for known-at-compile-time values.
  3. std::string Basics — std::string is C++'s safe, dynamic alternative to C's raw character arrays, handling memory management automatically. It supports concatenation with +, comparison with ==, and methods like .length(), .substr(), and .find().

Why This Week Matters: const and constexpr are everywhere in professional C++ code — they communicate intent, prevent bugs, and enable optimizations. std::string eliminates the notorious buffer-overflow bugs of C-style char arrays that caused decades of security vulnerabilities.

Code Example

#include <iostream>
#include <string>

int main() {
    constexpr double PI = 3.14159; // Compile-time constant
    std::string name;
    std::cout << "Enter your name: ";
    std::getline(std::cin, name);  // Reads full line including spaces
    std::cout << "Hello, " << name << "! PI = " << PI << "\n";
    return 0;
}

Phase 2 — Control Flow & Logic

Weeks 5–8: Branching, loops, functions, references, and STL data structures.

Week 5 — Conditional Statements

Write branching logic that makes decisions based on runtime data.

Core Focus

Branching logic and decision-making.

Topics & Subtopics

  1. if / else if / else — These statements run different code blocks based on whether a condition evaluates to true or false. They are the most fundamental form of branching in any language and the basis of all decision-making logic.
  2. switch Statements — switch compares a single variable against multiple constant cases, offering cleaner syntax than long if-else chains for discrete values. Always use break to prevent unintended fall-through to the next case.
  3. Ternary Operator — The ternary operator (condition ? a : b) compactly expresses a simple conditional assignment in a single line. It improves readability for short, simple branches where a full if-else would be verbose.

Why This Week Matters: Every non-trivial program branches. Choosing the right construct — if-else for complex conditions, switch for discrete values, ternary for simple inline choices — is a mark of readable, professional code. switch fall-through is also a classic C++ interview topic.

Code Example

#include <iostream>

int main() {
    int score = 85;
    // Chained ternary for compact grade assignment
    char grade = (score >= 90) ? 'A' : (score >= 80) ? 'B' : 'C';
    std::cout << "Grade: " << grade << "\n";

    int day = 3;
    switch (day) {
        case 1: std::cout << "Monday\n";    break;
        case 2: std::cout << "Tuesday\n";   break;
        case 3: std::cout << "Wednesday\n"; break;
        default: std::cout << "Other\n";
    }
    return 0;
}

Week 6 — Loops & Iteration

Repeat logic efficiently and idiomatically using all three C++ loop types.

Core Focus

Repeating logic efficiently and idiomatically.

Topics & Subtopics

  1. for and Range-Based for Loops — A traditional for loop repeats code a set number of times with a counter, while a range-based for loop (for (auto x : container)) iterates directly over a collection's elements. The latter is the modern, preferred style for any STL container.
  2. while and do-while Loops — while repeats while a condition holds, checking before execution. do-while checks after, guaranteeing at least one run regardless of the condition. Choose based on whether the loop body must always execute at least once.
  3. break and continue — break exits a loop entirely and transfers control past it, while continue skips the rest of the current iteration and jumps to the next check. Both give precise control over loop execution flow.

Why This Week Matters: Loops power algorithms — sorting, searching, processing data. The range-based for loop is idiomatic modern C++ and eliminates off-by-one errors common with index-based loops. Understanding break and continue is essential for writing efficient search and filter logic.

Code Example

#include <iostream>
#include <vector>

int main() {
    // Traditional for loop with continue
    for (int i = 1; i <= 5; i++) {
        if (i == 3) continue; // Skip 3
        std::cout << i << " ";
    }
    std::cout << "\n";

    // Range-based for loop — modern, preferred style
    std::vector<std::string> names = {"Alice", "Bob", "Carol"};
    for (const auto &name : names) {
        std::cout << name << "\n";
    }
    return 0;
}

Week 7 — Functions & References

Write modular, reusable, and efficient code using functions, overloading, and references.

Core Focus

Modular, reusable, and efficient code organization.

Topics & Subtopics

  1. Function Declaration & Default Arguments — Functions are named, reusable blocks of logic that can specify default parameter values, reducing the need for overloaded versions. Forward declarations (prototypes) let you call a function before its full definition appears in the file.
  2. Pass by Reference (&) — Passing a parameter by reference (int &x) lets a function modify the caller's original variable without copying it. This is C++'s safer, cleaner alternative to C's raw pointers for most modification use cases and avoids expensive copies of large objects.
  3. Function Overloading — Overloading allows multiple functions with the same name but different parameter lists. The compiler picks the correct version at compile time based on argument types, improving API readability for related operations without needing different names.

Why This Week Matters: Functions are the primary unit of code organization in C++. Pass-by-reference is one of the most important concepts in the language — it is how the STL's algorithms and containers avoid unnecessary copies. Overloading is used throughout the standard library and every major C++ codebase.

Code Example

#include <iostream>

void increment(int &x) { x++; } // Reference: modifies the original variable

// Overloaded add functions — same name, different parameter types
int    add(int a, int b)           { return a + b; }
double add(double a, double b)     { return a + b; }
int    add(int a, int b, int c)    { return a + b + c; }

int main() {
    int val = 10;
    increment(val);
    std::cout << "val = " << val << "\n"; // 11
    std::cout << add(1, 2) << "\n";       // int version
    std::cout << add(1.5, 2.5) << "\n";   // double version
    return 0;
}

Week 8 — Arrays, Vectors & Strings

Store and manage collections of data using C-style arrays and the modern STL std::vector.

Core Focus

Storing and managing collections of data.

Topics & Subtopics

  1. C-Style Arrays — A C-style array stores a fixed number of same-typed elements in contiguous memory, indexed from zero. It is foundational knowledge but has a fixed, non-resizable size and does not know its own length — making it error-prone in real code.
  2. std::vector — The Dynamic Array — std::vector is a resizable array from the Standard Template Library that grows automatically as elements are added with .push_back(). It is the default, idiomatic choice for sequential data in all modern C++ programs.
  3. String Manipulation — std::string supports operations like .substr(), .find(), .replace(), and concatenation for powerful text processing. These methods replace unsafe manual character-array logic inherited from C.

Why This Week Matters: std::vector is the most used container in C++ — it appears in virtually every non-trivial program. Understanding the difference between stack-allocated C arrays (fixed size, fast) and heap-managed vectors (dynamic, safe) is essential for writing correct, performant code.

Code Example

#include <iostream>
#include <vector>
#include <string>

int main() {
    // C-style array — fixed size, no bounds checking
    int scores[3] = {90, 85, 92};

    // std::vector — dynamic, safe, self-aware
    std::vector<int> nums = {3, 1, 4, 1, 5};
    nums.push_back(9);                        // Grows automatically
    std::cout << "Size: " << nums.size() << "\n"; // 6

    // String manipulation
    std::string word = "Hello, World!";
    std::cout << word.substr(7, 5) << "\n"; // "World"
    return 0;
}

Phase 3 — Intermediate & Advanced Core

Week 9 — Object-Oriented Programming I — Classes & Objects

Model real-world entities with encapsulation using C++ classes.

Core Focus

Modeling real-world entities with encapsulation.

Topics & Subtopics

  1. Classes & Objects — A class is a blueprint defining data (member variables) and behavior (member functions/methods), and an object is a concrete instance of that class. This is the foundation of object-oriented design in C++ and enables modeling complex domains as self-contained entities.
  2. Constructors & Destructors — A constructor initializes an object's state when it is created, and a destructor automatically cleans up resources when the object is destroyed. They automate setup and teardown — the foundation of C++'s RAII (Resource Acquisition Is Initialization) philosophy.
  3. Encapsulation (public/private) — Encapsulation hides internal data (private) and exposes only controlled access through public methods (getters/setters). This protects an object's internal invariants from invalid external modification and is the first pillar of OOP.

Why This Week Matters: Classes are the primary abstraction mechanism in C++. RAII — the idea that resource lifetime is tied to object lifetime via constructors and destructors — is the key concept that makes C++ memory and resource management safe and deterministic without a garbage collector.

Code Example

#include <iostream>
#include <string>

class Person {
private:
    std::string name; // Hidden internal state — encapsulation
    int age;
public:
    // Constructor with member initializer list
    Person(std::string n, int a) : name(n), age(a) {}
    // Destructor — called automatically when object goes out of scope
    ~Person() { std::cout << name << " destroyed.\n"; }
    void greet() const { std::cout << "Hi, I'm " << name << ", age " << age << "\n"; }
};

int main() {
    Person p("Alice", 30);
    p.greet();
    return 0; // Destructor called here automatically
}

Week 10 — OOP II — Inheritance & Polymorphism

Build extensible class hierarchies using inheritance, virtual functions, and abstract classes.

Core Focus

Building extensible hierarchies of behavior.

Topics & Subtopics

  1. Inheritance — Inheritance lets a derived class reuse and extend the data and behavior of a base class using class Dog : public Animal syntax. This promotes code reuse and models "is-a" relationships, keeping shared logic in one place while allowing specialization.
  2. Polymorphism & Virtual Functions — Marking a function virtual allows a derived class to override its behavior, and calling it through a base class pointer or reference invokes the derived version at runtime. The override keyword makes the intent explicit and catches typos at compile time.
  3. Abstract Classes & Interfaces — A class with at least one pure virtual function (= 0) cannot be instantiated and acts as a contract that all derived classes must fulfill. This enforces that every subclass implements required behavior, creating a reliable interface.

Why This Week Matters: Polymorphism is what makes C++ design flexible — you can write code that operates on a base class pointer and automatically get the right behavior for any derived type added later. Always use override in modern C++ and make base class destructors virtual to prevent resource leaks through base pointers.

Code Example

#include <iostream>
#include <memory>

class Animal {
public:
    virtual void speak() const { std::cout << "...\n"; }
    virtual ~Animal() = default; // Virtual destructor — critical for polymorphism
};

class Dog : public Animal {
public:
    void speak() const override { std::cout << "Woof!\n"; }
};

class Cat : public Animal {
public:
    void speak() const override { std::cout << "Meow!\n"; }
};

int main() {
    std::unique_ptr<Animal> a = std::make_unique<Dog>();
    a->speak(); // Calls Dog::speak() — runtime polymorphism
    return 0;
}

Week 11 — File Handling & Error Handling

Persist data to disk and write robust, exception-safe programs.

Core Focus

Persisting data and writing robust, exception-safe programs.

Topics & Subtopics

  1. File I/O (fstream) — std::ifstream reads from files and std::ofstream writes to them using the same familiar <</>> operators as console I/O. Always check that a file stream opened successfully with .is_open() before attempting to use it.
  2. Exceptions (try/catch/throw) — C++ exceptions let you signal and handle errors by throw-ing an object and catching it in a try/catch block at any point up the call stack. This cleanly separates error-handling logic from normal program flow without requiring error-code returns.
  3. Standard Exception Types — The <stdexcept> header provides built-in exception classes like std::runtime_error, std::invalid_argument, and std::out_of_range. Using them (or deriving from std::exception) creates consistent, descriptive, and catchable errors.

Why This Week Matters: Real programs persist data. File I/O with proper error handling is in every production C++ application — from game save files to database systems. Exception-based error handling, combined with RAII, ensures resources like file handles are always properly closed even when errors occur.

Code Example

#include <fstream>
#include <stdexcept>
#include <iostream>

void writeScore(int score, const std::string &filename) {
    if (score < 0) throw std::invalid_argument("Score cannot be negative");
    std::ofstream file(filename);
    if (!file.is_open()) throw std::runtime_error("Failed to open file: " + filename);
    file << score;
} // File closed automatically here — RAII in action

int main() {
    try {
        writeScore(95, "scores.txt");
        writeScore(-1, "scores.txt"); // Throws
    } catch (const std::invalid_argument &e) {
        std::cerr << "Input error: " << e.what() << "\n";
    } catch (const std::exception &e) {
        std::cerr << "Error: " << e.what() << "\n";
    }
    return 0;
}

Week 12 — Templates, STL & Smart Pointers

Write generic algorithms and manage heap memory safely with modern C++ tools.

Core Focus

Generic programming and modern, safe memory management.

Topics & Subtopics

  1. Function & Class Templates — Templates let you write a single function or class that works generically across multiple types, with the correct version generated by the compiler for each type used. This avoids duplicating identical logic for int, double, std::string, etc.
  2. The Standard Template Library (STL) — The STL provides battle-tested generic containers (std::vector, std::map, std::set, std::unordered_map) and algorithms (std::sort, std::find, std::transform) that dramatically reduce the code you need to write by hand.
  3. Smart Pointers (unique_ptr, shared_ptr) — Smart pointers wrap raw pointers and automatically call delete when the object is no longer needed, eliminating most manual memory leaks and dangling pointer bugs. Use unique_ptr for exclusive ownership and shared_ptr for shared ownership.

Why This Week Matters: Templates are the foundation of the entire C++ STL. Smart pointers have essentially replaced raw new/delete in modern C++ — the C++ Core Guidelines say "never use raw delete" in new code. Mastering these three topics makes you immediately productive in any professional C++ codebase.

Code Example

#include <iostream>
#include <vector>
#include <algorithm>
#include <memory>

// Generic template function — works for int, double, string, etc.
template <typename T>
T maxVal(T a, T b) { return (a > b) ? a : b; }

// Generic sum using template
template <typename T>
T sum(const std::vector<T> &v) {
    T total = T{};
    for (const auto &x : v) total += x;
    return total;
}

int main() {
    std::vector<int> nums = {5, 2, 8, 1, 9};
    std::sort(nums.begin(), nums.end()); // STL algorithm

    auto ptr = std::make_unique<int>(42); // Smart pointer — auto-freed
    std::cout << "Max: " << maxVal(10, 20) << ", Sum: " << sum(nums) << "\n";
    return 0;
}

Week 13 — Lambdas & the Decorator Pattern

Treat behavior as first-class data and wrap functions flexibly using lambdas and std::function.

Core Focus

Treating behavior as data and wrapping functions flexibly.

Note: C++ does not have Python-style decorators as a language keyword. Master-level C++ engineers achieve decorator-like behavior via templates, lambdas, and the Decorator design pattern. This week covers the genuine, idiomatic C++ equivalents.

Topics & Subtopics

  1. Lambda Expressions — A lambda is an anonymous, inline function object capable of capturing surrounding variables by value ([=]) or reference ([&]). Lambdas are heavily used with STL algorithms like std::sort, std::for_each, and std::find_if as concise, local callbacks.
  2. std::function & Higher-Order Functions — std::function is a generic wrapper that can hold any callable — a function pointer, lambda, or functor — matching a given signature. This enables passing and storing behavior just like data, enabling higher-order function patterns.
  3. The Decorator Design Pattern — In C++, "decorating" a function means wrapping it inside another callable that adds behavior (timing, logging, retry logic) before and/or after invoking the original. This is the idiomatic OOP/functional equivalent of Python's @decorator syntax, and is used in production logging, profiling, and middleware systems.

Code Example

#include <functional>
#include <iostream>
#include <chrono>

// Decorator pattern: wraps any void() callable with logging
auto logger(std::function<void()> fn) {
    return [fn]() {
        std::cout << "[LOG] Start\n";
        fn();
        std::cout << "[LOG] End\n";
    };
}

// Lambda with capture — captures 'multiplier' by value
int main() {
    int multiplier = 3;
    auto triple = [multiplier](int x) { return x * multiplier; };
    std::cout << triple(7) << "\n"; // 21

    auto loggedHello = logger([]() { std::cout << "Hello!\n"; });
    loggedHello();
    return 0;
}

Week 14 — Generators via Iterators & C++20 Coroutines

Produce values lazily on demand using custom iterators and C++20 co_yield.

Core Focus

Lazy, on-demand value production.

Note: C++ does not have Python-style generators as a keyword. Master-level C++ engineers achieve lazy, on-demand value production via custom iterators or C++20 coroutines with co_yield. This is the genuine, idiomatic C++ equivalent.

Topics & Subtopics

  1. Custom Iterators — A custom iterator class implements operator++, operator*, and operator!= so your own data structures work with range-based for loops and STL algorithms. This is the traditional C++ approach to lazy, step-by-step iteration over custom sequences.
  2. C++20 Coroutines (co_yield) — co_yield lets a function pause and return a value, then resume execution from exactly where it left off on the next call — just like a Python generator. This is the most direct, modern equivalent of generators in C++, enabling memory-efficient lazy sequences.
  3. Linked Lists & Recursion — A linked list is a chain of dynamically allocated nodes connected by smart pointers, allowing flexible, non-contiguous growth. Recursion solves problems by having a function call itself on smaller sub-problems until a base case stops the chain — the basis of many classic algorithms.

Code Example

// C++20 Coroutine-based generator (conceptual — requires coroutine support)
// Produces Fibonacci numbers lazily, one at a time
Generator<int> fibonacci() {
    int a = 0, b = 1;
    while (true) {
        co_yield a;             // Pause here, return 'a' to the caller
        a = std::exchange(b, a + b); // Resume here on next call
    }
}

// Custom Iterator example — lets range-for work on a custom Range class
class Range {
    int current, end;
public:
    Range(int start, int end) : current(start), end(end) {}
    Range& operator++() { ++current; return *this; }
    int operator*() const { return current; }
    bool operator!=(const Range& o) const { return current != o.current; }
    Range begin() { return *this; }
    Range end()   { return Range(end, end); }
};

Week 15 — Concurrency with std::thread

Write safe, parallel modern C++ code using threads, mutexes, and async futures.

Core Focus

Writing safe, parallel modern C++ code.

Topics & Subtopics

  1. Creating Threads (std::thread) — std::thread launches a function to run concurrently with the rest of the program. Calling .join() on the thread object blocks the caller until that thread finishes. This is the standard, portable way to achieve parallelism in modern C++.
  2. Race Conditions & std::mutex — A race condition occurs when multiple threads access shared data simultaneously without synchronization, producing unpredictable, non-reproducible results. std::mutex combined with std::lock_guard (RAII locking) ensures only one thread modifies shared data at a time.
  3. std::async & Futures — std::async runs a function asynchronously and returns a std::future that will eventually hold its result. Calling .get() on the future blocks until the result is ready. This is a higher-level, simpler abstraction than manually managing threads for one-off concurrent tasks.

Why This Week Matters: Modern CPUs have many cores — sequential code that does not use them leaves performance on the table. Concurrency is used in game engines, web servers, data pipelines, and any performance-critical system. Understanding race conditions and mutexes is essential for writing concurrent code that is correct, not just fast.

Code Example

#include <thread>
#include <mutex>
#include <iostream>

int counter = 0;
std::mutex mtx;

void incrementMany(int times) {
    for (int i = 0; i < times; i++) {
        std::lock_guard<std::mutex> lock(mtx); // RAII lock — auto-released
        counter++;
    }
}

int main() {
    std::thread t1(incrementMany, 100000);
    std::thread t2(incrementMany, 100000);
    t1.join();
    t2.join();
    std::cout << "Counter: " << counter << "\n"; // Always 200000 with mutex
    return 0;
}

Code Example

// Project 3 — thread worker: counts words in one chunk of the file
void countChunk(const std::string &chunk,
                std::mutex &mtx,
                std::unordered_map<std::string, int> &counts) {
    std::istringstream stream(chunk);
    std::string word;
    while (stream >> word) {
        std::lock_guard<std::mutex> lock(mtx); // Protects shared map
        counts[word]++;
    }
}
// Build: g++ -std=c++20 -pthread -O2 -o wordcount main.cpp
