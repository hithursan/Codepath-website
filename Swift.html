Swift: Beginner to Master

Complete 12-Week Curriculum — From Swift Syntax & Optionals to Concurrency, Protocols & Production-Ready Apps

Intro W1 W2 W3 W4 W5 W6 W7 W8 W9 W10 W11 W12 Summary

About This Curriculum

This is a 12-week, phase-based Swift curriculum designed to take a complete beginner to a professional-level Swift developer. The roadmap moves from syntax, variables, optionals, and collections through control flow, functions, structs, classes, and enums, into protocols, error handling, generics, and networking, all the way to modern concurrency, functional patterns, and three complete capstone projects.

The curriculum is organized into four phases: Phase 1 covers absolute basics including syntax and primitive types, Phase 2 covers control flow, functions, and core data structures, Phase 3 covers intermediate and advanced core topics like OOP, error handling, and libraries, and Phase 4 covers master-level concurrency, functional patterns, and architecture.

By Week 12, you will have built three complete, portfolio-ready applications and be confident building production-grade Swift applications from scratch.

Level: Beginner → Master  |  Duration: 12 weeks  |  Stack: Swift 5.9+, Xcode, Swift Playgrounds  |  Prerequisites: Basic programming logic

Phase 1 — Absolute Basics

Weeks 1–3: Fundamentals, data types, optionals, and collections.

Week 1 — Swift Environment, Syntax & Primitive Types

Set up your toolchain and learn Swift's basic syntax and built-in types.

Core Focus

Setting up your toolchain and learning Swift's basic syntax and built-in primitive types.

Topics & Subtopics

  1. Xcode/Swift Playgrounds & print() — Swift code can be written and run instantly using Xcode Playgrounds, making experimentation fast and visual. The print() function is the simplest way to output values while learning.
  2. Variables & Constants (var, let) — var declares a mutable variable that can change later, while let declares an immutable constant whose value is fixed after assignment. Swift encourages using let by default for safer code.
  3. Type Inference & Type Annotation — Swift can automatically infer a variable's type from its initial value, reducing boilerplate. You can also explicitly annotate a type using a colon when clarity or specific typing is needed.
  4. Primitive Types (Int, Double, String, Bool) — Swift's core primitive types represent whole numbers, decimal numbers, text, and true/false logic. These types are strongly enforced, preventing accidental mixing of incompatible data.

Why This Week Matters: Everything in Swift is built on these primitives. Understanding the difference between var and let, and how type inference works, forms the mental model you will rely on every single week going forward.

Code Example

// Constants and variables with type inference
let name: String = "Swift Learner" // explicit annotation
var age = 25                       // inferred as Int
var isLearning = true              // inferred as Bool
print("\(name) is \(age) years old. Learning: \(isLearning)")

Create a Playground file. Declare constants for your name and favorite number, and a variable for your current mood (as a String). Print a sentence combining all three using string interpolation.

Week 2 — Operators, Optionals & Type Conversion

Master Swift's operators and its signature safety feature: Optionals.

Core Focus

Mastering Swift's operators and its signature safety feature — Optionals.

Topics & Subtopics

  1. Arithmetic & Comparison Operators — Swift supports the standard set of operators (+, -, *, /, %, ==, <, >) for numeric and logical operations. Unlike some languages, Swift checks for overflow by default and will trap on errors.
  2. Optionals (?) — Handling the Absence of a Value — An Optional represents a value that might be present or might be nil (nothing), modeled with a trailing ? on the type. This forces developers to explicitly handle missing data instead of crashing unexpectedly.
  3. Optional Binding & Unwrapping (if let, guard let) — Optional binding safely extracts the value inside an Optional only if it actually exists. if let and guard let are the idiomatic, crash-safe ways to unwrap Optionals.
  4. Type Conversion (Explicit Casting) — Swift never converts between types automatically, so converting an Int to a Double requires an explicit initializer call. This strictness avoids silent, unintended data loss.

Why This Week Matters: Optionals are Swift's most defining feature — they eliminate an entire class of null pointer crashes that plague other languages. Mastering if let and guard let is the single most important skill for writing safe Swift code.

Code Example

var score: Int? = 95          // Optional Int, may be nil
if let safeScore = score {    // safely unwrap
    let percentage = Double(safeScore) / 100.0 // explicit conversion
    print("Score: \(percentage * 100)%")
}

🛠 Weekly Task

Week 3 — Collections: Arrays, Dictionaries & Sets

Learn Swift's core collection types used to group and organize data.

Core Focus

Learning Swift's core collection types used to group and organize related data.

Topics & Subtopics

  1. Arrays (Ordered Collections) — An Array stores an ordered list of values of the same type, allowing duplicates. Arrays support adding, removing, and accessing elements by their numeric index.
  2. Dictionaries (Key-Value Pairs) — A Dictionary stores unordered key-value pairs, allowing fast lookup of a value using its unique key. Keys must be unique and of a consistent, hashable type.
  3. Sets (Unique, Unordered Collections) — A Set stores unique values with no defined order and no duplicates allowed. Sets are ideal for membership checks and operations like unions or intersections.
  4. Tuples (Lightweight Grouped Values) — A tuple groups multiple values into a single compound value without creating a named type. Tuples are great for quickly returning or passing related values together.

Why This Week Matters: Arrays, Dictionaries, and Sets are the backbone of data organization in every Swift application. Understanding when to use each collection type — and combining them with loops next week — unlocks the ability to build real, data-driven programs.

Code Example

var fruits = ["Apple", "Banana"]              // Array
var prices = ["Apple": 1.5, "Banana": 0.5]    // Dictionary
var uniqueIDs: Set = [101, 102, 103]          // Set
let coordinate = (x: 10, y: 20)               // Tuple
fruits.append("Mango")
print(fruits, prices, uniqueIDs, coordinate)

Phase 2 — Control Flow & Logic

Weeks 4–6: Functions, modules, and core data structures.

Week 4 — Control Flow: Conditionals & Loops

Direct your program's execution path using Swift's expressive control structures.

Core Focus

Directing your program's execution path using Swift's expressive control structures.

Topics & Subtopics

  1. if / else if / else Statements — Conditional statements let your program branch its logic based on Boolean expressions. Swift's if doesn't require parentheses around the condition, keeping syntax clean.
  2. switch Statements & Pattern Matching — Swift's switch is exceptionally powerful, supporting ranges, tuples, and value binding without needing a break. It must be exhaustive, ensuring every possible case is handled.
  3. for-in Loops — The for-in loop iterates over sequences like arrays, dictionaries, ranges, and strings. It's the primary way to repeat an action across a collection of values in Swift.
  4. while & repeat-while Loops — A while loop repeats as long as a condition remains true, checked before each iteration. A repeat-while loop guarantees the body runs at least once before checking the condition.

Why This Week Matters: Swift's switch with pattern matching is one of the most powerful control flow tools in any language. It eliminates entire chains of if-else statements and makes complex logic readable, safe, and exhaustive at compile time.

Code Example

for i in 1...5 {                  // closed range loop
    switch i {
    case 1, 2: print("low")
    case 3...5: print("high: \(i)")
    default: break
    }
}

🛠 Weekly Task

Week 5 — Functions & Closures

Structure logic into reusable functions and learn Swift's powerful closure syntax.

Core Focus

Structuring logic into reusable functions and learning Swift's powerful closure syntax.

Topics & Subtopics

  1. Function Declaration & Parameters — Functions are declared with func, specifying parameter names, types, and an optional return type. Swift supports parameter labels that make function calls read like natural language.
  2. Default & Variadic Parameters — A default parameter value lets you omit an argument when calling a function, falling back to a preset value. A variadic parameter (using ...) accepts any number of values of the same type.
  3. Multiple Return Values (Tuples) — Swift functions can return multiple values at once by bundling them into a tuple. This is useful for returning related results, such as a minimum and maximum, from a single function.
  4. Closures (Anonymous Functions) — A closure is a self-contained block of functionality that can be passed around and executed later, similar to a lambda. Closures capture variables from their surrounding context, enabling powerful callback patterns.

Why This Week Matters: Closures are everywhere in Swift — from .map() and .filter() to completion handlers and SwiftUI's view builders. The trailing closure syntax and shorthand argument names ($0, $1) are patterns you will read and write daily.

Code Example

func calculate(_ a: Int, _ b: Int, using operation: (Int, Int) -> Int) -> Int {
    return operation(a, b) // closure passed as parameter
}

let result = calculate(5, 3) { $0 + $1 } // trailing closure syntax
print(result)

🛠 Weekly Task

Write a variadic function total(_ numbers: Int...) -> Int that sums any number of integers. Then write a closure stored in a variable that multiplies two numbers, and pass it into a function that uses it to compute a result.

Week 6 — Structs, Classes & Enums

Model real-world data using Swift's value types and reference types.

Core Focus

Modeling real-world data using Swift's value types (structs) and reference types (classes).

Topics & Subtopics

  1. Structs (Value Types) — A struct groups related properties and methods into a single custom type, and is copied by value whenever assigned or passed. Structs are Swift's preferred way to model simple, independent data.
  2. Classes (Reference Types) & Inheritance — A class is similar to a struct but is passed by reference and supports inheritance, letting one class extend another's behavior. Classes are used when shared mutable state or object hierarchies are needed.
  3. Properties & Methods — Properties store data on a type, while methods define behavior or actions that type can perform. Computed properties can calculate a value on-the-fly instead of storing it directly.
  4. Enums with Associated Values — An enum defines a fixed set of related values, such as directions or states, improving code safety and clarity. Swift enums can also attach associated data to each case, making them more powerful than simple constants.

Why This Week Matters: The struct-vs-class distinction is one of the most important design decisions in Swift. Choosing value types by default leads to safer, more predictable code. Swift enums with associated values are among the most expressive features in any modern language.

Code Example

enum Direction { case north, south, east, west }

struct Robot {
    var name: String
    var heading: Direction
    func describe() -> String { "\(name) faces \(heading)" } // method
}

let bot = Robot(name: "R2", heading: .north)
print(bot.describe())

🛠 Weekly Task

Phase 3 — Intermediate & Advanced Core

Weeks 7–9: OOP, file handling, error handling, and libraries.

Week 7 — Protocols, Extensions & Polymorphism

Understand Swift's protocol-oriented programming approach to OOP.

Core Focus

Understanding Swift's "protocol-oriented programming" approach to object-oriented design.

Topics & Subtopics

  1. Protocols (Defining Contracts) — A protocol defines a blueprint of methods and properties that a type must implement, without providing the implementation itself. Any struct, class, or enum can conform to a protocol, enabling shared behavior across unrelated types.
  2. Protocol Conformance & Polymorphism — A function can accept a protocol type as a parameter, allowing it to work with any conforming type interchangeably. This achieves polymorphism without requiring a shared class hierarchy.
  3. Extensions (Adding Functionality to Existing Types) — An extension adds new methods, computed properties, or protocol conformance to an existing type, even types you don't own like Int or String. This promotes clean, modular code organization.
  4. Inheritance vs Composition — Inheritance shares behavior through a class hierarchy, while composition combines small, focused protocols and structs to build complex behavior. Swift favors composition through protocols for more flexible, decoupled designs.

Why This Week Matters: Protocol-Oriented Programming is the foundation of Swift's design philosophy and the entire SwiftUI framework. Writing code against protocols instead of concrete types makes your code testable, modular, and far easier to change later.

Code Example

protocol Greetable { func greet() -> String }

struct Person: Greetable {
    let name: String
    func greet() -> String { "Hello, I'm \(name)" } // protocol conformance
}

func sayHello(to entity: Greetable) { print(entity.greet()) }
sayHello(to: Person(name: "Mia"))

Week 8 — Error Handling & File I/O

Write robust, production-grade code that anticipates and recovers from failure.

Core Focus

Writing robust, production-grade code that anticipates and recovers from failure.

Topics & Subtopics

  1. The Error Protocol & Custom Errors — Swift models recoverable errors as values conforming to the Error protocol, often defined using an enum. This provides clear, typed information about what went wrong.
  2. throws, try, try?, try! — A function marked throws can propagate an error to its caller, who must handle it using try. try? converts a thrown error into an Optional, while try! force-unwraps and crashes if an error occurs.
  3. do-catch Blocks — A do-catch block executes code that might throw an error and lets you handle specific error cases gracefully. This is Swift's primary structure for managing recoverable runtime failures.
  4. Reading & Writing Files (FileManager, String contents) — FileManager provides access to the filesystem for creating, reading, and deleting files and directories. Strings and Data can be written to and read from disk using simple, built-in initializers.

Why This Week Matters: Every production app deals with failure — network timeouts, missing files, corrupt data. Swift's typed error system forces you to think about failure paths at compile time, making your programs dramatically more reliable than unchecked exception systems.

Code Example

enum FileError: Error { case notFound }

func readContent(path: String) throws -> String {
    guard FileManager.default.fileExists(atPath: path) else {
        throw FileError.notFound // throw a custom error
    }
    return try String(contentsOfFile: path, encoding: .utf8)
}

Week 9 — Generics, Modules & Popular Frameworks

Write reusable, type-safe code and leverage Swift's package ecosystem.

Core Focus

Writing reusable, type-safe code and leveraging Swift's package ecosystem.

Topics & Subtopics

  1. Generics (Type Parameters) — Generics let you write functions and types that work with any type while preserving full compile-time type safety. They eliminate code duplication that would otherwise require separate versions for each type.
  2. Swift Package Manager (SPM) — Swift Package Manager handles building, testing, and managing external dependencies for Swift projects via a Package.swift manifest. It's Swift's built-in, official dependency management solution.
  3. Codable for JSON Encoding/Decoding — The Codable protocol allows structs and classes to automatically convert to and from JSON with minimal code. Combined with JSONEncoder / JSONDecoder, it's the standard way to handle structured data exchange.
  4. Networking Basics with URLSession — URLSession provides the tools to make HTTP requests and fetch data from web APIs directly within Swift. It's commonly paired with Codable to decode JSON responses into native Swift types.

Why This Week Matters: Codable and URLSession are used in virtually every iOS and macOS app that touches the internet. Generics make your own types and functions as flexible as Swift's built-in collections. These are the tools of day-to-day professional Swift development.

Code Example

struct User: Codable { let name: String; let age: Int } // auto Codable

func decodeUser(from json: Data) throws -> User {
    return try JSONDecoder().decode(User.self, from: json) // generic decode
}

func makePair<T>(_ a: T, _ b: T) -> [T] { [a, b] } // generic function

Phase 4 — Master Level Concepts

Weeks 10–12: Concurrency, functional patterns, and three complete capstone projects.

Week 10 — Concurrency: async/await & Tasks

Harness Swift's modern structured concurrency model for responsive applications.

Core Focus

Harnessing Swift's modern structured concurrency model for responsive, non-blocking applications.

Topics & Subtopics

  1. async Functions & await — An async function can pause execution while waiting for a long-running operation, without blocking the thread. The await keyword marks the point where execution suspends until the result is ready.
  2. Task — Launching Concurrent Work — A Task creates a new unit of asynchronous work that runs concurrently with the rest of your program. Tasks can be awaited, cancelled, or grouped together for coordinated execution.
  3. async let & Structured Concurrency — async let runs multiple asynchronous operations in parallel and lets you await their combined results together. This structured approach ensures child tasks complete before their parent scope exits.
  4. Actors — Safe Shared Mutable State — An actor is a reference type that automatically protects its internal state from simultaneous access by multiple tasks. This prevents data races without requiring manual locks.

Why This Week Matters: Swift's structured concurrency model is one of the most innovative concurrency systems in any mainstream language. Understanding async/await and actor is mandatory for modern iOS development — virtually every network call and UI update uses these tools.

Code Example

func fetchData(id: Int) async -> String {
    try? await Task.sleep(nanoseconds: 500_000_000) // simulate delay
    return "Data for id \(id)"
}

Task {
    async let first = fetchData(id: 1)
    async let second = fetchData(id: 2)
    print(await [first, second]) // wait for both concurrently
}

Week 11 — Generators, Decorators & Functional Patterns

Simulate advanced patterns from other languages using Swift's native features.

Core Focus

Simulating advanced patterns from other languages using Swift's native, idiomatic features.

Topics & Subtopics

  1. "Generators" via AsyncStream / Iterators — Swift doesn't have native generator syntax, but AsyncStream lets you lazily produce a sequence of asynchronous values over time, much like a generator. Custom IteratorProtocol conformance achieves similar lazy, on-demand value production synchronously.
  2. "Decorators" via Function Wrapping & Property Wrappers — Since Swift lacks Python-style decorators, the same effect is achieved by wrapping a function inside another to add behavior before and after a call. @propertyWrapper provides a more native Swift mechanism to inject reusable logic around property access.
  3. Higher-Order Functions (map, filter, reduce) — map, filter, and reduce are built-in higher-order functions that transform, select, and combine elements in a collection respectively. They enable concise, expressive, declarative-style data processing.
  4. Custom Property Wrappers — A @propertyWrapper lets you encapsulate reusable logic, like validation or clamping, around a property's getter and setter. This is a clean, native alternative to runtime decorators found in other languages.

Why This Week Matters: map, filter, and reduce are used constantly in professional Swift. Property wrappers power SwiftUI's @State, @Binding, and @Published — understanding them reveals how the entire SwiftUI data-flow model works under the hood.

Code Example

@propertyWrapper
struct Clamped {
    private var value: Int
    let range: ClosedRange<Int>
    var wrappedValue: Int {
        get { value }
        set { value = min(max(newValue, range.lowerBound), range.upperBound) }
    }
    init(wrappedValue: Int, _ range: ClosedRange<Int>) {
        self.range = range
        self.value = min(max(wrappedValue, range.lowerBound), range.upperBound)
    }
}

Code Example

struct Source { let url: String }

func fetch(_ source: Source) async -> String {
    "fetched: \(source.url)" // simulated network fetch
}

Task {
    let sources = [Source(url: "a.com"), Source(url: "b.com")]
    let results = await withTaskGroup(of: String.self) { group in
        for s in sources { group.addTask { await fetch(s) } }
        return await group.reduce(into: []) { $0.append($1) }
    }
    print(results)
}
