CSS: Beginner to Master

Complete 14-Week Curriculum — From Selectors & Box Model to Architecture, Animations & Production Component Libraries

About This Curriculum

This is a 14-week, phase-based CSS curriculum designed to take a complete beginner to a professional-level CSS developer. The roadmap moves from syntax, selectors, the cascade, and the box model through Flexbox, Grid, responsive design, and typography, into animations, transforms, Sass/SCSS, and CSS frameworks, all the way to CSS architecture, performance, accessibility, and three complete capstone projects.

The curriculum is organized into four phases: Phase 1 covers absolute basics including syntax and core visual properties, Phase 2 covers layout systems and responsive design, Phase 3 covers intermediate and advanced topics like animations, preprocessors, and component systems, and Phase 4 covers master-level architecture, performance, accessibility, and creative techniques.

By Week 14, you will have built three complete, portfolio-ready projects and be confident architecting scalable, accessible, and performant CSS for any size application.

Level: Beginner → Master  |  Duration: 14 weeks  |  Stack: CSS3, Sass/SCSS, Tailwind CSS  |  Prerequisites: Basic HTML knowledge

Phase 1 — Absolute Basics

Weeks 1–3: Fundamentals, syntax, and core visual properties.

Week 1 — Introduction to CSS & Core Syntax

Understand what CSS is, how it connects to HTML, and write your very first style rules.

Core Focus

Understanding what CSS is, how it connects to HTML, and writing your very first style rules.

Topics & Subtopics

  1. What is CSS & How It Works — CSS (Cascading Style Sheets) is the language used to describe the visual presentation of HTML documents. It separates content (HTML) from design, making web pages easier to maintain and style consistently.
  2. CSS Syntax: Rules, Properties & Values — A CSS rule consists of a selector (what to style) and a declaration block (how to style it). Each declaration is a property-value pair separated by a colon and ended with a semicolon.
  3. Three Ways to Add CSS (Inline, Internal, External) — CSS can be written directly on an element (inline), inside a <style> tag (internal), or in a separate .css file (external). External stylesheets are the professional standard because they keep concerns separated.
  4. CSS Colors (Named, Hex, RGB, HSL) — CSS supports multiple color formats: named colors (red), hex codes (#ff0000), RGB (rgb(255,0,0)), and HSL (hsl(0,100%,50%)). All four can represent the same color — choose the format that fits your workflow.

Why This Week Matters: CSS is how every website gets its visual identity. Understanding the selector-declaration pattern and the three ways to apply CSS from Day 1 prevents the most common beginner mistake — mixing concerns between HTML structure and visual presentation.

Code Example

/* CSS is linked to HTML via a link tag in the head */
/* This rule makes ALL paragraph text blue and 18px large */
p {
  color: blue;         /* text color */
  font-size: 18px;     /* text size in pixels */
}

/* selector { property: value; } — the core pattern of CSS */
h1 {
  color: #333333;      /* dark gray text */
  font-size: 32px;
  font-weight: bold;
}

/* Four ways to write the same red color */
.box-named  { color: red; }
.box-hex    { color: #ff0000; }
.box-rgb    { color: rgb(255, 0, 0); }
.box-hsl    { color: hsl(0, 100%, 50%); }

🛠 Weekly Task

Create an index.html file linked to an external styles.css. Style the page title (h1), a paragraph (p), and the body background using at least three different color formats. Add comments to every rule explaining what it does.

Week 2 — Selectors & The Cascade

Learn how CSS selectors target elements and understand specificity and cascade rules.

Core Focus

Learning how CSS selectors target elements and understanding specificity and cascade rules.

Topics & Subtopics

  1. Basic Selectors (Element, Class, ID) — Element selectors target all tags of that type; class selectors (.classname) target reusable groups; ID selectors (#idname) target a single unique element. Classes are preferred in professional development for their reusability.
  2. Combinator Selectors (Descendant, Child, Sibling) — Combinator selectors let you target elements based on their relationship in the HTML tree. Descendant (space), direct child (>), adjacent sibling (+), and general sibling (~) give you precise control.
  3. Pseudo-Classes & Pseudo-Elements — Pseudo-classes (:hover, :focus, :nth-child) style elements based on state or position. Pseudo-elements (::before, ::after, ::first-line) style specific parts of an element's content.
  4. Specificity & The Cascade — When multiple rules target the same element, the cascade resolves conflicts using specificity scores: IDs (100) > classes (10) > elements (1). The !important flag overrides all — use it sparingly.

Why This Week Matters: Specificity wars are the most common source of CSS frustration — rules that "don't work" almost always lose to a more specific rule. Understanding the cascade calculation makes you able to predict exactly which rule wins without trial and error.

Code Example

p      { color: gray; }          /* element — all p tags */
.intro { color: navy; }          /* class — any element with class="intro" */
#hero  { font-size: 48px; }      /* ID — the single element with id="hero" */

nav a        { color: white; }        /* descendant: a inside nav */
ul > li      { list-style: none; }    /* child: direct li of ul */
h2 + p       { color: gray; }         /* adjacent sibling: first p after h2 */

a:hover          { color: orange; }        /* style link on mouse hover */
input:focus      { border-color: blue; }   /* style focused form field */
p::first-line    { font-weight: bold; }    /* bold only the first line */

p            { color: gray; }    /* specificity: 001 */
.intro       { color: navy; }    /* specificity: 010 — wins over p */
#hero        { color: black; }   /* specificity: 100 — wins over .intro */

🛠 Weekly Task

Build a navigation bar in HTML. Use descendant, child, and pseudo-class selectors to style links, hover states, and active states. Test specificity conflicts by overriding element styles with classes and IDs.

Week 3 — The Box Model & Display

Understand how every HTML element is a box and how spacing, borders, and sizing work.

Core Focus

Understanding how every HTML element is a box and how spacing, borders, and sizing work.

Topics & Subtopics

  1. The Box Model (Content, Padding, Border, Margin) — Every HTML element is rendered as a rectangular box with four layers: content, padding (inner space), border, and margin (outer space). Mastering the box model is essential for controlling layout and spacing.
  2. box-sizing: border-box — By default (content-box), padding and border are added on top of the declared width. Setting box-sizing: border-box makes the declared width include padding and border — far more intuitive for layout.
  3. Display Property (block, inline, inline-block, none) — The display property controls how an element participates in layout. Block elements stack vertically; inline elements flow horizontally; inline-block allows width/height on inline elements; none hides the element.
  4. Width, Height, Overflow & Units (px, %, em, rem) — CSS offers multiple units: px (fixed), % (relative to parent), em (relative to parent font-size), and rem (relative to root font-size). Choosing the right unit makes designs flexible and accessible.

Why This Week Matters: The box model is the single most fundamental concept in CSS layout. Every layout problem — unwanted gaps, elements overflowing, sizes not matching expectations — traces back to a misunderstanding of how content, padding, border, and margin interact. border-box is the first line of every professional stylesheet.

Code Example

/* Apply border-box sizing globally — industry standard reset */
*, *::before, *::after {
  box-sizing: border-box; /* width now includes padding + border */
}

.card {
  width: 300px;            /* content width */
  padding: 20px;           /* inner spacing around content */
  border: 2px solid #ccc;  /* visible border around padding */
  margin: 16px;            /* outer spacing from other elements */
}

span   { display: inline-block; width: 100px; } /* inline but accepts dimensions */
div    { display: block; }                        /* takes full row */
.hide  { display: none; }                         /* hidden from page and layout */

.container {
  width: 90%;              /* 90% of parent width */
  max-width: 1200px;       /* never wider than 1200px */
  font-size: 1rem;         /* relative to root (default 16px) */
  overflow: hidden;        /* clip content that exceeds bounds */
}

🛠 Weekly Task

Create a profile card component using the full box model. Set explicit widths, paddings, borders, and margins. Switch between content-box and border-box and observe how total size changes. Use rem for all typography sizing.

Phase 2 — Control Flow & Layout

Weeks 4–6: Flexbox, Grid, responsive design, and visual styling.

Week 4 — Typography & Visual Styling

Master font, text, and background properties to produce professional-looking designs.

Core Focus

Mastering font, text, and background properties to produce professional-looking designs.

Topics & Subtopics

  1. Web Fonts & font-family — The font-family property sets the typeface. Always provide a font stack (preferred → fallback → generic) so the browser has options if a font isn't available. Google Fonts can be imported via a <link> tag.
  2. Text Properties (size, weight, line-height, letter-spacing) — Fine-tuning typography makes content more readable and visually balanced. line-height controls vertical space between lines; letter-spacing adjusts horizontal character spacing — both are critical for readability.
  3. Background Properties (color, image, gradient, size, position) — CSS backgrounds support solid colors, image URLs, and generated gradients. Control placement with background-position, scale with background-size: cover, and layer multiple backgrounds using comma syntax.
  4. CSS Custom Properties (Variables) — CSS variables (declared with --) let you define reusable values in one place and reference them anywhere with var(). Changing the variable updates every usage — ideal for theming and design tokens.

Why This Week Matters: CSS custom properties are the foundation of every modern design system. Defining all colors, spacing, and font sizes as variables at the root level means you can retheme an entire application by changing a handful of values — the pattern used by every major design system including Material and Carbon.

Code Example

body {
  font-family: 'Inter', 'Helvetica Neue', Arial, sans-serif;
}

p {
  font-size: 1rem;         /* base size */
  font-weight: 400;        /* normal weight */
  line-height: 1.6;        /* 1.6x the font-size — ideal for body text */
  letter-spacing: 0.02em;  /* subtle character spacing */
}

.hero {
  background: linear-gradient(135deg, #667eea, #764ba2);
  background-size: cover;
  background-position: center;
}

:root {
  --color-primary: #4f46e5;
  --font-size-base: 1rem;
}
h1 { color: var(--color-primary); font-size: calc(var(--font-size-base) * 2); }

🛠 Weekly Task

Week 5 — Flexbox Layout

Master Flexbox — the most important one-dimensional layout system in modern CSS.

Core Focus

Mastering Flexbox — the most important one-dimensional layout system in modern CSS.

Topics & Subtopics

  1. Flex Container Properties (display:flex, flex-direction, flex-wrap) — Setting display: flex on a parent activates Flexbox for its direct children. flex-direction sets the main axis (row or column); flex-wrap allows children to wrap to the next line instead of overflowing.
  2. Alignment (justify-content, align-items, align-content) — justify-content distributes children along the main axis; align-items aligns them on the cross axis. Together, they enable centering — one of the most common CSS tasks — in just two lines.
  3. Flex Item Properties (flex-grow, flex-shrink, flex-basis) — These three properties control how individual flex children grow or shrink relative to available space. The shorthand flex: 1 means grow equally, shrink equally, and start at 0 basis — perfect for equal columns.
  4. Flexbox Patterns (Navbar, Card Row, Holy Grail) — Flexbox excels at common UI patterns: navbars, card rows, and the classic "holy grail" layout. Understanding these patterns gives you reusable mental templates for virtually any one-dimensional layout challenge.

Why This Week Matters: Flexbox replaced floats and replaced table-based layouts entirely. It is the go-to tool for navbars, card rows, and any linear arrangement. The two-line centering trick (justify-content: center + align-items: center) alone will save you hours of frustration that plagued web developers before 2015.

Code Example

.navbar {
  display: flex;
  flex-direction: row;
  flex-wrap: wrap;
}

.card-grid {
  display: flex;
  justify-content: space-between;
  align-items: center;
}

.sidebar { flex: 0 0 280px; }  /* fixed 280px, never grow or shrink */
.main    { flex: 1; }          /* take all remaining space */
.aside   { flex: 0 0 200px; }  /* fixed 200px right column */

/* Perfect vertical + horizontal centering */
.modal-overlay {
  display: flex;
  justify-content: center;
  align-items: center;
  min-height: 100vh;
}

🛠 Weekly Task

Build a responsive navigation bar using Flexbox (logo left, links right). Then create a row of three feature cards that stack on mobile using flex-wrap. Finally, center a modal dialog perfectly on screen using Flexbox.

Week 6 — CSS Grid & Responsive Design

Master CSS Grid for two-dimensional layouts and learn responsive design with media queries.

Core Focus

Mastering CSS Grid for two-dimensional layouts and learning responsive design with media queries.

Topics & Subtopics

  1. Grid Container Basics (grid-template-columns, rows, gap) — CSS Grid defines a two-dimensional layout using rows and columns. grid-template-columns sets column sizes; gap adds gutters between cells. The fr unit distributes space fractionally.
  2. Grid Placement (grid-column, grid-row, grid-area) — Items can span multiple columns or rows using grid-column: 1 / 3. Named areas via grid-template-areas provide a visual map of your layout structure directly in CSS.
  3. Responsive Design & Media Queries — Media queries apply styles only when certain conditions are met — most commonly viewport width. Use a mobile-first approach: write base styles for small screens, then add min-width breakpoints for larger screens.
  4. auto-fill, auto-fit & minmax() — auto-fill and auto-fit with minmax() create self-adapting grid columns without any media queries. Items automatically reflow based on available space — perfect for image galleries and card grids.

Why This Week Matters: CSS Grid is the most powerful layout system ever added to CSS. The repeat(auto-fit, minmax(280px, 1fr)) pattern creates a fully responsive card grid with zero media queries — a single line of CSS that would have required dozens of lines and JavaScript in the past.

Code Example

.layout {
  display: grid;
  grid-template-columns: 1fr 2fr 1fr; /* three columns: 25% / 50% / 25% */
  gap: 24px;
}

.header  { grid-column: 1 / -1; }  /* span ALL columns */
.sidebar { grid-column: 1 / 2; }
.main    { grid-column: 2 / 4; }
.footer  { grid-column: 1 / -1; }

/* Mobile first responsive */
.card-grid { grid-template-columns: 1fr; }
@media (min-width: 768px)  { .card-grid { grid-template-columns: 1fr 1fr; } }
@media (min-width: 1024px) { .card-grid { grid-template-columns: repeat(3, 1fr); } }

/* Responsive without media queries */
.gallery {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(280px, 1fr));
  gap: 20px;
}

🛠 Weekly Task

Build a full-page portfolio layout using CSS Grid: a fixed header, sidebar navigation, main content area, and footer. Make it fully responsive using media queries and a mobile-first strategy. Add a responsive image gallery using auto-fit + minmax.

Phase 3 — Intermediate & Advanced Core

Weeks 7–10: Animations, preprocessors, frameworks, and component systems.

Week 7 — Transforms, Transitions & Animations

Bring your UI to life with smooth CSS transforms, transitions, and keyframe animations.

Core Focus

Bringing your UI to life with smooth CSS transforms, transitions, and keyframe animations.

Topics & Subtopics

  1. CSS Transforms (translate, scale, rotate, skew) — The transform property moves, scales, rotates, or skews elements without affecting page flow. Transforms are GPU-accelerated and do not trigger layout reflow — making them the go-to for performant visual effects.
  2. CSS Transitions — Transitions animate property changes smoothly over a specified duration. Define what to transition, how long it takes, and the easing function. Use transition: all sparingly — it's better to specify individual properties.
  3. CSS Keyframe Animations (@keyframes) — @keyframes defines multi-step animations. Assign them with animation-name, animation-duration, and animation-iteration-count. Combine with animation-fill-mode: forwards to keep the final state.
  4. Performance: will-change & Animation Best Practices — Only animate transform and opacity for smooth 60fps performance — they don't trigger layout or paint. Use will-change: transform to hint the browser to pre-optimize, but use it sparingly to avoid memory overhead.

Why This Week Matters: The difference between a hobbyist and a professional CSS developer is understanding which properties are safe to animate. Animating width, height, or top causes layout reflow on every frame — your animation drops to single-digit fps. Animating only transform and opacity keeps it at a silky 60fps.

Code Example

.card:hover {
  transform: translateY(-8px) scale(1.02);
}

.button {
  background-color: #4f46e5;
  transition: background-color 0.3s ease, transform 0.2s ease;
}
.button:hover { background-color: #4338ca; transform: scale(1.05); }

@keyframes fadeInUp {
  from { opacity: 0; transform: translateY(30px); }
  to   { opacity: 1; transform: translateY(0); }
}
.hero-text { animation: fadeInUp 0.6s ease-out forwards; }

.animated-element {
  will-change: transform, opacity;
  transition: transform 0.3s ease, opacity 0.3s ease;
}

🛠 Weekly Task

Create an interactive product card: on hover, it lifts with translateY, scales up, and its image zooms. Add a "fade in up" entrance animation triggered by a CSS class toggle. Build a loading spinner using @keyframes with infinite rotation.

Week 8 — Pseudo-Elements, Clipping & Advanced Selectors

Use advanced CSS techniques to craft complex visual effects without extra HTML.

Core Focus

Using advanced CSS techniques to craft complex visual effects without extra HTML.

Topics & Subtopics

  1. Advanced Pseudo-Elements (::before, ::after) — ::before and ::after insert generated content before/after an element's content. They require content: "" to render and are treated as inline children — useful for decorative elements, badges, and tooltips.
  2. clip-path & Shape Clipping — clip-path clips an element to a geometric shape, revealing only the clipped area. Combine with transition for animated shape reveals. Tools like Clippy help generate polygon values visually.
  3. CSS Counters — CSS counters allow automatic numbering without JavaScript. Define a counter with counter-reset, increment it with counter-increment, and display it using content: counter() in a pseudo-element.
  4. :is(), :where(), :has() — Modern Selectors — :is() and :where() simplify complex selector lists. :has() — a "parent selector" — lets you style a parent based on its children's state, enabling patterns previously only possible with JavaScript.

Why This Week Matters: :has() is arguably the most powerful selector ever added to CSS. Being able to style a parent based on what it contains — without JavaScript — unlocks an entirely new class of CSS-only UI patterns. ::before and ::after effectively give every element two free extra child elements for decoration.

Code Example

.section-title::after {
  content: "";
  display: block;
  width: 60px;
  height: 3px;
  background: #4f46e5;
  margin-top: 8px;
}

.hero-image {
  clip-path: polygon(0 0, 100% 0, 100% 85%, 0 100%);
  transition: clip-path 0.4s ease;
}

ol.steps { counter-reset: step-counter; }
ol.steps li { counter-increment: step-counter; }
ol.steps li::before {
  content: "Step " counter(step-counter);
  font-weight: bold;
}

/* :is() — simplify repeated selectors */
:is(h1, h2, h3) { font-family: 'Georgia', serif; }

/* :has() — style a card differently if it contains an image */
.card:has(img)  { padding: 0; overflow: hidden; }

🛠 Weekly Task

Build a styled numbered steps component using CSS counters and ::before. Create a "ribbon" badge on a card using ::before and ::after absolutely positioned. Apply clip-path to a hero section image for a diagonal crop effect.

Week 9 — Sass/SCSS Preprocessor

Write more maintainable, modular, and powerful CSS with variables, nesting, and mixins.

Core Focus

Learning Sass to write more maintainable, modular, and powerful CSS with variables, nesting, and mixins.

Topics & Subtopics

  1. Sass Variables & Nesting — Sass variables (prefixed with $) store reusable values. Nesting mirrors HTML structure in your styles, reducing repetition — but avoid nesting more than 3 levels deep to keep specificity manageable.
  2. Sass Mixins & @include — Mixins are reusable blocks of CSS declarations, optionally accepting arguments. Use @mixin to define and @include to use. They're ideal for vendor-prefixed properties, media query shortcuts, or component patterns.
  3. Sass @extend & Placeholder Selectors — @extend lets one selector inherit styles from another. %placeholder selectors define extendable rules without outputting CSS on their own — useful for base patterns shared across multiple components.
  4. Sass Partials, @use & File Architecture (7-1 Pattern) — Split SCSS into partials (files prefixed with _) and import them with @use. The 7-1 pattern organizes partials into seven folders (abstracts, base, components, layout, pages, themes, vendors) and one main file.

Why This Week Matters: Even though native CSS now has variables and nesting, Sass remains the industry standard for large codebases. The 7-1 architecture pattern is used in production at companies worldwide. Understanding Sass mixins and the @use module system makes working on any professional CSS codebase immediately approachable.

Code Example

/* SCSS syntax — compiled to standard CSS */
$color-primary: #4f46e5;
$spacing-lg: 2rem;

.navbar {
  background: $color-primary;
  a { color: white; &:hover { opacity: 0.8; } }
}

@mixin flex-center($direction: row) {
  display: flex;
  justify-content: center;
  align-items: center;
  flex-direction: $direction;
}
.hero { @include flex-center(column); }

%button-base {
  padding: 0.75rem 1.5rem;
  border-radius: 6px;
  font-weight: 600;
  cursor: pointer;
}
.btn-primary { @extend %button-base; background: #4f46e5; }

/* main.scss — imports all partials */
@use 'abstracts/variables';
@use 'base/reset';
@use 'components/button';
@use 'layout/grid';

🛠 Weekly Task

Refactor a plain CSS project into SCSS using the 7-1 pattern. Create mixins for flex-center, responsive breakpoints, and a button generator. Define all design tokens as Sass variables and replace all magic numbers in your CSS.

Week 10 — CSS Frameworks & Component Systems

Learn to use Tailwind CSS and understand utility-first design, then explore building a component design system.

Core Focus

Learning to use Tailwind CSS and understanding utility-first design, then exploring building a component design system.

Topics & Subtopics

  1. Utility-First CSS with Tailwind CSS — Tailwind provides low-level utility classes (flex, pt-4, text-center) that compose directly in HTML. This removes the need to write custom CSS for most styling tasks and eliminates the cognitive overhead of naming classes.
  2. Tailwind Configuration & Custom Design Tokens — Tailwind's tailwind.config.js lets you extend or replace the default design system with your brand tokens. Custom colors, fonts, spacing, and breakpoints integrate seamlessly with all utility classes.
  3. CSS Modules & Scoped Styles — CSS Modules scope class names locally to the component file, preventing global style collisions. They're native to React, Vue, and Next.js. Class names are transformed into unique hashes at build time.
  4. Building a Component Design System — A design system defines reusable components (buttons, cards, inputs) built on shared design tokens. It ensures visual consistency across a product and is the standard at companies like Google (Material), IBM (Carbon), and Airbnb.

Why This Week Matters: Tailwind CSS is now the most popular CSS framework in the world, used by companies from startups to Fortune 500s. Understanding utility-first CSS and how to configure a design token system is a required skill for most modern frontend job descriptions.

Code Example

/* Traditional CSS */
.card { display: flex; padding: 1rem; border-radius: 8px; background: white; }

/* Tailwind equivalent in HTML: */
/* <div class="flex p-4 rounded-lg bg-white shadow-md"> */

/* CSS Modules — Button.module.css */
.button {
  background: #4f46e5;
  padding: 0.75rem 1.5rem;
  border-radius: 6px;
}
.button:hover { background: #4338ca; }

/* Design token layer */
:root {
  --color-action: #4f46e5;
  --radius-md: 6px;
  --space-md: 1rem;
}
/* Component layer — consumes tokens */
.btn {
  background: var(--color-action);
  border-radius: var(--radius-md);
  padding: var(--space-md);
}

Phase 4 — Master Level Concepts

Weeks 11–14: CSS architecture, performance, accessibility, creative techniques, and three complete projects.

Week 11 — CSS Architecture & Methodology

Learn industry-standard CSS architecture patterns to write scalable, maintainable stylesheets.

Core Focus

Learning industry-standard CSS architecture patterns to write scalable, maintainable stylesheets.

Topics & Subtopics

  1. BEM (Block, Element, Modifier) Naming Convention — BEM is a naming methodology that organizes CSS into Blocks (standalone components), Elements (children, using __), and Modifiers (variants, using --). It prevents naming collisions and makes components self-documenting.
  2. ITCSS (Inverted Triangle CSS) — ITCSS organizes CSS from most-global to most-specific: Settings → Tools → Generic → Elements → Objects → Components → Utilities. This controls the cascade deliberately and prevents specificity wars at scale.
  3. CSS Logical Properties — Logical properties (margin-inline-start, padding-block-end) replace directional properties (margin-left, padding-bottom). They adapt automatically to writing direction (ltr/rtl), making internationalisation trivial.
  4. CSS Layers (@layer) — @layer lets you explicitly control the cascade order of style groups. Styles in a lower layer lose to higher layers regardless of specificity — solving specificity wars between resets, frameworks, and custom code.

Why This Week Matters: Every CSS codebase without architecture eventually becomes unmaintainable. BEM and ITCSS are the two most adopted methodologies in the industry. @layer is the biggest improvement to the CSS cascade since specificity was introduced — it lets you integrate third-party CSS and utilities without specificity battles.

Code Example

/* BEM pattern: .block__element--modifier */
.card { }
.card__title { }
.card__image { }
.card--featured { border: 2px solid gold; }

/* ITCSS Generic layer */
*, *::before, *::after { box-sizing: border-box; margin: 0; padding: 0; }
/* ITCSS Components layer */
.c-card { /* scoped component styles with c- namespace */ }

/* Logical properties */
.card { margin-inline-start: 1rem; }    /* works in LTR and RTL */
.button { padding-block: 0.75rem; padding-inline: 1.5rem; }

/* @layer cascade control */
@layer reset, base, components, utilities;
@layer reset     { * { box-sizing: border-box; } }
@layer components { .btn { background: blue; } }
@layer utilities  { .bg-red { background: red; } }

🛠 Weekly Task

Refactor a complex multi-component CSS codebase using BEM naming and ITCSS file structure. Add @layer declarations to control the cascade between a third-party reset, your base styles, and utility overrides. Convert all directional properties to logical properties.

Week 12 — Accessibility, Performance & Modern CSS

Write CSS that is accessible, performant, and uses the latest CSS features.

Core Focus

Writing CSS that is accessible, performant, and uses the very latest CSS features available in modern browsers.

Topics & Subtopics

  1. CSS Accessibility (Focus Styles, Reduced Motion, High Contrast) — Accessible CSS ensures keyboard users see clear focus indicators and respects user system preferences. Never use outline: none without a custom replacement; always support prefers-reduced-motion for users sensitive to animation.
  2. CSS Performance Optimization — Expensive CSS properties (box-shadow, filter, border-radius on large elements) can cause paint performance issues. Use browser DevTools Performance panel to identify paint bottlenecks; prefer transform/opacity for animations.
  3. CSS Container Queries — Container queries allow components to respond to their container's size rather than the viewport. This enables truly portable components that adapt no matter where they're placed — a landmark upgrade over media queries.
  4. CSS Native Nesting & @scope — CSS Nesting (now in all modern browsers) brings Sass-like nesting to native CSS without a build tool. @scope lets you limit styles to a DOM subtree. Together with @layer, these form the new architecture of modern CSS.

Why This Week Matters: Container queries are the most requested CSS feature for over a decade — they finally make components truly independent of their context. prefers-reduced-motion support is now an accessibility requirement, not an optional nicety. Native CSS nesting means you can write clean, structured CSS without any build tools.

Code Example

:focus-visible { outline: 3px solid #4f46e5; outline-offset: 3px; }

@media (prefers-reduced-motion: reduce) {

Week 13 — CSS Art, Creative Techniques & Debugging

Push CSS to its creative limits and develop expert-level debugging skills.

Core Focus

Pushing CSS to its creative limits and developing expert-level debugging skills using DevTools.

Topics & Subtopics

  1. CSS Gradients (Conic, Radial, Mesh & Advanced) — Beyond linear gradients, CSS supports radial (circular) and conic (angle-based) gradients. Layering gradients with background and using mask-image unlocks complex visual effects without any images or JavaScript.
  2. CSS Filters & Blend Modes — filter applies visual effects (blur, brightness, grayscale, drop-shadow) to elements. mix-blend-mode controls how an element blends with its background — enabling Photoshop-like compositing effects purely in CSS.
  3. CSS Grid Subgrid — subgrid allows a grid item's own children to align with the outer grid's tracks — solving the longstanding problem of nested content alignment. It's now supported in all major browsers.
  4. CSS Debugging Techniques — Use Chrome/Firefox DevTools to inspect computed styles, visualise the box model, and audit accessibility. The outline trick and CSS Grid/Flexbox overlays are invaluable for debugging layout issues without editing source files.

Why This Week Matters: Conic gradients and mix-blend-mode enable visual designs previously only achievable in Photoshop — without a single image file. Subgrid solves one of the oldest layout problems in CSS. Expert-level DevTools usage can cut debugging time from hours to minutes.

  );

}

}

  mix-blend-mode: overlay;
  color: white;

  display: grid;

}

/* Debug trick — visualise all elements */
