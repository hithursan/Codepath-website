Complete 15-Week Curriculum — From Data Fundamentals & SQL to Power BI, Tableau, Data Warehousing & BI Strategy

About This Curriculum

This is a 15-week, project-driven Business Intelligence curriculum designed to take a complete beginner to a job-ready, portfolio-equipped BI professional. The roadmap covers the complete BI stack: data fundamentals, Excel, SQL, Power BI, DAX, Tableau, data warehousing, ETL pipelines, KPIs, storytelling, Python for BI, and BI strategy.

The curriculum is organized into five phases: Phase 1 covers BI foundations including data concepts, Excel, and database modeling, Phase 2 covers SQL from fundamentals through advanced window functions, Phase 3 covers Power BI including DAX and dashboard design, Phase 4 covers Tableau, data warehousing, ETL, and BI strategy, and Phase 5 covers Python for BI, portfolio building, and the capstone project.

By Week 15, you will have built 12+ mini-projects and one comprehensive capstone BI solution covering the full stack from raw data to executive dashboards — ready for a BI Analyst, Data Analyst, or BI Developer role.

Level: Beginner → Professional  |  Duration: 15 weeks (~38–42 hrs/week)  |  Stack: SQL, Excel, Power BI, Tableau, Python  |  Prerequisites: Basic computer literacy

Phase 1 — Foundations

Weeks 1–3: BI concepts, Excel for analysis, database and data modeling fundamentals.

Week 1 — Introduction to Business Intelligence & Data Fundamentals

Build a solid conceptual foundation of what BI is, why it matters, and how to think like a BI professional.

Core Focus

Building a solid conceptual foundation of what BI is, how data flows through organizations, and how to think like a BI professional before touching any tool.

Topics & Subtopics

  1. What Business Intelligence Really Is — BI vs data science vs data analytics; the BI value chain (data → information → insight → decision → value); historical evolution from static reports to self-service analytics; real-world BI use cases across industries.
  2. The BI Ecosystem — Data sources (transactional databases, CRMs, ERPs, flat files, APIs); data integration tools (ETL and ELT concepts); data storage (databases, data warehouses, data lakes); analytical tools (SQL, Excel, Power BI, Tableau, Python); consumers (dashboards, reports, embedded analytics).
  3. Data Fundamentals — Structured vs unstructured vs semi-structured data; quantitative vs qualitative data; primary vs secondary data; data types (text, number, date/time, boolean, currency); data quality dimensions: accuracy, completeness, consistency, timeliness, validity, uniqueness.
  4. Organizational Context of BI — How BI teams are structured in organizations; stakeholders (executives, managers, analysts, IT); the difference between operational reporting and strategic analytics; BI requirements gathering — asking the right business questions.
  5. Key BI Metrics Concepts — What a KPI is and how to define a good one; SMART KPIs (Specific, Measurable, Achievable, Relevant, Time-bound); leading vs lagging indicators; common KPI categories (financial, operational, customer, HR).

Why This Week Matters: BI is not just about tools — it is about thinking about data in the context of business decisions. Without this conceptual foundation, you will use tools without understanding why, build dashboards that look good but mean nothing, and miss the business purpose behind the analysis. Every tool skill you learn in subsequent weeks only has value when anchored to this business context.

Week 2 — Data in Spreadsheets: Excel & Google Sheets for BI

Master Excel and Google Sheets as BI starter tools — for data cleaning, analysis, pivot tables, and basic visualization.

Core Focus

Mastering Excel and Google Sheets as BI starter tools — for data cleaning, analysis, pivot tables, and basic interactive visualization.

Topics & Subtopics

  1. Spreadsheet Fundamentals for BI — Rows vs columns, cells, ranges, named ranges; data formatting (date, number, text); freeze panes, sorting, filtering; Excel Tables (Ctrl+T) — why they are better than plain ranges for BI work.
  2. Essential Excel Functions for Data Analysis — Text functions (TRIM, CLEAN, LEFT, RIGHT, MID, CONCATENATE, LEN); lookup functions (VLOOKUP, HLOOKUP, INDEX-MATCH, XLOOKUP); logical functions (IF, IFS, AND, OR, IFERROR); statistical functions (SUM, AVERAGE, MEDIAN, COUNTIF, COUNTIFS, SUMIF, SUMIFS, AVERAGEIF); date functions (TODAY, YEAR, MONTH, DATEDIF, NETWORKDAYS).
  3. Data Cleaning in Excel — Removing duplicates; find and replace; text to columns; flash fill; handling blank cells and error values; converting data types.
  4. Pivot Tables — The Most Powerful BI Feature in Excel — Creating pivot tables from a data range; rows, columns, values, and filters; summarizing by sum/count/average/percentage; calculated fields; pivot table formatting; slicers and timelines for interactive filtering; grouping dates by year/quarter/month.
  5. Pivot Charts & Basic Excel Dashboard — Creating charts from pivot tables; chart types (bar, column, line, pie, scatter, combo); formatting charts professionally; arranging pivot charts and slicers on a dashboard sheet; protecting and sharing an Excel dashboard.

Why This Week Matters: Despite the rise of dedicated BI tools like Power BI and Tableau, Excel remains the most widely used business analysis tool in the world. Almost every BI job requires Excel proficiency. Pivot tables are conceptually the same as GROUP BY aggregations in SQL and visual aggregations in Power BI — learning them in Excel builds intuition that transfers everywhere.

Week 3 — Data Modeling & Database Fundamentals

Understand how data is structured in relational databases — tables, relationships, keys, and schemas.

Core Focus

Understanding how data is structured in relational databases — tables, relationships, keys, and schemas — the foundation on which all BI query work and reporting is built.

Topics & Subtopics

  1. Relational Database Concepts — Tables, rows (records), columns (fields/attributes); primary keys (uniquely identifying each row); foreign keys (linking related tables); relationships (one-to-one, one-to-many, many-to-many); referential integrity.
  2. Data Normalization — First Normal Form (1NF): no repeating groups; Second Normal Form (2NF): no partial dependencies; Third Normal Form (3NF): no transitive dependencies; why normalization matters for OLTP; why BI sometimes denormalizes for OLAP performance.
  3. Entity-Relationship (ER) Diagrams — Entities and attributes; reading and drawing ER diagrams; crow's foot notation; converting an ER diagram to a physical table design; tools: draw.io (free, browser-based).
  4. Data Warehousing Concepts — OLTP vs OLAP; what a data warehouse is and why it exists separately from transaction databases; dimensions and facts (the two key concepts in data warehouse modeling); the Star Schema (one fact table surrounded by dimension tables); the Snowflake Schema; data marts.
  5. Introduction to MySQL — Installing MySQL and MySQL Workbench; creating a database and tables; inserting sample data; running a basic SELECT query.

Why This Week Matters: Data modeling is the architecture that everything else in BI is built on. A BI professional who does not understand database relationships, primary and foreign keys, or the star schema cannot design a proper data warehouse, write correct multi-table SQL queries, or build accurate Power BI or Tableau reports. This week's concepts underpin every tool and technique in the roadmap.

Phase 2 — SQL for Business Intelligence

Weeks 4–6: SQL querying, aggregations, joins, and advanced window functions.

Week 4 — SQL Fundamentals for BI: Querying & Filtering

Write SQL queries to retrieve, filter, sort, and examine data — the foundational BI skill used every single day.

Core Focus

Writing SQL queries to retrieve, filter, sort, and examine data — the foundational BI skill that every analyst uses every single day.

Topics & Subtopics

  1. SELECT Statement Structure — SELECT, FROM, WHERE, ORDER BY, LIMIT; selecting specific columns vs SELECT *; column aliases; DISTINCT; NULL values (IS NULL, IS NOT NULL, COALESCE, IFNULL).
  2. Filtering with WHERE — Comparison operators (=, !=, >, <, >=, <=); logical operators (AND, OR, NOT); BETWEEN (inclusive range filtering); IN (filtering against a list); LIKE and wildcards (% and _); operator precedence with parentheses.
  3. Sorting and Limiting Results — ORDER BY (ASC and DESC); ORDER BY multiple columns; LIMIT and OFFSET for pagination; TOP (SQL Server) vs LIMIT (MySQL/PostgreSQL).
  4. String Functions — UPPER, LOWER, TRIM, LTRIM, RTRIM; SUBSTRING/SUBSTR; CONCAT; LENGTH/LEN; REPLACE.
  5. Date and Time Functions — NOW(), CURDATE(), CURRENT_DATE; YEAR(), MONTH(), DAY(); DATEDIFF(), DATE_ADD(), DATE_SUB(); DATE_FORMAT().

Why This Week Matters: SELECT queries are how BI professionals extract data from databases. Before you can aggregate, join, or transform data, you must be able to retrieve and filter it precisely. Every BI tool — Power BI, Tableau, Looker — generates SQL under the hood, and understanding SQL means you can debug and optimize what your tools produce.

🛠 Weekly Mini Project

A "SQL Data Extraction Report" — using the retail database built in Week 3 (or a provided sample database), write at least 15 SQL queries that answer specific business questions. Questions should include: finding high-value customers, identifying top-selling products, filtering orders by date range, finding incomplete records (NULLs), and cleaning text data with string functions. Week 5 — SQL for Analysis: Aggregations, Grouping & Joins

Write analytical SQL queries that summarize, group, and combine data across multiple tables.

Core Focus

Writing analytical SQL queries that summarize, group, and combine data across multiple tables — the queries that actually generate business insights.

Topics & Subtopics

  1. Aggregate Functions — COUNT(*), COUNT(column); SUM(), AVG(), MIN(), MAX(); ROUND(), CEIL(), FLOOR(); DISTINCT within aggregates (COUNT(DISTINCT customer_id)).
  2. GROUP BY & HAVING — Grouping results by one or more columns; how GROUP BY and aggregate functions work together; every non-aggregated column in SELECT must appear in GROUP BY; HAVING filters aggregated results (WHERE filters rows BEFORE aggregation; HAVING filters AFTER).
  3. SQL Joins — INNER JOIN (returns only rows with matching keys in both tables); LEFT JOIN (all rows from left, matched from right); RIGHT JOIN; FULL OUTER JOIN; Self-join (joining a table to itself for hierarchies); joining on multiple conditions.
  4. Subqueries — Subquery in WHERE; subquery in FROM (inline view/derived table); correlated subqueries; when to use a subquery vs a join.
  5. CTEs (Common Table Expressions) — WITH clause for naming temporary result sets; multiple CTEs in one query; why CTEs improve readability over nested subqueries; recursive CTEs intro concept.

Why This Week Matters: GROUP BY with aggregates is the most-used SQL pattern in BI — it answers questions like "Total sales by region" or "Average order value by customer segment." JOINs are what make relational databases powerful — without them, you are limited to one table at a time. Understanding JOINs deeply is what allows you to correctly combine sales data with customer data with product data across a real warehouse.

🛠 Weekly Mini Project

A "SQL Analytics Dashboard Data Preparation" — write 10 SQL queries that would power a real business dashboard. Each query should answer a specific business question using GROUP BY, JOINs, and optionally subqueries or CTEs. Week 6 — Advanced SQL for BI: Window Functions & Query Optimization

Master advanced SQL techniques used specifically in BI work — window functions, period comparisons, and query optimization.

Core Focus

Mastering advanced SQL techniques used specifically in BI work — window functions for running totals, rankings, and period comparisons, plus query optimization for production environments.

Topics & Subtopics

  1. Window Functions — OVER clause (what it does and why it exists); PARTITION BY (dividing data into groups without collapsing rows); ORDER BY inside OVER; ROW_NUMBER() (unique sequential number); RANK() and DENSE_RANK() (ranking with and without gaps); LAG() and LEAD() (previous and next row values); SUM() OVER() (running total); AVG() OVER() (rolling average); NTILE(n) (quartiles, deciles).
  2. Advanced Date Analytics — Year-over-year comparison; Month-to-date (MTD), Quarter-to-date (QTD), Year-to-date (YTD); cohort analysis (grouping customers by first purchase month); days since last activity (recency analysis).
  3. Advanced CTEs and Recursive Queries — Multiple sequential CTEs (each building on the previous); recursive CTEs for organizational hierarchies; generating a date dimension table using a recursive CTE.
  4. Query Optimization — What query execution plans are (EXPLAIN in MySQL); index concepts (what an index is, how it speeds up reads, trade-offs); creating indexes on frequently filtered columns; avoiding functions on indexed columns in WHERE; efficient JOIN order.
  5. SQL Views & Stored Procedures (Intro) — Creating views for frequently used complex queries; using views to simplify reporting; creating simple stored procedures with parameters for automated ETL steps.

Why This Week Matters: Window functions are the single feature that separates a basic SQL user from a BI-capable SQL user. Running totals, rankings, period-over-period comparisons, and cohort analysis all use window functions — and these are among the most common BI requirements from business stakeholders. Query optimization ensures your BI queries run in seconds on large production databases instead of timing out.

🛠 Weekly Mini Project

An "Advanced SQL Analytics Package" — using a retail or financial dataset, write a complete analytical SQL script (minimum 15 queries using window functions, CTEs, and advanced date logic) producing: a ranked leaderboard of customers by lifetime value, a month-over-month and year-over-year revenue comparison table, a running total of cumulative sales for the current year, a cohort retention analysis (customers by first purchase month), and a product performance ranking within each category. Save the script to GitHub.

Week 7 — Power BI Fundamentals: Connecting Data & Building First Reports

Get fully productive in Power BI — connect to multiple data sources, build your first reports, and share them.

Core Focus

Getting fully productive in Power BI — connecting to multiple data sources, understanding the Power BI interface, building first reports, and sharing them through Power BI Service.

Topics & Subtopics

  1. Power BI Ecosystem & Architecture — Power BI Desktop vs Power BI Service vs Power BI Mobile; the three views in Power BI Desktop (Report, Data, Model); how Power BI fits into a BI workflow.
  2. Connecting to Data Sources — Getting data from Excel, CSV files, SQL databases (MySQL/SQL Server), web pages, SharePoint, and OneDrive; blank queries and entering data manually.

Week 8 — DAX: Data Analysis Expressions in Power BI

Learn DAX — Power BI's formula language — to create calculated columns, measures, and time intelligence functions.

Core Focus

Learning DAX — Power BI's formula language — to create calculated columns, measures, and time intelligence functions that go beyond simple aggregations.

Topics & Subtopics

  1. DAX Fundamentals — What DAX is and how it differs from Excel formulas; calculated columns vs measures (when to use each); the evaluation context (row context vs filter context); how measures are dynamically calculated based on filters applied in the report.
  2. Essential DAX Functions — Basic aggregations: SUM(), AVERAGE(), COUNT(), COUNTROWS(), DISTINCTCOUNT(), MAX(), MIN(); conditional: IF(), SWITCH(), IFERROR(); mathematical: ROUND(), ABS(), DIVIDE() (safer than division operator); date: YEAR(), MONTH(), DAY(), TODAY(), DATE(), DATEDIFF().
  3. CALCULATE — The Most Important DAX Function — What CALCULATE does (modifies the filter context); syntax and usage; adding filters; removing filters with ALL(); overriding filters with FILTER(); using CALCULATE with ALLEXCEPT().
  4. Time Intelligence Functions — TOTALYTD(), TOTALQTD(), TOTALMTD() (year/quarter/month-to-date totals); SAMEPERIODLASTYEAR() (same period in the previous year); DATEADD() (offset by days/months/quarters/years); PREVIOUSYEAR(), PREVIOUSMONTH(); year-over-year growth with DIVIDE.
  5. Advanced DAX Patterns — RELATED() and RELATEDTABLE(); RANKX() for ranking items within a context; SUMX(), AVERAGEX(), COUNTX() (iterator functions); VAR and RETURN keywords for variables that improve readability and performance.

Why This Week Matters: DAX is what makes Power BI capable of enterprise-level analytics. Without DAX, you are limited to simple aggregations of your existing data. With DAX, you can calculate profit margins, year-over-year growth, customer lifetime value, market share percentages, and complex business metrics dynamically in response to any filter a user applies. DAX is assessed in the PL-300 Power BI certification exam.

🛠 Weekly Mini Project

Build production-quality Power BI dashboards following design best practices, implement security, and understand the full publishing workflow.

Core Focus

Building production-quality Power BI dashboards following design best practices, implementing Row-Level Security, and understanding the full publishing and governance workflow.

Topics & Subtopics

  1. Dashboard Design Principles — The difference between a Power BI Report and a Dashboard in Power BI Service; F-pattern and Z-pattern reading (how users scan dashboards); visual hierarchy (size, position, color, contrast); choosing the right visual for each data type; color theory for data visualization (sequential, diverging, categorical palettes); accessibility (color blindness, alt text, contrast ratios); whitespace and clutter reduction.
  2. Advanced Power BI Visuals — Decomposition Tree (exploring data hierarchically); Smart Narrative (AI-generated text summaries); Key Influencers (AI visual showing what factors drive a metric); waterfall charts; bullet charts; custom visuals from AppSource; custom tooltip pages.

  5. Publishing, Sharing & Performance — Publishing to Power BI Service; workspaces; sharing reports (by link, workspace access, app); embedding reports in websites and SharePoint; scheduled automatic data refresh; using the Performance Analyzer to identify slow visuals; pre-aggregating data before loading into Power BI.

Why This Week Matters: Building a technically correct report is only half the job. A report that stakeholders cannot navigate, understand, or trust is useless. Dashboard design skills determine whether your work is actually used. Row-Level Security is required in almost every enterprise deployment — you must understand it for any professional BI role. These skills are the difference between a report that gets used and one that gets ignored.

Week 11 — Data Warehousing & ETL Pipeline Design

Understand the architecture, design, and implementation of a data warehouse and ETL processes — the infrastructure that powers enterprise BI.

Core Focus

Understanding the architecture, design, and implementation of a data warehouse and ETL processes — the infrastructure that powers enterprise BI.

Topics & Subtopics

  1. Data Warehouse Architecture — Three-tier architecture (source systems → staging → data warehouse → data marts → BI tools); why a separate data warehouse is needed; Operational Data Store (ODS); data warehouse approaches: Bill Inmon (enterprise-wide, normalized) vs Ralph Kimball (dimensional, data mart-first).
  2. Dimensional Modeling (Kimball Methodology) — Star schema and snowflake schema (deep review); Slowly Changing Dimensions (SCDs): Type 0 (fixed), Type 1 (overwrite), Type 2 (add new row with full history — most common), Type 3 (add new column); surrogate keys vs natural keys; degenerate dimensions; junk dimensions; fact table types (transaction, periodic snapshot, accumulating snapshot).
  3. ETL Process Design — Extract (connecting to source systems; full vs incremental extract); Transform (data type conversion, cleansing, deduplication, business rule application, null handling); Load (truncate and reload vs incremental load — UPSERT; bulk loading); ETL vs ELT (load first, transform in the warehouse — modern cloud approach); data lineage.
  4. ETL Tools Overview — Microsoft SSIS (Microsoft ETL standard); Pentaho Data Integration (free, open-source); Apache NiFi; modern cloud ETL: Fivetran, dbt, AWS Glue (conceptual awareness); Python-based ETL using pandas (covered in Week 13).
  5. Google Looker Studio & Warehouse Implementation — Connecting Looker Studio to Google Sheets, BigQuery, MySQL; creating reports and dashboards; sharing and embedding; designing and implementing a small data warehouse in MySQL with staging tables, SQL-based ETL queries, and dimension/fact tables following the star schema.

Why This Week Matters: Most BI tools sit on top of a data warehouse. Understanding how the warehouse is designed and populated is what separates a BI analyst who uses data from a BI developer or architect who builds the systems that make data available. Data warehousing is covered in BI certifications (PL-300, DP-900) and is a required skill for senior BI roles. SCD Type 2 is one of the most commonly asked interview questions for BI developer roles.

Week 12 — Advanced Analytics: KPIs, Storytelling & BI Strategy

Move from building reports to thinking strategically about BI — defining KPIs, telling compelling data stories, and governing data quality.

Core Focus

Moving from building reports to thinking strategically about BI — how to define meaningful KPIs, tell compelling stories with data, design a BI roadmap, and govern data quality.

Topics & Subtopics

Advanced KPI Framework Design — OKRs (Objectives and Key Results) vs KPIs; Balanced Scorecard (four perspectives: Financial, Customer, Internal Process, Learning & Growth); KPI hierarchies (organizational → departmental → individual); leading vs lagging indicators; diagnostic vs outcome metrics; North Star Metric; building a KPI catalog (definitions, formulas, data sources, owners, update frequency).
  2. Storytelling with Data — Why humans remember stories better than data tables; the STAR framework for data stories (Situation, Task, Action, Result); building a data narrative (context → complication → resolution); the five data story components (data, narrative, visuals, annotation, context); annotation techniques; the "so what?" test (every finding needs a recommendation); presentation best practices (slides vs dashboards; one message per slide).

Why This Week Matters: Technical skills build the BI solution; strategic skills make it valuable. Many technically skilled BI professionals fail to have impact because they build dashboards nobody uses, measure the wrong things, or cannot communicate findings to stakeholders. The ability to define the right KPIs, tell a compelling data story, and navigate organizational dynamics is what drives career advancement in BI from analyst to senior analyst to BI manager.

Using Python as a complement to SQL and Power BI for data analysis, automation, and more advanced analytics that go beyond what traditional BI tools offer.

  2. pandas for Data Analysis — DataFrames and Series; pd.read_csv(), pd.read_excel(), pd.read_sql(); exploring data (.info(), .describe(), .head(), .shape); filtering; GroupBy and aggregation; merging DataFrames (pd.merge() — equivalent of SQL JOIN); pivot tables in pandas; handling missing values; string operations; date handling.
  3. Matplotlib and Seaborn for Visualization — Bar charts, line charts, scatter plots with Matplotlib; statistical visualizations with Seaborn (histogram, box plot, heatmap, pair plot); formatting charts (titles, labels, colors, figure size); saving charts as PNG for reports.
  4. Automated Reporting with Python — Generating Excel reports automatically using openpyxl; sending email reports with smtplib (intro concept); scheduling Python scripts (Windows Task Scheduler or cron); Python as an ETL tool (reading from CSV, transforming, writing to database).
  5. Connecting Python to Databases & Statistical Analysis — SQLAlchemy for database connections; running SQL queries from Python and getting results as DataFrames; writing DataFrames to SQL tables; descriptive statistics; correlation analysis (df.corr()); simple linear regression; trend detection and forecasting basics.

Why This Week Matters: Python has become an essential BI skill as the line between BI and data science blurs. Python allows you to automate repetitive data tasks, perform statistical analysis beyond what SQL or Power BI offer, and connect to data sources that traditional BI tools cannot easily reach. Many modern BI job postings now list Python as a preferred or required skill — particularly for roles that involve data pipeline automation or advanced analytics.

🛠 Weekly Mini Project

A "Python BI Automation Script" — write a Python script that: reads sales data from a CSV or MySQL database, cleans and transforms the data, calculates 5 KPIs (revenue, profit margin, order count, average order value, year-over-year growth), generates a professionally formatted Excel report with formatted cells, charts, and a summary table using openpyxl, and saves it to a file with today's date in the filename.
