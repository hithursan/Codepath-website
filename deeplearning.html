About This Roadmap

This roadmap presents a complete, structured, beginner-to-advanced learning path for Deep Learning and Neural Networks using Python, TensorFlow, and Keras — built and documented as a college project.

Deep learning is the technology behind face recognition, voice assistants, medical image analysis, self-driving cars, and ChatGPT. This roadmap takes you from learning Python for the first time all the way to building, training, and deploying your own neural network that can classify images with real accuracy.

Duration: 15 weeks  |  Pace: Full-time (~35 hrs/week)  |  Stack: Python · NumPy · Pandas · Matplotlib · Scikit-learn · TensorFlow · Keras · OpenCV · Gradio

Hands-On: 14 weekly mini-projects + 1 complete deployed capstone model  |  Outcome: A trained DL model with explainability (Grad-CAM) and a shareable web demo (Gradio + Hugging Face Spaces)

5 Phases Overview

  • Phase 1 (Weeks 1–3): Python, Data Tools & Math Foundations
  • Phase 2 (Weeks 4–5): Machine Learning Fundamentals
  • Phase 3 (Weeks 6–9): Deep Learning Core — Neural Networks & CNNs
  • Phase 4 (Weeks 10–12): Advanced DL — Sequences, NLP & Deployment
  • Phase 5 (Weeks 13–15): Capstone — VisioNet Build, Evaluation & Presentation

By Week 15, you will have built 14 mini-projects and one complete, deployed capstone — "VisioNet," a smart image classifier with transfer learning, Grad-CAM explainability, and a live Gradio web demo on Hugging Face Spaces.

  • Confusing NumPy shape (3, 4) — it's 3 rows, 4 columns.
  • Using Python loops instead of NumPy vectorized ops (10–100x slower).

Week 3 — Mathematics for Deep Learning

Build the mathematical intuition for linear algebra, calculus, and probability that underpins every neural network operation.

Topics & Subtopics

  1. Vectors — Geometric meaning, dot product, magnitude, unit vectors, cosine similarity.
  2. Matrices — Representation, transpose (A.T), matrix-vector multiplication, matrix-matrix multiplication — core operation of every neural network layer.
  3. Why matrix math in DL — A neural network layer is literally matrix multiplication + activation function.
  4. Derivatives — Rate of change, power rule, chain rule — needed for backpropagation.
  5. Partial derivatives — Derivatives w.r.t. one variable holding others constant — how gradients computed in multi-weight networks.
  6. Gradient — Vector of all partial derivatives; direction of steepest ascent.
  7. Gradient descent — Moving opposite to gradient to reduce loss; learning rate; 1D optimization visualization.
  8. Probability — Sample space, probability rules, conditional probability.
  9. Distributions — Gaussian/normal distribution, mean, variance, standard deviation.
  10. Sigmoid & Softmax — Turning raw outputs into probabilities.

Why This Week Matters: Deep learning is applied mathematics. Without understanding gradients, you cannot understand why backpropagation works or how to debug a training problem. You don't need to derive equations from scratch in a viva — but you must understand conceptually what each operation does and why.

Daily Study Plan

Day Focus

Day 2 3Blue1Brown Linear Algebra videos 5–9 (dot product, cross product, matrix multiplication). Implement.

Day 4 Chain rule, partial derivatives. Manually compute gradient of simple 2-variable function.
Day 5 Implement gradient descent from scratch on 1D function. Animate the loss decreasing.
Day 6 Probability basics — Gaussian distribution, mean, variance. Plot bell curve.
Day 7 Sigmoid, softmax. Mini project.

Practice Tasks

  • Multiply two 3×3 matrices by hand; verify with NumPy np.dot().
  • Implement gradient descent from scratch on f(x) = (x-3)² — find minimum, plot path.
  • Compute numerical derivative of sin(x) at x=π/4; compare to analytical cos(π/4).
  • Generate 1000 random Gaussian samples; plot histogram; overlay theoretical curve.

🛠 Mini Project

Common Mistakes

  • Memorizing equations instead of understanding what they represent geometrically.
  • Skipping math entirely assuming "Keras will handle it" — leads to inability to debug.
  • Confusing vector's direction with magnitude.
  • Not implementing gradient descent from scratch — reading is not doing.

How to Avoid Them

  • For every equation, ask "what does this represent geometrically?" before "how do I compute it?"
  • Do gradient descent from scratch even though Keras will do it automatically — understanding is what matters.
  • Implement every math concept in NumPy the same day you watch the explanation.

Understand how classical machine learning works — concepts, workflow, and evaluation that apply to deep learning as well.

Topics & Subtopics

  2. Train/val/test split — Why all 3 needed; data leakage and why it's dangerous.
  3. Scikit-learn workflow — fit(), predict(), score(), train_test_split, Pipeline.
  4. Classification algorithms — Logistic Regression, Decision Tree, Random Forest, K-Nearest Neighbors (conceptual + sklearn practice).

Why This Week Matters: Deep learning is built on ML concepts. Overfitting, evaluation metrics, train/test splits, and cross-validation all appear directly in deep learning. Understanding them with simpler sklearn models first makes them much clearer with neural networks.

🛠 Mini Project

Build a binary classifier on diabetes prediction dataset (Kaggle). Train 3 models (Logistic Regression, Decision Tree, Random Forest). Compare with accuracy, F1, ROC-AUC. Choose best model and explain why for this specific medical dataset. Push to GitHub.

Common Mistakes

  • Using accuracy as only metric, especially on imbalanced datasets.
  • Data leakage: fitting scaler on full dataset before splitting train/test.
  • Overfitting on test set by tuning hyperparameters until test looks good.
  • Not understanding what cross-validation actually does.

How to Avoid Them

  • Always report precision and recall alongside accuracy for medical/imbalanced data.
  • Always fit scalers/encoders on training data only, then transform test data.
  • Test set is used exactly once, at very end — treat as sacred.
  • CV trains k models on different splits and averages performance; final model usually retrained on all training data.

Week 5 — Data Preprocessing, Feature Engineering & Model Optimization

Learn how to prepare raw data for ML and DL models, and understand overfitting/underfitting so you can fix them.

Topics & Subtopics

  1. Scaling — StandardScaler (zero mean, unit variance), MinMaxScaler (0–1); why scaling matters for gradient-based optimization.
  2. Encoding — One-Hot, Label, Ordinal — when each is appropriate.
  3. Missing data — Mean/median/mode imputation, dropping rows/columns, when dropping is dangerous.
  4. Feature engineering — Creating new features (e.g., day of week from date), polynomial features, interaction terms.
  5. Overfitting — Model memorizes training data; high train accuracy, low val accuracy.
  6. Underfitting — Model too simple; low train AND val accuracy.
  7. Regularization — L1 (Lasso — feature selection), L2 (Ridge — smooth weight reduction), dropout (Week 7).
  8. Hyperparameter tuning — GridSearchCV, RandomizedSearchCV.
  9. Pipelines — Chaining preprocessing + model to prevent data leakage.

Why This Week Matters: Real-world datasets are never clean and perfectly formatted. Deep learning models are particularly sensitive to feature scaling — unnormalized inputs cause slow or failed training. Every skill from this week applies directly to preparing image and text data in later weeks.

Daily Study Plan

Day Focus
Day 1 Feature scaling. StandardScaler vs MinMaxScaler. Fit on train, transform test.
Day 2 Categorical encoding — one-hot and label encoding. Handle mixed-type dataset.
Day 3 Missing data strategies. Practice on dataset with intentionally introduced NaN values.
Day 4 Feature engineering — create new features, compare model performance before/after.
Day 5 Learning curves — plot train vs val loss. Diagnose overfitting/underfitting.
Day 6 Regularization — add L2 to logistic regression. GridSearchCV on random forest.
Day 7 Pipelines — chain all preprocessing + model. Review + mini project.

Practice Tasks

  • Take diabetes dataset; manually introduce 10% missing values. Apply 3 imputation strategies; compare model performance.
  • Plot learning curve; deliberately create overfitting with unlimited-depth decision tree. Fix with max_depth=5.
  • Build complete sklearn Pipeline: StandardScaler + OneHotEncoder + RandomForestClassifier handling full preprocessing automatically.

🛠 Mini Project

Build complete "ML Pipeline Notebook" on dataset with mixed types, missing values, categorical features (suggested: House Prices dataset on Kaggle). Notebook must: clean data with documented reasoning, engineer at least 2 new features, compare 3 models (before/after feature engineering), produce final best model chosen based on appropriate metrics. Push to GitHub.

Common Mistakes

  • Fitting scaler on all data (train + test) before splitting — data leakage.
  • Using label encoding for non-ordinal features (implies false ordering: red=1, green=2, blue=3).
  • Treating every imbalanced dataset same way without analyzing class frequencies first.
  • Conflating regularization (L1/L2 penalize weights) with normalization (scaling features).

How to Avoid Them

  • Always split first, then fit preprocessing on train set only.
  • OneHotEncoder for nominal categories; OrdinalEncoder only for ordinal (S/M/L/XL).
  • Check class distribution with value_counts() before choosing metrics and modeling approach.
  • Regularization and normalization both important but solve different problems — apply both.

Week 6 — Neural Networks from Scratch

Build a neural network using only NumPy — no TensorFlow, no Keras — to truly understand forward pass, activation functions, loss, and backpropagation.

Topics & Subtopics

  1. Perceptron — Weighted sum of inputs + bias → activation → output.
  2. Activation functions — Why linear activation insufficient; ReLU (max(0, x)); Sigmoid for output probability; Softmax for multi-class.
  3. Forward pass — Layer by layer: Z = W·X + b, A = activation(Z), repeated per layer.
  4. Loss function — Binary Cross-Entropy (2-class), Categorical Cross-Entropy (multi-class), MSE (regression).
  5. Backpropagation — Chain rule applied backward to compute each weight's gradient w.r.t. loss.
  6. Weight update — W = W - learning_rate × gradient.
  7. Implementation — Random weight initialization (not zero — why); forward pass; loss; backward pass; update loop.

Why This Week Matters: Building from scratch is what separates someone who can use TensorFlow from someone who understands deep learning. Understanding the math of forward and backward pass enables you to debug training problems, explain your model in a viva, and understand why design choices matter.

Daily Study Plan

Day Focus
Day 1 3Blue1Brown neural networks series (all 4 videos). Take notes on backprop math.
Day 2 Implement single neuron: z = w1*x1 + w2*x2 + b; a = sigmoid(z).
Day 3 Implement forward pass for 2-layer network using NumPy matrix operations.
Day 4 Implement loss function. Verify by computing loss for perfect vs random predictions.
Day 5 Derive gradients on paper for 1 layer. Implement backpropagation for output layer.
Day 6 Implement full backpropagation (hidden + output). Implement weight update.
Day 7 Train on XOR problem. Plot loss vs epoch. Verify convergence. Mini project.

Practice Tasks

  • Verify network with zero weight initialization never learns (symmetry problem).
  • Train from-scratch network on XOR; show loss decreasing from ~0.7 to <0.05.
  • Experiment with 3 learning rates (too high, too low, just right); plot all curves on same graph.

🛠 Mini Project

"Neural Network from Scratch" — fully documented Jupyter Notebook implementing 2-layer NN using only NumPy. Must: train on XOR, achieve <0.05 loss, plot loss over training, visualize decision boundary learned, explain every step in markdown. This is a portfolio piece — make it polished. Push to GitHub.

Common Mistakes

  • Initializing all weights to zero (symmetry problem — all neurons learn identically).
  • Getting confused about matrix shapes during backpropagation — shape errors most common bug.
  • Learning rate too high (loss explodes) or too low (loss barely decreases) without investigating.
  • Not testing intermediate outputs — print shapes and values after each step during dev.

How to Avoid Them

  • Always initialize weights randomly: np.random.randn(n_in, n_out) * 0.01 never zeros.
  • Print shape of every matrix at every step during initial implementation.
  • Start with lr=0.1, observe loss curve, adjust systematically.

🛠 Mini Project

Build Keras NN to classify Fashion-MNIST (10 clothing classes). Notebook must show: architecture table (layer/size/activation), training curves (loss + accuracy for train/val), comparison with/without Dropout, final test accuracy. Push to GitHub.

Common Mistakes

  • Using categorical_crossentropy with integer labels instead of sparse_categorical_crossentropy.
  • Forgetting model.evaluate() returns loss first, accuracy second — reading wrong order.
  • Too many Dropout layers or rate too high (e.g., 0.9) causing severe underfitting.
  • Not setting save_best_only=True in ModelCheckpoint — saves every epoch.

How to Avoid Them

  • If labels are integers, use sparse_categorical_crossentropy; if one-hot, use categorical_crossentropy.
  • Always unpack: loss, acc = model.evaluate(X_test, y_test).
  • Typical Dropout: 0.2–0.5; start with 0.3 and observe val accuracy.
  • Always use save_best_only=True, monitor='val_loss' in ModelCheckpoint.

Week 8 — Convolutional Neural Networks (CNNs) for Image Classification

Understand the architecture of CNNs, build one from scratch in Keras, and train it on a real image dataset.

Topics & Subtopics

  1. Why CNNs — 224×224 RGB = 150,528 pixels. Dense to 1000 neurons = 150M params. Conv layer shares weights with small filters.
  2. Convolution — Sliding small filter (kernel) over image, computing dot product; detects local patterns (edges, textures).
  3. Filters — Each filter detects one pattern; 32 filters → 32 feature maps; deeper layers detect complex patterns.
  4. MaxPooling — Reduces spatial size (28×28 → 14×14), reducing computation and providing positional invariance.
  5. Padding — "same" preserves spatial dims; "valid" (default) reduces.
  6. Stride — How far filter moves per step; stride=2 halves spatial dim.
  7. Standard CNN block — Conv2D → BatchNorm → ReLU → MaxPool.
  8. Classification head — Flatten (or GlobalAveragePooling2D) → Dense → Dropout → Softmax.
  9. CIFAR-10 — 60,000 images, 32×32 pixels, 10 classes — standard CNN benchmark.

Why This Week Matters: CNNs are the foundation of almost every computer vision model ever deployed. Transfer learning (Week 9) builds directly on CNN architecture understanding. Image classification is the primary skill used in the VisioNet capstone.

Daily Study Plan

Day Focus
Day 1 Convolution — manually apply 3×3 filter to 5×5 image on paper. Then implement in NumPy.
Day 2 Pooling, padding, stride. Read CS231n CNN notes.
Day 3 Build 3-block CNN in Keras. Load CIFAR-10. Train 20 epochs.
Day 4 Add BatchNormalization to each conv block. Retrain; compare curves.
Day 5 Visualize feature maps from first conv layer on sample image.
Day 6 Analyze predictions — confusion matrix, correct and wrong predictions with images.
Day 7 Mini project.

Practice Tasks

  • Manually compute output of 5×5 image convolved with 3×3 sharpening filter by hand.
  • Train same CNN with and without BatchNormalization on CIFAR-10. Compare val accuracy after 20 epochs.
  • Display 10 incorrect predictions with true and predicted labels. Look for patterns.

🛠 Mini Project

Train CNN image classifier on CIFAR-10. Notebook must include: architecture table, training curves for 30+ epochs, confusion matrix, 5 correct predictions, 5 incorrect predictions with analysis, your test accuracy. Target at least 70% test accuracy. Push to GitHub.

Common Mistakes

  • Using Flatten instead of GlobalAveragePooling2D (too many params, more overfitting).
  • Forgetting to normalize pixel values to [0, 1] (divide by 255.0).
  • Stacking many conv layers without MaxPooling — slow, large spatial dims.
  • Not saving best model checkpoint — losing good model when Colab disconnects.

How to Avoid Them

  • Prefer GlobalAveragePooling2D over Flatten in modern CNNs.
  • Always normalize: X_train = X_train / 255.0.
  • Add MaxPooling after every 1–2 conv layers.
  • Always use ModelCheckpoint with save_best_only=True before long training runs.

Week 9 — Transfer Learning, Data Augmentation & Fine-Tuning

Use pre-trained CNN models (MobileNetV2) to achieve high accuracy on a custom image dataset in a fraction of the time required to train from scratch — the core skill of the VisioNet capstone.

Topics & Subtopics

  1. Why transfer learning works — Early CNN layers learn universal features (edges, textures); only top classification layers need to adapt.
  2. ImageNet — 1.4M images across 1000 classes — most common pre-training dataset for vision.
  3. MobileNetV2 — Lightweight CNN; 3.4M parameters; ideal for beginner capstone with limited Colab GPU.
  4. Feature extraction — Freeze all MobileNetV2 layers, add new classifier head, train only head.
  5. Fine-tuning — After head training, unfreeze top N layers and train with very small learning rate.
  6. Data augmentation — RandomFlip, RandomRotation, RandomZoom, RandomBrightness — multiply training data, improve generalization.
  7. Grad-CAM — Compute gradient of class score w.r.t. final conv activations; use as heatmap overlay.

Why This Week Matters: Transfer learning is the most practically impactful DL technique. With as few as 500 images per class, you can achieve 85–95% accuracy. Without it you'd need millions of images and weeks of training. Grad-CAM adds the "explainability" that makes VisioNet stand out.

Daily Study Plan

Day Focus
Day 1 Read TensorFlow Transfer Learning tutorial. Load MobileNetV2; inspect architecture.
Day 2 Data augmentation pipeline. Visualize augmented images before training.
Day 3 Feature extraction phase — freeze base, add head, train 15 epochs. Plot curves.
Day 4 Fine-tuning phase — unfreeze top 20 layers, train with lr=1e-5 for 10 epochs. Compare.
Day 5 Implement Grad-CAM (tf-keras-vis or from scratch with guided tutorials).
Day 6 Generate Grad-CAM heatmaps for 10 images — 5 correct, 5 incorrect. Analyze.
Day 7 Mini project — capstone model v1.

Practice Tasks

  • Train feature extraction and fine-tuning separately. Record accuracy after each phase to measure improvement.
  • Intentionally choose dataset with too few images (50/class); observe poor results; add data augmentation; observe improvement.
  • Generate Grad-CAM for incorrect prediction; describe in writing what model was looking at instead.

🛠 Mini Project

Train MobileNetV2 via transfer learning on your chosen VisioNet dataset. Notebook must show: data augmentation visualization, feature extraction results, fine-tuning results, comparison table (accuracy before/after FT), 10 Grad-CAM heatmaps with written analysis. Push to GitHub — this is v1 of your capstone model.

Common Mistakes

  • Fine-tuning with too high learning rate, destroying pre-trained weights.
  • Not freezing base model during feature extraction (catastrophic forgetting on small dataset).
  • Forgetting training=False when calling base model during feature extraction (BatchNorm issues).
  • Expecting Grad-CAM to always produce clean intuitive heatmaps — model sometimes focuses on texture/background.

How to Avoid Them

  • Fine-tuning LR should be 10–100× smaller than normal (typical: 1e-5 for FT, 1e-3 for head only).
  • Always base_model.trainable = False during feature extraction; only True during FT.
  • Always call base_model(x, training=False) in model definition during feature extraction.

Week 10 — Recurrent Neural Networks & LSTMs

Understand how neural networks process sequential data — text, audio, and time series — using RNNs and LSTMs.

Topics & Subtopics

  1. Sequential data — Where order matters; sentence different from shuffled words; tomorrow's temp depends on today's.
  2. RNN — Processes one element at a time, maintaining hidden state summarizing what seen so far.
  3. Vanishing gradient — Gradients backpropagated through many time steps shrink exponentially; early steps impossible to learn.
  4. LSTM gates — Forget (discard from memory), Input (store new info), Output (decide what to output); maintain long-term cell state alleviating vanishing gradient.
  5. GRU — Simpler LSTM with only 2 gates; often performs similarly with fewer parameters.
  6. Keras LSTM — LSTM(units, return_sequences=True/False), Bidirectional(LSTM(...)) for reading both directions.
  7. Use cases — Stock price prediction, music generation, text generation, sentiment analysis.

Why This Week Matters: LSTMs were the dominant architecture for text/sequence tasks for years before Transformers. Understanding them is essential for the NLP week that follows, and the skills transfer directly to sentiment analysis.

Daily Study Plan

Day Focus

Day 2 Vanishing gradient problem — draw gradient flow through 10 time steps.
Day 3 Read Christopher Olah's LSTM post. Draw all 3 gates yourself. No coding yet.
Day 4 Implement LSTM in Keras for simple time series (predict next value in sine wave).
Day 5 LSTM text predictor — trained on small text corpus (short story or poem).
Day 6 Bidirectional LSTM. GRU comparison. Plot predictions.
Day 7 Mini project.

Practice Tasks

  • Generate sine wave + noise; train LSTM to predict next value. Plot predictions vs ground truth.
  • Train LSTM on first 100 lines of book; generate 50 characters of new text from it.
  • Compare LSTM vs GRU on same time series — which is faster? More accurate?

🛠 Mini Project

"LSTM Next-Word Predictor" — train LSTM on text corpus (5,000+ words; book, Wikipedia, etc.). Given 3-word seed, model generates next 20 words. Show 5 examples with different seeds. Document training loss curve. Push to GitHub.

Common Mistakes

  • Setting return_sequences=False when stacking multiple LSTM layers.
  • Not reshaping time series into (samples, time_steps, features) before LSTM.
  • Expecting grammatically perfect text from char-level LSTM on small corpus.

How to Avoid Them

  • Stacking LSTMs: intermediate need return_sequences=True; only final uses False.
  • LSTM input shape: always (batch_size, timesteps, features).
  • Char-level LSTM on 5,000 words produces plausible but nonsensical text — expected.

Week 11 — Natural Language Processing with Deep Learning

Apply deep learning to text — preprocessing text data, learning word embeddings, and building a sentiment analysis classifier.

Topics & Subtopics

  1. Text preprocessing — Lowercasing, removing punctuation/stopwords, stemming/lemmatization.
  2. Tokenization — Converting words to integers; tf.keras.preprocessing.text.Tokenizer or TextVectorization.
  3. Padding — Sequences have different lengths; pad_sequences pads shorter to fixed length.
  4. One-hot vs embeddings — One-hot: V-dim sparse vector per word; embeddings: dense low-dim vector capturing semantic relationships.
  5. Word embeddings — "king" - "man" + "woman" ≈ "queen" — vectors capture meaning.
  6. Keras Embedding layer — Learns embeddings from scratch during training; size = (vocab_size, embedding_dim).
  7. GloVe — Pre-trained word vectors on billions of words; load as initial weights for Embedding layer.
  8. Sentiment architecture — Embedding → LSTM → Dense (sigmoid for binary) — classify reviews positive/negative.

Why This Week Matters: NLP is one of the largest application areas of deep learning. Text preprocessing and embedding skills are the foundation for understanding Transformers, BERT, and LLMs — which dominate modern NLP. Sentiment analysis is a well-scoped, demo-able project that shows real utility.

Common Mistakes

  • Using Python's str.split() for tokenization instead of proper tokenizer.
  • Forgetting to pad sequences — LSTM crashes or gives wrong shapes.
  • Building GloVe embeddings matrix incorrectly (index off-by-one — most tokenizers start at 1, not 0).
  • Overfitting on IMDB with too complex model — Embedding + LSTM with dropout should reach 85–88%.

How to Avoid Them

  • Use TextVectorization or Tokenizer from Keras, not manual string splitting.
  • Always check all sequences in batch are same length after padding.
  • Build GloVe matrix carefully — print first 5 entries and verify words match expected vectors.

🧠 VisioNet — Smart Image Classifier with Explainability

VisioNet is a complete, deployed deep learning image classifier built using transfer learning, with Grad-CAM explainability and a live Gradio web demo on Hugging Face Spaces. Built incrementally from Week 9 onwards and finalized in Weeks 13–15.

Core Features

  1. Custom Image Classifier — Trained on a domain-specific dataset of your choice (e.g., medical images, plant species, food categories, vehicle types).
  2. Transfer Learning Architecture — MobileNetV2 pre-trained on ImageNet, with custom classification head; feature extraction phase followed by fine-tuning phase.
  3. Data Augmentation Pipeline — RandomFlip, RandomRotation, RandomZoom, RandomBrightness for improved generalization.
  4. Explainability (Grad-CAM) — Heatmap overlay on input image showing which regions the model focused on for its prediction.
  5. Confidence Scores — Returns probability for top predicted class.
  6. Live Gradio Web Demo — Deployed on Hugging Face Spaces; anyone with the URL can upload an image and see both the prediction and the Grad-CAM heatmap.
  7. Complete Evaluation — Confusion matrix, per-class precision/recall/F1, error analysis on confidently wrong predictions.
  8. Comprehensive Documentation — README with problem statement, dataset description, architecture, results table, Grad-CAM examples, live demo link, setup instructions.

Training Strategy

Phase 1 (Feature Extraction): Freeze entire MobileNetV2 base; train only custom classification head for 15–20 epochs with lr=1e-3.

Phase 2 (Fine-Tuning): Unfreeze top 20–30 layers of MobileNetV2; train with lr=1e-5 for 10–15 epochs. Compare accuracy improvement.

  • Training notebook (transfer learning, feature extraction + fine-tuning, training curves)
  • Evaluation notebook (confusion matrix, classification report, error analysis)

  • System design document (architecture diagram, data flow, technology choices)
