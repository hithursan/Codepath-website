About This Curriculum

This is a structured, progressive Python curriculum designed to take a learner from absolute beginner level to professional mastery. Each week builds directly on the previous one and includes a core focus, topic cards, and a hands-on practical task.

The curriculum is organized into four phases: Phase 1 covers Python fundamentals and data types, Phase 2 focuses on control flow, functions, modules, and data structures, Phase 3 develops intermediate and advanced core skills such as OOP, file handling, and error handling, and Phase 4 introduces master-level concepts including decorators, generators, concurrency, and real-world portfolio projects.

By the end of this roadmap, you should be able to write clean, reusable, production-grade Python code, build practical applications, and organize projects professionally for portfolio and career growth.

Level: Beginner → Master  |  Duration: 20 weeks  |  Study Mode: Full-time or structured self-paced  |  Main Focus: Python Programming

Week 2 — Variables, Data Types & Type System

Understand how Python stores data, what kinds of data exist, and how Python’s type system works.

Core Focus

Understand how Python stores data, what kinds of data exist, and how Python’s type system works.

Topics & Topic Cards

Topic Card Description
Variables & Assignment A variable is a named label that points to a value stored in memory and is created the moment you write something like x = 10. Python is dynamically typed, so you do not declare the type explicitly.
Integers (int) Integers are whole numbers with no decimal point, and Python integers grow as large as memory allows. You can write values like 1_000_000 with underscores for readability.
Floats (float) Floats represent decimal numbers using binary floating-point format. This is why 0.1 + 0.2 does not equal exactly 0.3, and why the decimal module is important for money calculations.
Booleans (bool) Booleans have only two values: True and False. Almost every conditional and loop in Python ultimately depends on Boolean evaluation.
type() and isinstance() type(x) returns the exact type of a variable, while isinstance(x, int) checks if a value matches a type or subclass. In real-world code, isinstance() is usually preferred.
Variable Naming Rules Variable names can include letters, digits, and underscores, but cannot start with a digit or contain spaces. Python convention uses snake_case for variables and functions.

Why This Week Matters: Variables and types are the foundation of all Python programs. If you understand how Python stores and identifies data, you can reason about almost every later concept more clearly.

Master Python strings — the most frequently used data type in real-world programming.

Core Focus

Master Python strings — the most frequently used data type in real-world programming.

Topics & Topic Cards

Topic Card Description
String Creation & Quotes Strings can be created with single quotes, double quotes, or triple quotes. Triple-quoted strings span multiple lines and are also used for docstrings.
Indexing & Slicing Every character has an index, starting from 0 on the left and -1 on the right. Slicing with s[start:stop:step] extracts substrings, and s[::-1] reverses a string.
String Immutability Strings cannot be changed after creation. Methods like .replace() and .upper() always return new strings rather than modifying the original.
Essential String Methods Methods such as .upper(), .lower(), .strip(), .replace(), .split(), and .join() cover most everyday string processing tasks.
f-Strings f-strings embed expressions directly inside strings using curly braces, like f"Hello, {name}!". They are readable, fast, and now standard in modern Python code.
String Searching & Membership The in keyword checks if a substring exists, while methods like .find(), .count(), and .startswith() provide more detailed text search capabilities.

Why This Week Matters: Text processing is everywhere in Python — user input, file content, logs, web data, and reports. Strong string skills make everything else easier.

Topic Card Description
if, elif, else Conditional statements let your program follow different execution paths depending on whether a condition is True or False. Python uses indentation instead of curly braces.

First-Class Functions Functions in Python can be stored in variables, passed as arguments, and returned from other functions. This makes decorators possible.

Week 17 — Concurrency: Threading, Multiprocessing & Asyncio

Run multiple tasks simultaneously using Python’s three concurrency models — and know which to use for which problem.

Core Focus

Run multiple tasks simultaneously using Python’s three concurrency models — and know which to use for which problem.

Topics & Topic Cards

Topic Card Description
Concurrency vs Parallelism Concurrency means tasks overlap in time, while parallelism means tasks run at the exact same moment on different cores. Choosing the right model depends on the nature of the work.
threading Module Threads are good for many I/O-bound tasks, but because they share memory, locks are required when shared state is modified to prevent race conditions.
concurrent.futures ThreadPoolExecutor and ProcessPoolExecutor offer a simpler, higher-level interface for concurrency than manually managing threads or processes.
multiprocessing Module Multiprocessing uses separate processes and bypasses the GIL, making it the correct choice for CPU-bound work.
asyncio — Async/Await asyncio uses an event loop and coroutines to handle large numbers of I/O-bound tasks efficiently with cooperative multitasking.
Choosing the Right Model Use asyncio or threading for I/O-bound work and multiprocessing for CPU-heavy work. Picking the wrong model can reduce performance instead of improving it.

Why This Week Matters: Modern software often needs to do many things at once. Understanding concurrency helps you build faster, more scalable, and more realistic Python applications.
