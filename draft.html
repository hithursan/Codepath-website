About This Curriculum

This is a 16-week, phase-based Dart curriculum designed to take a complete beginner to a professional-level Dart developer. The roadmap moves from syntax, variables, data types, and null safety through control flow, functions, loops, and collections, into OOP, error handling, async programming, and libraries, all the way to generics, generators, streams, isolates, and three complete capstone projects.

The curriculum is organized into four phases: Phase 1 covers absolute basics including syntax and the type system, Phase 2 covers control flow, functions, and advanced collections, Phase 3 covers intermediate and advanced core topics like OOP, error handling, and packages, and Phase 4 covers master-level generics, generators, concurrency, and production projects.

By Week 16, you will have built three complete, portfolio-ready applications and be confident building production-grade Dart applications — including Flutter apps and server-side services.

Level: Beginner → Master  |  Duration: 16 weeks  |  Stack: Dart 3+, DartPad, pub.dev  |  Prerequisites: Basic programming logic

Week 1 — Setting Up & First Steps

Environment setup, basic syntax, and program structure.

Core Focus

Setting up your environment, understanding basic syntax, and writing your first Dart programs.

Topics & Subtopics

  1. Installing Dart & DartPad — Dart can be installed via the Dart SDK or used instantly online through DartPad. This removes setup friction so you can start writing code immediately without any local installation.
  2. Variables & var — Variables store data that a program can reference and manipulate later. The var keyword lets Dart infer the type automatically from the assigned value, keeping code concise without sacrificing type safety.
  3. Comments & Print Statements — Comments document code without affecting execution, using // for single lines or /* */ for blocks. The print() function outputs values to the console for debugging and learning.

Why This Week Matters: Every Dart program — from a simple script to a full Flutter application — starts with a main() function and uses print() for output. Understanding var and type inference from Day 1 establishes the mental model that Dart is both type-safe and concise — you don't have to choose one or the other.

Code Example

// The entry point of every Dart application
void main() {
  print('Hello, Dart World!'); // Prints text to console
}

void main() {
  var name = 'Alice'; // Inferred as String
  var age = 25;       // Inferred as int
  print('$name is $age years old');
}

void main() {
  // This is a single-line comment
  /* This is a
     multi-line comment */
  print('Comments help readability'); // inline comment
}

Week 2 — Data Types & Type System

Understanding Dart's static type system and primitive types.

Core Focus

Understanding Dart's static type system and all primitive data types.

Topics & Subtopics

  1. int and double — int represents whole numbers while double represents decimal/floating-point numbers. Dart enforces these types strictly, preventing accidental mixing without explicit conversion.
  2. String & Boolean — String holds text data and supports interpolation using $, while bool holds only true or false. Both are foundational for logic and display in nearly every program you will ever write.
  3. Type Safety with final and const — final creates a variable that can be set only once at runtime, while const is a compile-time constant. Both improve safety by preventing accidental reassignment of values that should never change.

Why This Week Matters: Dart's sound type system is what makes Flutter apps so reliable — type errors are caught at compile time, not at runtime in front of users. Understanding the distinction between var (inferred, mutable), final (set once), and const (compile-time fixed) is essential before moving to any real Dart programming.

Code Example

void main() {
  int wholeNumber = 42;
  double decimalNumber = 3.14;
  print(wholeNumber + decimalNumber); // Auto-promotes to double
}

void main() {
  String city = 'Colombo';
  bool isCapital = false;
  print('$city is capital: $isCapital');
}

void main() {
  final currentYear = 2026; // set once, runtime
  const pi = 3.14159;       // fixed at compile time
  print('$currentYear, $pi');
}

Week 3 — Operators & Expressions

Performing operations and building expressions in Dart.

Core Focus

Performing arithmetic, comparison, and logical operations, and building safe expressions using null-aware operators.

Topics & Subtopics

  1. Arithmetic Operators — Arithmetic operators (+, -, *, /, %, ~/) perform mathematical calculations on numeric values. The ~/ operator performs integer (floor) division — a unique and useful Dart feature.
  2. Comparison & Logical Operators — Comparison operators (==, !=, >, <) evaluate relationships between values and return booleans. Logical operators (&&, ||, !) combine multiple boolean expressions for complex conditions.
  3. Assignment & Null-aware Operators — Compound assignment operators (+=, -=) shorten value updates, while null-aware operators (??, ??=) safely handle null values. These are essential for Dart's strong null-safety model.

Why This Week Matters: The null-aware operators (?? and ??=) are among Dart's most used features — they appear constantly in Flutter code for providing default values and safely accessing potentially-null properties. Mastering them now prevents dozens of null-safety errors in future weeks.

Code Example

void main() {
  int a = 17, b = 5;
  print(a / b);  // 3.4 (double division)
  print(a ~/ b); // 3 (integer division)
  print(a % b);  // 2 (remainder)
}

void main() {
  int age = 20;
  bool canVote = age >= 18 && age < 120;
  print('Can vote: $canVote');
}

void main() {
  String? nickname; // nullable String
  nickname ??= 'Guest'; // assign only if null
  print(nickname);
}

Week 4 — Null Safety & Collections Intro

Dart's sound null safety and basic collection types.

Core Focus

Mastering Dart's sound null safety system and working with the three core collection types.

Topics & Subtopics

  1. Nullable Types (?) — By default, variables in Dart cannot be null unless explicitly marked with ?. This compile-time check prevents the common "null reference" runtime crashes seen in other languages — Dart's most celebrated safety feature.
  2. Lists Basics — A List is an ordered collection of items, similar to arrays in other languages. Dart lists are generic, meaning you specify the type of elements they hold for full type safety.
  3. Maps Basics — A Map stores data as key-value pairs, allowing fast lookups by a unique key. Maps are ideal for representing structured data like JSON objects or configuration settings.

Why This Week Matters: Dart's sound null safety is what makes it different from most languages — the compiler guarantees that a non-nullable variable can never be null, eliminating an entire class of runtime crashes. Lists and Maps are used in virtually every Dart and Flutter program — learning them here unlocks all future collection work.

Code Example

void main() {
  String? middleName; // can be null
  middleName = null;  // valid
  print(middleName ?? 'No middle name');
}

void main() {
  List<String> fruits = ['Apple', 'Banana', 'Mango'];
  fruits.add('Orange');
  print(fruits); // [Apple, Banana, Mango, Orange]
}

void main() {
  Map<String, int> ages = {'Alice': 25, 'Bob': 30};
  ages['Charlie'] = 28;
  print(ages);
}

Week 5 — Conditional Logic

Decision-making constructs in Dart.

Core Focus

Directing program execution using Dart's expressive conditional and decision-making constructs.

Topics & Subtopics

  1. if / else if / else — Conditional statements allow a program to execute different code blocks based on whether conditions are true or false. Chaining else if lets you test multiple conditions sequentially in a clean, readable structure.
  2. switch Statements — The switch statement compares a single value against multiple possible cases, offering a cleaner alternative to long if-else chains. Dart's enhanced switch also supports pattern matching in newer versions.
  3. Ternary & Conditional Expressions — The ternary operator (condition ? a : b) provides a compact way to write simple if-else logic in a single line. It is especially useful for assigning values based on conditions without verbose blocks.

Why This Week Matters: Dart 3 introduced exhaustive pattern matching in switch expressions — one of the most powerful features added to any language in recent years. Starting with the basics here prepares you for the advanced switch patterns used throughout modern Dart and Flutter code.

Code Example

void main() {
  int score = 75;
  if (score >= 90) {
    print('A grade');
  } else if (score >= 70) {
    print('B grade');
  } else {
    print('C grade');
  }
}

void main() {
  String day = 'Mon';
  switch (day) {
    case 'Mon':
      print('Start of week');
      break;
    default:
      print('Another day');
  }
}

void main() {
  int temp = 30;
  String status = temp > 25 ? 'Hot' : 'Cool';
  print(status);
}

Week 6 — Loops & Iteration

Repeating actions efficiently in Dart.

Core Focus

Repeating actions efficiently using Dart's full range of loop constructs and iteration patterns.

Topics & Subtopics

  1. for Loop — The for loop repeats a block of code a specific number of times, controlled by an initializer, condition, and increment. It is most useful when the number of iterations is known in advance.
  2. while & do-while Loops — A while loop repeats as long as a condition remains true, checking before each iteration. A do-while loop guarantees at least one execution since it checks the condition afterward.
  3. for-in & forEach — The for-in loop iterates directly over collection elements without needing an index. The forEach method achieves the same result using a functional callback style — cleaner and more expressive.

Why This Week Matters: The for-in loop and forEach are the most common iteration patterns in real Dart and Flutter code. Understanding when to use each — and the difference between a statement-based loop and a functional callback — is a mark of idiomatic Dart style.

Code Example

void main() {
  for (int i = 1; i <= 5; i++) {
    print('Count: $i');
  }
}

void main() {
  int n = 0;
  do {
    print('n is $n');
    n++;
  } while (n < 3);
}

void main() {
  var colors = ['Red', 'Green', 'Blue'];
  for (var color in colors) {
    print(color);
  }
}

Week 7 — Functions

Reusable, modular blocks of logic in Dart.

Core Focus

Writing clean, reusable, and modular logic using Dart's flexible function system.

Topics & Subtopics

  1. Function Declaration & Parameters — Functions encapsulate reusable logic and can accept input through parameters. Dart supports positional, named, and optional parameters for flexible function signatures that read like natural language.
  2. Arrow Functions — Arrow functions (=>) provide concise syntax for functions that contain a single expression. They are widely used for short utility functions and callbacks throughout Dart and Flutter code.
  3. Anonymous Functions & Closures — Anonymous functions have no name and are often passed directly as arguments to other functions. Closures capture variables from their surrounding scope, allowing state to persist across calls.

Why This Week Matters: Dart's named parameters ({}) with default values are used everywhere in Flutter — almost every widget constructor uses named parameters. Understanding closures is essential for working with callbacks, event handlers, and the functional collection methods you will use in Week 8 and beyond.

Code Example

// Named optional parameter with default value
int add(int a, int b, {int bonus = 0}) {
  return a + b + bonus;
}
void main() => print(add(3, 4, bonus: 2));

// Arrow function
int square(int x) => x * x;
void main() {
  print(square(6)); // 36
}

// Closure
Function makeMultiplier(int factor) {
  return (int x) => x * factor; // closure captures 'factor'
}
void main() => print(makeMultiplier(3)(5)); // 15

Week 8 — Collections Deep Dive

Advanced manipulation of Lists, Sets, and Maps.

Core Focus

Advanced manipulation of Lists, Sets, and Maps using functional methods and modern collection syntax.

Topics & Subtopics

  1. Sets — A Set is an unordered collection that automatically eliminates duplicate values. It is ideal for tasks requiring uniqueness checks, like tracking distinct tags, visited URLs, or unique IDs.
  2. Collection Methods: map, where, reduce — map transforms each element, where filters elements by condition, and reduce combines elements into a single value. These functional methods replace many manual loop patterns with cleaner, chainable expressions.
  3. Spread & Collection-if/for — The spread operator (...) inserts all elements of one collection into another. Collection-if and collection-for allow conditional or generated elements directly inside literals — a powerful feature unique to Dart.

Why This Week Matters: Collection-if and collection-for are used constantly in Flutter's widget trees — they allow you to conditionally include or generate widgets directly in a list literal. The where/map/reduce chain is the functional programming pattern that replaces most verbose loop logic in professional Dart code.

Code Example

void main() {
  Set<int> uniqueNumbers = {1, 2, 2, 3, 3, 3};
  uniqueNumbers.add(4);
  print(uniqueNumbers); // {1, 2, 3, 4}
}

void main() {
  var nums = [1, 2, 3, 4, 5];
  var evens = nums.where((n) => n % 2 == 0).toList();
  print(evens); // [2, 4]
}

void main() {
  var base = [1, 2, 3];
  var extended = [...base, 4, if (base.isNotEmpty) 5];
  print(extended); // [1, 2, 3, 4, 5]
}

Week 9 — Object-Oriented Programming I

Classes, objects, and constructors in Dart.

Core Focus

Understanding classes, objects, constructors, and property encapsulation — the foundation of Dart's OOP model.

Topics & Subtopics

  1. Classes & Objects — A class is a blueprint defining properties and behaviors, while an object is a specific instance created from that blueprint. This is the foundation of organizing code around real-world entities in a structured, maintainable way.
  2. Named & Factory Constructors — Named constructors allow a class to offer multiple ways to create instances with different initialization logic. Factory constructors can return existing instances or subtypes instead of always creating new objects.
  3. Getters & Setters — Getters and setters control how a property's value is read or written, enabling validation or computed values. They allow encapsulation while keeping clean, field-like access syntax from the outside.

Why This Week Matters: In Flutter, every UI component is a class. Understanding Dart classes, constructors, and getters is the direct prerequisite for building Flutter widgets. Factory constructors are used extensively in Flutter — TextStyle.lerp(), Color.fromARGB(), and many more all follow this pattern.

Code Example

class Car {
  String brand;
  Car(this.brand); // constructor shorthand
  void honk() => print('$brand says beep!');
}
void main() => Car('Toyota').honk();

class Point {
  double x, y;
  Point(this.x, this.y);
  Point.origin() : x = 0, y = 0; // named constructor
}
void main() => print(Point.origin().x);

class Circle {
  double radius;
  Circle(this.radius);
  double get area => 3.14 * radius * radius; // computed getter
}
void main() => print(Circle(2).area);

Week 10 — Object-Oriented Programming II

Inheritance, polymorphism, abstraction, and mixins.

Core Focus

Mastering inheritance, polymorphism, abstract classes, and Dart's powerful mixin system.

Topics & Subtopics

  1. Inheritance with extends — Inheritance allows a class to acquire the properties and methods of another class, promoting code reuse. The super keyword is used to access the parent class's constructor or members from within a subclass.
  2. Abstract Classes & Interfaces — Abstract classes define a contract that subclasses must implement, and cannot be instantiated directly. Dart uses implements for interface-like behavior, enforcing structural contracts on classes without requiring a shared parent.
  3. Mixins — Mixins allow a class to reuse code from multiple sources without traditional multiple inheritance. They are added using the with keyword and are useful for sharing behavior across unrelated classes cleanly and safely.

Why This Week Matters: Mixins are one of Dart's most distinctive and powerful features — Flutter uses them extensively (e.g., TickerProviderStateMixin, AutomaticKeepAliveClientMixin). Understanding when to use inheritance versus a mixin versus an interface is a mark of a professional Dart developer.

Code Example

class Animal {
  void speak() => print('Some sound');
}
class Dog extends Animal {
  @override
  void speak() => print('Woof!'); // method overriding
}
void main() => Dog().speak();

abstract class Shape {
  double area(); // must be implemented by subclasses
}
class Square extends Shape {
  double side;
  Square(this.side);
  @override
  double area() => side * side;
}

mixin Logger {
  void log(String msg) => print('LOG: $msg');
}
class Service with Logger {}
void main() => Service().log('Started');

Week 11 — Error Handling & Async Basics

Managing failures and asynchronous code in Dart.

Core Focus

Managing failures gracefully and writing non-blocking asynchronous code using Futures and async/await.

Topics & Subtopics

  1. try / catch / finally — try-catch blocks let a program gracefully handle runtime errors instead of crashing. The finally block always executes regardless of whether an exception occurred, making it ideal for cleanup operations like closing files or connections.
  2. Custom Exceptions — Custom exception classes let you define meaningful, domain-specific error types beyond Dart's built-ins. They typically implement the Exception interface for clarity and provide structured information about what went wrong.
  3. Futures & async/await — A Future represents a value that will be available at some point in the future, typically from an asynchronous operation like a network call. The async/await keywords let you write asynchronous code that reads like synchronous code.

Why This Week Matters: Every Flutter app uses async/await — for API calls, database reads, file operations, and more. Understanding Future and custom exceptions is the direct prerequisite for all real-world Dart development. Without this week, the next four weeks of content will not make sense.

Code Example

void main() {
  try {
    int result = 10 ~/ 0; // throws exception
  } catch (e) {
    print('Error caught: $e');
  } finally {
    print('Cleanup done');
  }
}

class InsufficientFundsException implements Exception {
  String message = 'Not enough balance';
}
void main() {
  try {
    throw InsufficientFundsException();
  } catch (e) {
    print(e);
  }
}

Future<String> fetchData() async {
  await Future.delayed(Duration(seconds: 1));
  return 'Data loaded';
}
void main() async {
  print(await fetchData());
}

Week 12 — Libraries, Packages & File I/O

Modular code organization and external interaction in Dart.

Core Focus

Organizing code with imports, leveraging the pub.dev package ecosystem, and reading and writing files with dart:io.

Topics & Subtopics

  1. Imports & Library Structure — The import keyword brings in code from Dart's core libraries, packages, or your own files. Prefixing imports (as) helps avoid naming conflicts between libraries when two libraries export the same name.
  2. Using pub.dev Packages — pub.dev is Dart's official package repository, hosting reusable libraries for common tasks like HTTP requests or JSON parsing. Packages are declared in pubspec.yaml and fetched via dart pub get.
  3. File Reading & Writing (dart:io) — The dart:io library provides classes like File for reading from and writing to the filesystem. This is essential for command-line tools and server-side applications that need to persist data between runs.

Why This Week Matters: The pub.dev ecosystem is what makes Dart practical for real projects — the http, dio, and json_serializable packages are used in virtually every Flutter app. Understanding pubspec.yaml and package versioning is a required skill for any professional Dart or Flutter developer.

Week 13 — Generics & Advanced Functions

Writing flexible, type-safe, reusable code in Dart.

Core Focus

Writing flexible, type-safe, and reusable code using generics, type constraints, and higher-order functions.

Topics & Subtopics

  1. Generics (<T>) — Generics allow classes and functions to operate on multiple types while preserving type safety. This avoids code duplication that would otherwise require writing separate versions for each type.
  2. Generic Constraints (extends) — Generic type parameters can be constrained using extends to restrict which types are valid. This ensures the generic type supports the operations your code relies on inside the generic body.
  3. Higher-Order Functions — Higher-order functions accept other functions as parameters or return them as results. This pattern enables powerful abstractions like custom sorting, filtering, event callbacks, and middleware pipelines.

Why This Week Matters: Generics are everywhere in Dart's standard library — List<T>, Map<K, V>, Future<T>, Stream<T>. Understanding how to write your own generic classes and functions makes you capable of building reusable libraries and frameworks, not just consuming them.

Code Example

class Box<T> {
  T value;
  Box(this.value);
}
void main() {
  Box<int> intBox = Box<int>(42);
  print(intBox.value);
}

T findMax<T extends num>(T a, T b) => a > b ? a : b;
void main() => print(findMax<double>(3.5, 7.2));

void repeat(int times, void Function(int) action) {
  for (int i = 0; i < times; i++) action(i);
}
void main() => repeat(3, (i) => print('Run $i'));

Week 14 — Generators & Iterables

Lazy evaluation and custom sequences in Dart.

Core Focus

Building lazy, memory-efficient sequences using synchronous and asynchronous generators and custom iterables.

Topics & Subtopics

  1. Synchronous Generators (sync*) — A sync* function returns an Iterable lazily using the yield keyword instead of building a full list upfront. This is memory-efficient for large or infinite sequences.
  2. Asynchronous Generators (async*) — An async* function returns a Stream lazily, emitting values over time using yield. This is ideal for representing ongoing data sources like sensor readings, live feeds, or paginated API responses.
  3. Custom Iterables & Iterators — Implementing the Iterable and Iterator interfaces lets your custom classes work seamlessly with for-in loops and collection methods — useful for building domain-specific collection types.

Why This Week Matters: async* generators that return Stream are the exact foundation of Flutter's reactive programming model — used in BLoC pattern, StreamBuilder widgets, and real-time data flows. Understanding generators here makes the Stream deep-dive in Week 15 immediately intuitive.

Code Example

Iterable<int> countTo(int n) sync* {
  for (int i = 1; i <= n; i++) yield i;
}
void main() => print(countTo(5).toList());

Stream<int> tick(int n) async* {
  for (int i = 1; i <= n; i++) {
    await Future.delayed(Duration(milliseconds: 200));
    yield i;
  }
}

class Evens extends Iterable<int> {
  final int max;
  Evens(this.max);
  @override
  Iterator<int> get iterator =>
      [for (var i = 0; i <= max; i += 2) i].iterator;
}

Week 15 — Concurrency: Streams & Isolates

Handling parallelism and reactive data flows in Dart.

Core Focus

Handling true parallelism with Isolates and building reactive data pipelines using Streams and StreamControllers.

Topics & Subtopics

  1. Streams Deep Dive — A Stream is a sequence of asynchronous events that can be listened to over time, similar to a Future but for multiple values. Streams are central to handling things like user input, WebSockets, or file watchers.
  2. StreamControllers & Broadcast Streams — A StreamController lets you manually create and manage a stream, pushing events into it programmatically. Broadcast streams allow multiple listeners to subscribe to the same event source simultaneously.
  3. Isolates for True Parallelism — Dart uses isolates instead of threads to achieve true parallel execution, since each isolate has its own memory and event loop. Isolate.spawn() runs separate code paths concurrently without shared-memory race conditions.

Why This Week Matters: Flutter uses Isolates for heavy computation — image processing, JSON parsing of large payloads, and machine learning inference all run in separate Isolates to keep the UI thread smooth. Broadcast streams power real-time features in Flutter apps including chat, live scores, and notifications.

Code Example

void main() {
  Stream<int> numberStream = Stream.fromIterable([1, 2, 3]);
  numberStream.listen((n) => print('Received: $n'));
}

import 'dart:async';
void main() {
  final controller = StreamController<String>.broadcast();
  controller.stream.listen((msg) => print('Got: $msg'));
  controller.add('Hello listeners!');
}

import 'dart:isolate';
void heavyTask(SendPort sendPort) {
  sendPort.send('Computation done');
}
void main() async {
  final receivePort = ReceivePort();
  await Isolate.spawn(heavyTask, receivePort.sendPort);
  receivePort.listen((msg) => print(msg));
}
