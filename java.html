About This Curriculum

This is a 24-week, phase-based Java curriculum designed to take a complete beginner to a professional-level Java developer. The roadmap moves from core syntax, memory, and types through control flow, OOP, file handling, error handling, and functional-style programming, all the way to concurrency, JVM internals, design patterns, and three complete capstone projects.

The curriculum is organized into four phases: Phase 1 covers absolute basics and the Java type system, Phase 2 covers control flow, methods, and core data structures, Phase 3 covers object-oriented design, file I/O, generics, and lambdas, and Phase 4 takes you into concurrency, JVM internals, design patterns, and production-level projects.

By Week 24, you will have written complete, working Java applications, understood how the JVM runs your code, and built three portfolio-ready projects from beginner to master level.

Level: Beginner → Master  |  Duration: 24 weeks  |  Study Time: ~5–8 hours/week  |  Main Focus: Java Programming

Phase 1 — Absolute Basics

Weeks 1–5: Build a rock-solid foundation in Java syntax, memory, and types.

Week 1 — Java Setup & Your First Program

Environment setup, the JVM, and writing and running your first Java program.

Core Focus

Environment setup, understanding the JVM, and writing and running your first Java program.

Topics & Subtopics

  1. JDK, JRE, and JVM — The JDK is the developer toolkit, the JRE runs Java apps, and the JVM executes compiled bytecode. Understanding this trio explains why Java is "write once, run anywhere."
  2. Compiling & Running Java — Java source files (.java) are compiled to bytecode (.class) by javac, then executed by java. This two-step process is core to how Java differs from purely interpreted languages.
  3. The main Method — Every standalone Java app starts from public static void main(String[] args). Its exact signature is mandatory and tells the JVM where to begin execution.
  4. Comments & Code Structure — Comments (//, /* */, /** */) document code without affecting execution. Java code is organized into classes, which live in files matching the public class name.

Why This Week Matters: Before writing any logic, you need to understand how Java programs are created, compiled, and executed. The JVM model is what makes Java unique, and the main method is the starting point for every program you will ever write.

Code Example

// HelloWorld.java
public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, Java World!");
    }
}

🛠 Practical Task

Install the JDK, write a HelloWorld.java program that prints your name and today's date as plain text, then compile and run it from the terminal using javac and java.

Week 2 — Variables & Primitive Data Types

Store data in memory using Java's strict, compile-time type system.

Core Focus

Storing data in memory using Java's strict type system.

Topics & Subtopics

  1. Primitive Types — Java has 8 primitives: int, double, boolean, char, byte, short, long, float. Each has a fixed memory size and value range.
  2. Variable Declaration & Initialization — Variables must be declared with an explicit type before use. Java enforces this at compile time to catch type errors early.
  3. Type Casting — Widening casts happen automatically, while narrowing casts require explicit syntax to prevent silent data loss.
  4. Constants with final — The final keyword locks a variable's value after assignment, defining unchangeable constants.

Why This Week Matters: Java's strict type system catches many bugs at compile time. Understanding primitives and how Java stores different kinds of data is the foundation for all logic you will write going forward.

Code Example

public class Variables {
    public static void main(String[] args) {
        int age = 25;
        double price = 19.99;
        char grade = 'A';
        boolean isActive = true;
        final double PI = 3.14159;
        System.out.println(age + " " + price + " " + grade);
    }
}

🛠 Practical Task

Week 3 — Operators & Type Conversion

Manipulate data using Java's full set of operator families.

Core Focus

Manipulating data using Java's operator families.

Topics & Subtopics

  1. Arithmetic Operators — +, -, *, /, %. Integer division truncates decimals — understanding this prevents common beginner bugs.
  2. Relational & Logical Operators — Relational operators compare values and return booleans, while logical operators (&&, ||, !) combine boolean expressions.
  3. Assignment & Compound Operators — Shortcuts like +=, -=, ++, -- update variables concisely, reducing repetitive code.
  4. String Concatenation vs Numeric Addition — The + operator concatenates strings but adds numbers. Mixing types follows left-to-right evaluation and is a classic source of confusion.

Why This Week Matters: Operators are the building blocks of all computation and decision-making. Without a solid understanding of how Java evaluates and converts types, many programs will produce unexpected results.

Code Example

public class Operators {
    public static void main(String[] args) {
        int a = 10, b = 3;
        System.out.println(a / b);            // 3 (integer division)
        System.out.println(a % b);            // 1 (remainder)
        boolean result = (a > b) && (b > 0); // true
        a += 5;
        System.out.println("Sum: " + a);
    }
}

Week 4 — Strings & the String Class

Work with text as immutable objects and learn the most important String methods.

Core Focus

Working with text as immutable objects in Java.

Topics & Subtopics

  1. String Immutability — Once created, a String object's content cannot change. Methods like concat() return new strings rather than modifying the original.
  2. Common String Methods — Methods like length(), substring(), toUpperCase(), and equals() are essential for nearly all real-world programs.
  3. String Comparison: == vs .equals() — == compares references while .equals() compares actual content. This distinction is one of Java's most important early lessons.
  4. StringBuilder Basics — StringBuilder is a mutable alternative for building strings efficiently in loops without creating new String objects each time.

Why This Week Matters: String manipulation is one of the most common tasks in any Java program. Knowing when to use String vs StringBuilder and how to compare strings correctly prevents serious bugs.

Code Example

public class Strings {
    public static void main(String[] args) {
        String name = "java";
        String upper = name.toUpperCase();
        System.out.println(name.equals("java")); // true
        StringBuilder sb = new StringBuilder();
        sb.append("Hello").append(", ").append("World!");
        System.out.println(sb.toString());
    }
}

Week 5 — Arrays & Basic Input

Store ordered collections of data and read user input from the console.

Core Focus

Storing collections of data and reading user input.

Topics & Subtopics

  1. Array Declaration & Initialization — Arrays hold a fixed-size ordered collection of same-typed elements, indexed from zero. They are objects in Java, even when holding primitives.
  2. Iterating Arrays — Standard for loops and the enhanced for-each loop traverse arrays, with for-each being simpler when indices are not needed.
  3. Multi-Dimensional Arrays — Java supports arrays of arrays like int[][] for grids or matrices, where each row can have a different length.
  4. Reading Input with Scanner — The Scanner class reads user input from the console for various types and is the standard beginner tool for interactive programs.

Why This Week Matters: Arrays are the foundation of data storage in Java. Understanding them prepares you for Java's full Collections Framework in the next phase.

Code Example

import java.util.Scanner;
public class ArraysAndInput {
    public static void main(String[] args) {
        int[] scores = {90, 85, 77};
        for (int score : scores) {
            System.out.println(score);
        }
        Scanner sc = new Scanner(System.in);
        System.out.print("Enter your name: ");
        String name = sc.nextLine();
    }
}

Phase 2 — Control Flow & Logic

Weeks 6–10: Learn to direct program execution, write reusable methods, and work with Java's powerful built-in data structures.

Week 6 — Conditional Statements

Make decisions in code using Java's conditional constructs.

Core Focus

Making decisions in code using conditional statements.

Topics & Subtopics

  1. if, else if, else — These statements branch execution based on boolean conditions evaluated top to bottom. Only the first true branch executes.
  2. switch Statements & Expressions — switch compares a single value against multiple cases. Modern Java supports arrow-style switch expressions that return values directly.
  3. Ternary Operator — The condition ? valueIfTrue : valueIfFalse syntax compresses simple if-else logic into one readable line.
  4. Nested Conditionals — Conditionals can be nested for multi-layered logic, though overuse hurts readability and should be avoided.

Why This Week Matters: Decision-making is the foundation of all program logic. Without conditionals, every program runs the same way regardless of input or state.

Code Example

public class Conditionals {
    public static void main(String[] args) {
        int age = 20;
        if (age >= 18) {
            System.out.println("Adult");
        } else {
            System.out.println("Minor");
        }
        String status = (age >= 18) ? "Eligible" : "Not Eligible";
        System.out.println(status);
    }
}

🛠 Practical Task

Week 7 — Loops & Iteration

Repeat actions efficiently using Java's loop constructs.

Core Focus

Repeating actions efficiently using Java's loop constructs.

Topics & Subtopics

  1. for Loop — The classic for loop combines initialization, condition, and increment in one line. It is ideal for a known number of iterations.
  2. while and do-while Loops — while checks the condition before each iteration, while do-while guarantees at least one execution. Choose based on whether the first run should be unconditional.
  3. break and continue — break exits a loop entirely, while continue skips to the next iteration, giving finer control over loop flow.
  4. Nested Loops — Loops inside loops process grids, matrices, and combinations, but must be used carefully to avoid performance issues.

Why This Week Matters: Loops are how programs do real work at scale. Almost every meaningful application involves repeating operations, and understanding all three loop types makes you flexible and precise.

Code Example

public class Loops {
    public static void main(String[] args) {
        for (int i = 1; i <= 5; i++) {
            if (i == 3) continue;
            System.out.println(i);
        }
        int n = 0;
        while (n < 3) {
            System.out.println("n=" + n);
            n++;
        }
    }
}

🛠 Practical Task

Week 8 — Methods (Functions)

Write reusable, organized blocks of logic using Java methods.

Core Focus

Writing reusable, organized blocks of logic.

Topics & Subtopics

  1. Method Declaration & Return Types — Methods bundle code under a name, accept parameters, and return a typed value or void. This is the primary unit of code reuse in Java.
  2. Parameters & Arguments — Parameters define what a method expects, while arguments are the actual values passed at call time. Java requires matching types.
  3. Method Overloading — Multiple methods can share a name if their parameter lists differ in type or count, letting one logical operation handle multiple input variations.
  4. Pass-by-Value Semantics — Java always passes copies of primitive values and copies of object references. Understanding this prevents confusion about whether a method can change a caller's variable.

Why This Week Matters: Methods are the most important unit of organization in Java. Once you write methods well, your code becomes reusable, testable, and much easier to understand.

Code Example

public class Methods {
    static int add(int a, int b) {
        return a + b;
    }
    static double add(double a, double b) {
        return a + b;
    }
    public static void main(String[] args) {
        System.out.println(add(2, 3));     // 5
        System.out.println(add(2.5, 3.5)); // 6.0
    }
}

Week 9 — Core Data Structures: ArrayList & HashMap

Move beyond fixed arrays to Java's flexible, dynamic collections.

Core Focus

Moving beyond fixed-size arrays to flexible collections from the Java Collections Framework.

Topics & Subtopics

  1. ArrayList Basics — ArrayList is a resizable array that grows automatically as elements are added. It is the default choice for ordered, dynamic lists.
  2. Common ArrayList Operations — Methods like add(), get(), remove(), and size() manage list contents, with for-each and streams for iteration.
  3. HashMap Basics — HashMap stores key-value pairs for fast lookups by key, ideal when data must be retrieved by a meaningful identifier.
  4. Generics Introduction (<T>) — Generics let collections specify the exact type they hold, like ArrayList<String>, providing compile-time type safety.

Why This Week Matters: Most real Java programs use ArrayList and HashMap far more than raw arrays. These two data structures cover the majority of everyday data storage and retrieval needs.

Code Example

import java.util.*;
public class DataStructures {
    public static void main(String[] args) {
        List<String> names = new ArrayList<>();
        names.add("Alice");
        names.add("Bob");
        Map<String, Integer> ages = new HashMap<>();
        ages.put("Alice", 30);
        System.out.println(names.get(0) + " is " + ages.get("Alice"));
    }
}

Week 10 — Sets, Queues & Sorting

Use specialized collections and learn to order data correctly.

Core Focus

Specialized collections and ordering data using Java's Collections utilities.

Topics & Subtopics

  1. HashSet & Uniqueness — HashSet stores unordered, duplicate-free elements and is ideal for membership checks and deduplication tasks.
  2. Queue & Deque — Queue models FIFO processing, while Deque supports insertion and removal from both ends — foundational for scheduling and buffering.
  3. Collections.sort() & Comparator — Collections.sort() orders lists naturally, while a custom Comparator defines alternate ordering logic, separating what to sort from how to compare.
  4. Iterating with Iterator — The Iterator interface provides a safe, explicit way to traverse and remove elements from a collection during iteration.

Why This Week Matters: Different problems require different data structures. Sets, queues, and sorting round out your Collections knowledge and give you the right tool for every situation.

Code Example

import java.util.*;
public class MoreCollections {
    public static void main(String[] args) {
        Set<Integer> uniqueNums = new HashSet<>(List.of(1, 2, 2, 3));
        List<String> names = new ArrayList<>(List.of("Zoe", "Amy", "Bo"));
        Collections.sort(names);
        Queue<String> tasks = new LinkedList<>();
        tasks.add("Task1");
        System.out.println(names + " " + uniqueNums);
    }
}

Phase 3 — Intermediate & Advanced Core

Weeks 11–17: Master object-oriented design, robust error handling, file I/O, and Java's functional programming tools.

Week 11 — Classes & Objects

Model real-world entities with Java's object-oriented programming model.

Core Focus

Modeling real-world entities with object-oriented programming.

Topics & Subtopics

  1. Defining Classes & Fields — A class is a blueprint defining fields (state) and methods (behavior) for objects. Each object maintains its own copy of instance fields.
  2. Constructors — Constructors initialize new objects, sharing the class name and having no return type. Java provides a default no-arg constructor if none is defined.
  3. this Keyword — this refers to the current object instance, used to disambiguate fields from parameters of the same name.
  4. Instantiating Objects with new — The new keyword allocates memory and invokes a constructor to produce a usable object reference.

Why This Week Matters: Classes and objects are the fundamental unit of Java design. Once you understand how to define and instantiate objects, you can model any real-world system in code.

Code Example

public class Car {
    String model;
    int year;
    Car(String model, int year) {
        this.model = model;
        this.year = year;
    }
    public static void main(String[] args) {
        Car myCar = new Car("Tesla", 2024);
        System.out.println(myCar.model + " " + myCar.year);
    }
}

🛠 Practical Task

Week 12 — Encapsulation & Access Modifiers

Protect and control access to object data using Java's encapsulation features.

Core Focus

Protecting and controlling access to object data.

Topics & Subtopics

  1. Access Modifiers (private, public, protected) — These control visibility of fields and methods across classes and packages. Restricting access is central to safe, maintainable design.
  2. Getters & Setters — Instead of exposing fields directly, classes provide controlled access via getter and setter methods, allowing validation when data is read or modified.
  3. Encapsulation Principle — Bundling data with the methods that operate on it while hiding internal details reduces unintended interference between unrelated parts of a program.
  4. Static Fields & Methods — static members belong to the class itself rather than any instance and are shared across all objects. They are useful for counters, constants, and utility methods.

Why This Week Matters: Encapsulation is one of the four pillars of OOP. Without it, any code can freely change any data, leading to unpredictable bugs. Good encapsulation makes systems predictable and safe.

Code Example

public class BankAccount {
    private double balance;
    public double getBalance() { return balance; }
    public void deposit(double amt) {
        if (amt > 0) balance += amt;
    }
    public static void main(String[] args) {
        BankAccount acc = new BankAccount();
        acc.deposit(100);
        System.out.println(acc.getBalance());
    }
}

Week 13 — Inheritance & Polymorphism

Build class hierarchies and write flexible, reusable behavior across related types.

Core Focus

Building class hierarchies and flexible, reusable behavior.

Topics & Subtopics

  1. extends & Superclasses — A subclass inherits fields and methods from its superclass using extends, promoting code reuse. Java supports single inheritance only.
  2. Method Overriding & @Override — A subclass can redefine a parent method with the same signature, marked with @Override for clarity and compile-time verification.
  3. Polymorphism — A parent-type reference can point to any child-type object, and the correct overridden method runs at runtime. This allows code to work across a whole family of related types.
  4. super Keyword — super accesses the parent class constructor or overridden methods, essential when a subclass needs to extend rather than fully replace parent behavior.

Why This Week Matters: Inheritance and polymorphism are the reason OOP is so powerful. They let you write flexible code that works across many types without having to know the exact type in advance.

Code Example

class Animal {
    void speak() { System.out.println("Some sound"); }
}
class Dog extends Animal {
    @Override
    void speak() { System.out.println("Woof!"); }
}
public class InheritanceDemo {
    public static void main(String[] args) {
        Animal a = new Dog();
        a.speak(); // prints "Woof!"
    }
}

Week 14 — Abstraction: Abstract Classes & Interfaces

Design contracts and partial implementations using Java's abstraction tools.

Core Focus

Designing contracts and partial implementations.

Topics & Subtopics

  1. Abstract Classes — An abstract class cannot be instantiated directly and may contain both abstract and concrete methods. It is used when subclasses share some logic but must implement certain parts differently.
  2. Interfaces — An interface defines a pure contract of method signatures that implementing classes must fulfill, enabling unrelated classes to share behavior without inheritance.
  3. implements vs extends — Classes use implements for interfaces and extends for classes, and a class can implement multiple interfaces simultaneously.
  4. Default & Static Interface Methods — Modern Java interfaces can include default methods with a body, allowing evolution of APIs without breaking existing implementations.

Why This Week Matters: Interfaces are the foundation of loosely coupled Java design. Every major Java framework and library is built on interfaces, and understanding them is non-negotiable for professional Java development.

Code Example

interface Drivable {
    void drive();
    default void honk() {
        System.out.println("Beep!");
    }
}
class Car implements Drivable {
    public void drive() { System.out.println("Driving..."); }
}
public class AbstractionDemo {
    public static void main(String[] args) {
        new Car().drive();
    }
}

Week 15 — Exception Handling

Write resilient code that handles errors gracefully instead of crashing.

Core Focus

Writing resilient code that handles errors gracefully.

Topics & Subtopics

  1. try-catch-finally — Code that might fail goes in try, errors are caught in catch, and finally always runs. This isolates risky operations from crashing the whole program.
  2. Checked vs Unchecked Exceptions — Checked exceptions must be declared or caught at compile time, while unchecked ones are not enforced. This distinction shapes how Java APIs signal recoverable vs programming errors.
  3. Custom Exceptions — You can create domain-specific exception classes by extending Exception or RuntimeException, making error handling more meaningful.
  4. throw & throws — throw raises an exception instance, while throws in a method signature declares that callers must handle it.

Why This Week Matters: Production Java code must handle failure gracefully. If exceptions crash the program, users lose data and trust. Proper exception handling makes software reliable.

Code Example

class InvalidAgeException extends RuntimeException {
    InvalidAgeException(String msg) { super(msg); }
}
public class ExceptionDemo {
    static void checkAge(int age) {
        if (age < 0) throw new InvalidAgeException("Age can't be negative");
    }
    public static void main(String[] args) {
        try { checkAge(-5); }
        catch (InvalidAgeException e) { System.out.println(e.getMessage()); }
    }
}

Week 16 — File I/O & Serialization

Persist and read data beyond program runtime using Java's file APIs.

Core Focus

Persisting and reading data beyond program runtime.

Topics & Subtopics

  1. Reading Files with BufferedReader/Files — Java provides multiple text file reading APIs, from classic streams to the modern Files.readAllLines(). Choosing depends on file size and required control.
  2. Writing Files with FileWriter/Files — Writing data to disk uses similarly paired APIs, with try-with-resources ensuring streams close properly.
  3. Try-With-Resources — This syntax automatically closes resources like file streams when a block finishes, even if an exception occurs, eliminating manual finally cleanup.
  4. Object Serialization — Implementing Serializable allows whole Java objects to be written to and read from files in binary form, useful for saving complex application state.

Why This Week Matters: Nearly every real application needs to read and write files — logs, configuration, user data, exports. File I/O connects your program to the outside world permanently.

Code Example

import java.io.*;
import java.nio.file.*;
public class FileDemo {
    public static void main(String[] args) throws IOException {
        Files.writeString(Path.of("notes.txt"), "Hello File!");
        String content = Files.readString(Path.of("notes.txt"));
        System.out.println(content);
    }
}

Week 17 — Generics, Lambdas & the Stream API

Write concise, type-safe, functional-style Java code using modern language features.

Core Focus

Writing concise, type-safe, functional-style Java code.

Topics & Subtopics

  1. Generic Classes & Methods — You can define classes and methods parameterized by type, like Box<T>, for reusable type-safe components beyond just collections.
  2. Lambda Expressions — Lambdas provide concise syntax for implementing functional interfaces inline, eliminating boilerplate anonymous class syntax for simple behaviors.
  3. Functional Interfaces — Interfaces with a single abstract method like Runnable or Comparator can be implemented via lambdas. Java also provides built-ins like Function, Predicate, and Consumer.
  4. Stream API Basics — Streams process collections declaratively using chains like filter(), map(), and collect(), shifting from manual loops to pipeline-style data transformations.

Why This Week Matters: Modern Java development relies heavily on lambdas and streams. They reduce boilerplate, improve readability, and make data processing far more expressive than equivalent loop-based code.

Code Example

import java.util.*;
import java.util.stream.*;
public class StreamsDemo {
    public static void main(String[] args) {
        List<Integer> nums = List.of(1, 2, 3, 4, 5, 6);
        List<Integer> evenSquares = nums.stream()
            .filter(n -> n % 2 == 0)
            .map(n -> n * n)
            .collect(Collectors.toList());
        System.out.println(evenSquares); // [4, 16, 36]
    }
}

Phase 4 — Master Level Concepts

Weeks 18–24: Tackle concurrency, JVM internals, design patterns, and build three complete production-grade projects.

Week 18 — Multithreading & Concurrency Basics

Run code in parallel safely using Java's threading and executor models.

Core Focus

Running code in parallel safely.

Topics & Subtopics

  1. Thread & Runnable — A Thread represents an independent execution path, while Runnable defines its task. Implementing Runnable is generally preferred over extending Thread.
  2. Thread Lifecycle — Threads move through states like New, Runnable, Blocked, and Terminated, managed by the JVM scheduler. Understanding this helps diagnose hangs and deadlocks.
  3. Synchronization (synchronized) — The synchronized keyword ensures only one thread accesses a critical section at a time, preventing race conditions on shared mutable state.
  4. ExecutorService — ExecutorService manages a pool of reusable threads instead of manually creating new ones for every task — the modern, recommended concurrency approach.

Why This Week Matters: Multi-core CPUs are standard. Any serious Java application — servers, tools, simulations — needs concurrency. Understanding threading basics is essential before moving to more advanced async patterns.

Code Example

import java.util.concurrent.*;
public class ThreadsDemo {
    public static void main(String[] args) throws InterruptedException {
        ExecutorService pool = Executors.newFixedThreadPool(2);
        pool.submit(() -> System.out.println("Task A running"));
        pool.submit(() -> System.out.println("Task B running"));
        pool.shutdown();
    }
}

🛠 Practical Task

Week 19 — Advanced Concurrency: Locks, Atomics & Concurrent Collections

Fine-grained control over thread safety and performance using modern concurrency tools.

Core Focus

Fine-grained control over thread safety and performance.

Topics & Subtopics

  1. ReentrantLock — Offers more flexible locking than synchronized, including try-locking and explicit unlock control for complex concurrency scenarios.
  2. Atomic Variables (AtomicInteger, etc.) — Atomic classes perform thread-safe operations without explicit locking, using low-level CPU instructions. Far more efficient than synchronized blocks for simple counters.
  3. ConcurrentHashMap — This thread-safe map allows concurrent reads and writes without locking the entire structure — the standard replacement for synchronized HashMap.
  4. CompletableFuture — Represents an asynchronous computation that can be chained, combined, and composed without blocking threads, central to modern non-blocking Java code.

Why This Week Matters: Basic synchronization with locks is often too coarse for high-performance systems. Atomics and concurrent collections reduce contention dramatically and are used throughout Java frameworks.

Code Example

import java.util.concurrent.atomic.*;
import java.util.concurrent.*;
public class AdvancedConcurrency {
    public static void main(String[] args) {
        AtomicInteger counter = new AtomicInteger(0);
        counter.incrementAndGet();
        CompletableFuture.supplyAsync(() -> "Done")
            .thenAccept(System.out::println);
    }
}

Week 20 — Design Patterns & Best Practices

Write maintainable, idiomatic Java at a professional level using proven design patterns.

Core Focus

Writing maintainable, idiomatic Java at a professional level.

Topics & Subtopics

  1. Singleton Pattern — Ensures a class has exactly one globally accessible instance. Thread-safe implementation requires careful handling in concurrent contexts.
  2. Factory Pattern — Centralizes object creation logic behind a method, decoupling client code from concrete class instantiation and making systems easier to extend.
  3. Builder Pattern — Builds complex objects step-by-step via chained method calls, avoiding unwieldy constructors with many parameters. Especially popular for immutable objects with optional fields.
  4. SOLID Principles Overview — SOLID (Single Responsibility, Open/Closed, Liskov Substitution, Interface Segregation, Dependency Inversion) guides maintainable OOP design at a professional level.

Why This Week Matters: Design patterns are proven solutions to recurring software design problems. Knowing them means you can communicate solutions clearly with other developers and write code that is far easier to maintain.

Code Example

public class User {
    private final String name;
    private final int age;
    private User(Builder b) { this.name = b.name; this.age = b.age; }
    static class Builder {
        String name; int age;
        Builder setName(String n) { this.name = n; return this; }
        Builder setAge(int a) { this.age = a; return this; }
        User build() { return new User(this); }
    }
}

Week 21 — JVM Internals & Performance Tuning

Understand what happens under the hood for optimized, efficient Java code.

Core Focus

Understanding what happens under the hood for optimized code.

Topics & Subtopics

  1. Garbage Collection Basics — The JVM automatically reclaims memory from unreferenced objects using algorithms like G1 or ZGC. Understanding GC helps diagnose memory leaks and pause-time issues.
  2. Heap vs Stack Memory — The stack stores method calls and local variables, while the heap stores objects managed by the garbage collector. Misunderstanding this causes StackOverflowError or OutOfMemoryError.
  3. JIT Compilation — The Just-In-Time compiler converts frequently-run bytecode into optimized native machine code at runtime. This is why long-running Java applications often get faster over time.
  4. Profiling Tools (JFR, VisualVM) — Tools like Java Flight Recorder and VisualVM inspect CPU, memory, and thread behavior of a running JVM. Profiling before optimizing is essential to avoid guessing at bottlenecks.

Why This Week Matters: Senior Java developers understand why their code is fast or slow, not just how to write it. JVM knowledge separates engineers who can truly debug and optimize production systems from those who cannot.

Code Example

public class MemoryDemo {
    public static void main(String[] args) {
        Runtime rt = Runtime.getRuntime();
        System.out.println("Max Heap: " + rt.maxMemory() / 1_000_000 + "MB");
        System.out.println("Free Memory: " + rt.freeMemory() / 1_000_000 + "MB");
        // Try running with -Xmx flags to see this change
    }
}
