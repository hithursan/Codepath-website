About This Curriculum

This is a 16-week, phase-based PHP curriculum designed to take a complete beginner to a professional-level PHP developer. The roadmap moves from PHP environment setup, syntax, variables, and operators through conditionals, loops, functions, forms, sessions, and file uploads, into object-oriented PHP, file I/O, PDO, and Composer, all the way to advanced patterns, REST APIs, security, testing, and three complete capstone projects.

The curriculum is organized into four phases: Phase 1 covers absolute basics and PHP fundamentals, Phase 2 covers control flow, functions, forms, and sessions, Phase 3 covers OOP, file I/O, error handling, and PDO/MySQL, and Phase 4 covers master-level patterns, REST APIs, security, and full production applications.

By Week 16, you will have built three complete, portfolio-ready applications and be confident building production-grade PHP applications from scratch.

Level: Beginner → Master  |  Duration: 16 weeks  |  Stack: PHP 8.2+, MySQL, Composer, Apache/Nginx  |  Prerequisites: Basic HTML & programming logic

Week 1 — Environment Setup & PHP Fundamentals

Installing PHP, understanding how it runs on a server, and writing your very first PHP scripts.

Core Focus

Installing PHP, understanding how it runs on a server, and writing your very first PHP scripts.

Topics & Subtopics

  1. What is PHP & How It Works — PHP (Hypertext Preprocessor) is a server-side scripting language that runs on a web server and generates HTML sent to the browser. Unlike JavaScript, PHP code is never visible to the end user — only its output is.
  2. Setting Up a Local Environment (XAMPP / Laragon) — XAMPP and Laragon bundle Apache, PHP, and MySQL into a single installer, giving you a full local server environment in minutes. Place your .php files in the htdocs (XAMPP) or www (Laragon) folder and access them via http://localhost.
  3. Variables & Data Types — PHP variables always start with a $ sign and are loosely typed — you don't declare the type; PHP infers it from the assigned value. PHP 8+ supports named types for stricter code, but the dynamic default remains fully valid.
  4. echo, print, & var_dump — echo and print both output text to the browser, but echo is faster and can accept multiple comma-separated values. var_dump() is your debugging best friend — it reveals a variable's type and value simultaneously.

Why This Week Matters: PHP is the language behind WordPress, Laravel, and a huge portion of the web. Understanding the server-side execution model — PHP runs on the server and sends only output to the browser — is the foundational mental model that makes every subsequent topic logical.

Code Example

<?php
// PHP code lives between <?php and ?> tags
echo "Hello, World!";

$name    = "Alice";   // string
$age     = 28;        // integer
$height  = 5.6;       // float
$isAdmin = true;      // boolean
$nothing = null;      // null

// var_dump() shows the value AND its type
var_dump($age);  // int(28)

$product = "Laptop";
$price   = 999.99;
echo "Product: ", $product, " — $", $price, "<br>";
?>
<p>This is plain HTML after the PHP block.</p>

Week 2 — Strings, Numbers & Operators

Manipulating text and numbers using PHP's rich library of built-in string and math functions.

Core Focus

Manipulating text and numbers using PHP's rich library of built-in string and math functions.

Topics & Subtopics

  1. String Functions — PHP ships with over 100 built-in string functions. The most essential ones are strlen() (length), strtolower()/strtoupper() (case), str_replace() (find and replace), trim() (whitespace removal), and substr() (extract a portion).
  2. String Interpolation & Heredoc — PHP supports embedding variables directly inside double-quoted strings — the variable is replaced with its value at runtime. For long blocks of text, the Heredoc (<<<) syntax avoids messy quote escaping.
  3. Arithmetic & Assignment Operators — PHP supports all standard arithmetic operators (+, -, *, /, %, **) plus shorthand assignment operators like += and -=. The integer division operator intdiv() and modulo % are especially useful for pagination and grid layouts.
  4. Math Functions — PHP's Math functions cover everything from rounding to trigonometry. The most commonly used in web development are round(), ceil(), floor(), abs(), max(), min(), and rand().

Why This Week Matters: String manipulation is constant in PHP — sanitizing user input, formatting output, building HTML responses. Mastering the core string functions and the difference between single-quoted (literal) and double-quoted (interpolated) strings prevents an entire category of beginner bugs.

Code Example

<?php
$sentence = "  Hello, PHP World!  ";
echo strlen(trim($sentence));            // 19
echo strtoupper($sentence);
echo str_replace("PHP", "8", $sentence);
echo substr(trim($sentence), 0, 5);     // "Hello"

$name = "Bob";
$role = "developer";
echo "Welcome, $name! You are a $role.<br>";

echo <<<HTML
    <h1>Hello, {$name}!</h1>
    <p>Your role is: {$role}</p>
HTML;

$a = 10; $b = 3;
echo $a % $b;        // 1
echo $a ** $b;       // 1000
echo intdiv($a, $b); // 3

echo round(4.567, 2); // 4.57
echo ceil(4.1);        // 5
echo max(3, 7, 1, 9); // 9
echo rand(1, 100);

Week 3 — Arrays

Storing and manipulating collections of data using PHP's versatile indexed and associative arrays.

Core Focus

Storing and manipulating collections of data using PHP's versatile indexed and associative arrays.

Topics & Subtopics

  1. Indexed Arrays — An indexed array stores values in sequential numeric positions starting at 0. You can create them with square bracket syntax [] or the array() function and access any element by its numeric index.
  2. Associative Arrays — Associative arrays use named string keys instead of numeric indexes, making them ideal for representing structured records like a user profile or product. They behave like dictionaries or hash maps in other languages.
  3. Array Functions — PHP provides dozens of powerful array functions. Essential ones include array_push(), array_pop(), array_merge(), array_slice(), in_array(), array_search(), sort(), and array_map().
  4. Multidimensional Arrays — A multidimensional array is an array where each element is itself an array — perfect for representing tables of data like a list of users, products, or database rows before a real database is introduced.

Why This Week Matters: Arrays are the most-used data structure in PHP — every database query result, form submission, and JSON response is processed as an array. Mastering foreach, array_map, array_filter, and associative arrays is non-negotiable for real PHP work.

$fruits = ["Apple", "Banana", "Cherry", "Date"];

    ["name" => "Alice", "grade" => 92],

Week 4 — Type System, Type Casting & Constants

Understanding PHP's type juggling, explicit type casting, strict types, and defining constants.

Core Focus

Understanding PHP's type juggling, explicit type casting, strict types, and defining constants.

Topics & Subtopics

  1. Type Juggling & strict_types — PHP automatically converts types when performing operations (e.g., "5" + 3 becomes 8). Adding declare(strict_types=1) at the top of a file disables this juggling and enforces strict type matching for function arguments.
  2. Type Casting — You can explicitly cast a value to a different type using cast operators: (int), (float), (string), (bool), and (array). This is safer than relying on PHP's automatic juggling.
  3. Constants with define() and const — Constants hold values that never change during script execution — like application version numbers, tax rates, or API endpoint URLs. Unlike variables, they have no $ prefix and are globally accessible anywhere in the script.
  4. Type Declaration Functions (is_* & gettype) — PHP provides a family of is_* functions to check a variable's type at runtime without triggering errors. gettype() returns the type as a string, and intval(), floatval(), and strval() are safer alternatives to casting.

Why This Week Matters: PHP's type juggling is famous for producing surprising results ("0" == false, "abc" == 0 in older versions). Enabling strict_types=1 and using strict comparison (===) from the start prevents an entire category of subtle bugs that have plagued PHP codebases for decades.

Code Example

<?php
declare(strict_types=1);

var_dump("5" + 3);     // int(8)
var_dump("5" . 3);     // string("53")
var_dump(0 === false); // bool(false)

$price   = "29.99 USD";
$amount  = (float) $price;
$rounded = (int) $amount;
$text    = (string) 42;
$flag    = (bool) "";

define("APP_VERSION", "2.1.0");
define("TAX_RATE", 0.08);
const MAX_UPLOAD_SIZE = 5242880;

echo APP_VERSION;

$values = [42, "hello", 3.14, true, null, []];
foreach ($values as $v) {
    echo gettype($v) . ": ";
    echo is_numeric($v) ? "numeric" : "not numeric";
    echo "<br>";
}

Week 5 — Conditionals & Loops

Controlling program execution flow using if/else statements, match, and all loop types.

Core Focus

Controlling program execution flow using if/else statements, match, and all loop types.

Topics & Subtopics

  1. if, elseif, else & the Ternary Operator — PHP's conditional statements work exactly like most C-family languages. The ternary operator ? : provides a compact one-line alternative, and the null coalescing operator ?? (PHP 7+) is perfect for handling missing values.
  2. match Expression (PHP 8+) — The match expression is PHP 8's cleaner, stricter replacement for switch. It uses strict comparison (no type juggling), each arm returns a value directly, and it throws an UnhandledMatchError if no arm matches (unlike switch which silently falls through).
  3. for, while, and do...while Loops — for loops are best when you know the iteration count in advance. while loops run as long as a condition is true. do...while always executes at least once before checking the condition — useful for menus and input validation.
  4. foreach Loop — foreach is PHP's most-used loop, designed specifically for iterating over arrays and objects. You can iterate over just values, or destructure both key and value in every iteration using the => syntax.

Why This Week Matters: PHP 8's match expression is a major upgrade over switch — strict comparison, no fall-through bugs, and it returns a value directly. foreach with key destructuring is the most-used pattern in real PHP code for processing arrays and database results.

Code Example

<?php
$score = 78;
if ($score >= 90)      echo "A";
elseif ($score >= 80)  echo "B";
elseif ($score >= 70)  echo "C";
else                   echo "F";

$username = $_GET["user"] ?? "Guest";

$status  = 404;
$message = match($status) {
    200, 201 => "Success",
    301, 302 => "Redirect",
    404      => "Not Found",
    500      => "Server Error",
    default  => "Unknown Status",
};
echo $message;

for ($i = 1; $i <= 5; $i++) echo "$i ";

$cart = ["Apple" => 1.20, "Banana" => 0.50, "Cherry" => 2.99];
$total = 0;

    echo "$item: \$$price<br>";
    $total += $price;
}
echo "Total: \$" . number_format($total, 2);

Week 6 — Functions

Defining reusable, type-safe functions with default arguments, variadic parameters, and return types.

Core Focus

Defining reusable, type-safe functions with default arguments, variadic parameters, and return types.

Topics & Subtopics

  1. Defining Functions & Return Types — Functions in PHP are defined with the function keyword. PHP 7+ allows you to declare parameter types and return types with a colon — this makes your functions self-documenting and catches type errors early.
  2. Default & Named Arguments (PHP 8+) — Default parameter values make arguments optional by providing a fallback. PHP 8's named arguments let you pass values by their parameter name rather than position, making function calls with many optional parameters far more readable.
  3. Variadic Functions (...$args) — Variadic functions accept any number of arguments by prefixing the last parameter with .... The arguments are collected into an array, letting you write functions like sum() or log() that work with any quantity of inputs.
  4. Arrow Functions & Closures — Anonymous functions (closures) are functions without a name, stored in variables or passed as arguments. PHP 7.4's arrow functions (fn() =>) are a short-syntax closure that automatically capture variables from the outer scope.

Why This Week Matters: Typed function signatures with PHP 8's named arguments produce self-documenting, IDE-friendly code that catches errors at the boundary. Arrow functions are essential for clean array_map/array_filter chains — the functional style now standard in modern PHP.

Code Example

<?php
function calculateArea(float $width, float $height): float {
    return $width * $height;
}
echo calculateArea(5.5, 3.0);

function greet(string $name, string $greeting = "Hello", bool $shout = false): string {
    $msg = "$greeting, $name!";
    return $shout ? strtoupper($msg) : $msg;
}
echo greet(name: "Bob", shout: true);

function sum(string $label, int ...$numbers): string {
    $total = array_sum($numbers);
    return "$label: $total";
}
echo sum("Score", 10, 20, 5, 15);

$multiplier = 3;
$triple = fn(int $n): int => $n * $multiplier;
$tripled = array_map($triple, [1, 2, 3, 4, 5]);
echo implode(", ", $tripled);

Week 7 — Superglobals, Forms & Input Handling

Processing HTML form submissions securely using $_GET, $_POST, and input sanitization.

Core Focus

Processing HTML form submissions securely using $_GET, $_POST, and input sanitization.

Topics & Subtopics

  1. $_GET & Query Strings — $_GET is a superglobal array populated with data from the URL query string (everything after ?). It's appropriate for non-sensitive data like search queries, filters, and pagination, but never for passwords or personal information.
  2. $_POST & Form Handling — $_POST holds data submitted via an HTML form with method="POST". POST data is not visible in the URL and is slightly more secure for sensitive data, though HTTPS is required for true security.
  3. Input Validation & Sanitization — Sanitization cleans input (removing dangerous characters); validation checks that input meets your rules (correct format, required fields). PHP's filter_var() function handles both tasks cleanly for common cases.
  4. $_SERVER & Request Information — The $_SERVER superglobal contains information about the web server and the current HTTP request — including the request method, client IP, script path, and HTTP headers. It's essential for routing and security checks.

Why This Week Matters: Unsanitized user input is the #1 source of vulnerabilities in PHP applications — XSS, SQL injection, and CSRF all start here. Mastering htmlspecialchars(), filter_var(), and the validation-before-use pattern is non-negotiable for any production PHP code.

Code Example

<?php
$query = htmlspecialchars($_GET["query"] ?? "");
$page  = (int) ($_GET["page"] ?? 1);

if ($_SERVER["REQUEST_METHOD"] === "POST") {
    $name  = trim($_POST["name"] ?? "");
    $email = filter_input(INPUT_POST, "email", FILTER_SANITIZE_EMAIL);
    echo "Name: $name | Email: $email";
}

$rawEmail = "  USER@Example.COM  ";
$email    = filter_var(trim($rawEmail), FILTER_SANITIZE_EMAIL);
$isValid  = filter_var($email, FILTER_VALIDATE_EMAIL);
echo $isValid ? "Valid: $email" : "Invalid email";

echo $_SERVER["REQUEST_METHOD"];
echo $_SERVER["REMOTE_ADDR"];
echo $_SERVER["HTTP_USER_AGENT"];

Week 8 — Sessions, Cookies & File Uploads

Persisting user data across requests with sessions and cookies, and safely handling file uploads.

Core Focus

Persisting user data across requests with sessions and cookies, and safely handling file uploads.

Topics & Subtopics

  1. Sessions — HTTP is stateless — every request is independent. PHP Sessions solve this by storing data on the server and giving the browser a session ID cookie. All pages in the same session can read and write to the same $_SESSION array.
  2. Cookies — Cookies store small amounts of data in the browser. They persist across browser sessions (unlike PHP sessions, which expire when the browser closes by default). Never store sensitive data in cookies — they're readable by the browser.
  3. File Uploads with $_FILES — PHP handles file uploads via the $_FILES superglobal. You must validate the file type, size, and use move_uploaded_file() to move it from the temporary directory to your target location safely.
  4. Session-Based Authentication Pattern — The most common use of sessions is implementing a login system. After verifying credentials, you set a session variable to mark the user as authenticated, and every protected page checks for that variable before rendering.

Why This Week Matters: Sessions are the foundation of every login system, shopping cart, and personalized PHP application on the web. Mishandled file uploads are a top vulnerability — accepting any file without type/size validation can let attackers execute arbitrary PHP on your server.

Code Example

<?php
session_start();

$_SESSION["username"] = "Alice";
$_SESSION["role"]     = "admin";
echo "Logged in as: " . ($_SESSION["username"] ?? "Guest");

$expires = time() + (7 * 24 * 60 * 60);
setcookie("theme", "dark", $expires, "/");
$theme = $_COOKIE["theme"] ?? "light";

if ($_SERVER["REQUEST_METHOD"] === "POST" && isset($_FILES["photo"])) {
    $file    = $_FILES["photo"];
    $allowed = ["image/jpeg", "image/png", "image/webp"];
    if (in_array($file["type"], $allowed) && $file["size"] < 2_000_000) {
        move_uploaded_file($file["tmp_name"], "uploads/" . basename($file["name"]));
        echo "Upload successful!";
    }
}

function requireLogin(): void {
    if (empty($_SESSION["user_id"])) {
        header("Location: /login.php");
        exit;
    }
}

Week 9 — Object-Oriented PHP: Classes & Objects

Modeling real-world entities with classes, properties, methods, visibility modifiers, and constructors.

Core Focus

Modeling real-world entities with classes, properties, methods, visibility modifiers, and constructors.

Topics & Subtopics

  1. Classes, Properties & Methods — A class is a blueprint for creating objects — it defines the properties (data) and methods (behavior) that every object of that type will have. In PHP 8+, constructor property promotion lets you declare and assign properties in a single line.
  2. Visibility: public, protected, private — Visibility modifiers control access to class members. public is accessible anywhere, private only within the same class, and protected within the class and its subclasses. This encapsulation protects an object's internal state from outside interference.
  3. Inheritance & parent:: — Inheritance lets a child class extend a parent class, reusing its properties and methods while adding or overriding behaviour. The parent:: keyword calls the parent class's version of a method, commonly used in __construct() to chain initialisation.
  4. Static Methods & Properties — Static members belong to the class itself, not to any specific instance. They're accessed with :: and are ideal for utility/helper methods, counters, or factory patterns where you don't need (or want) to instantiate an object.

Why This Week Matters: PHP 8's constructor property promotion eliminates massive amounts of boilerplate. Combined with strict visibility and typed properties, modern PHP OOP is now as clean and safe as Java or C# — a far cry from the messy PHP 5 era.

Code Example

<?php
class Product {
    public function __construct(
        private string $name,
        private float  $price,
        private int    $stock = 0,
    ) {}

    public function getLabel(): string {
        return "{$this->name} — \${$this->price}";
    }
}
$p = new Product("Laptop", 999.99, 10);
echo $p->getLabel();

class BankAccount {
    private float $balance;
    public function __construct(float $initial) { $this->balance = $initial; }
    public function deposit(float $amount): void {
        if ($amount > 0) $this->balance += $amount;
    }
    public function getBalance(): float { return $this->balance; }
}

class Animal {
    public function __construct(protected string $name) {}
    public function speak(): string { return "..."; }
}
class Dog extends Animal {
    public function speak(): string {
        return $this->name . " says: Woof!";
    }
}

class MathHelper {
    private static int $callCount = 0;
    public static function square(float $n): float {
        self::$callCount++;
        return $n * $n;
    }
}
echo MathHelper::square(5);

Week 10 — Advanced OOP: Interfaces, Traits, Abstracts & Enums

Designing flexible, extensible systems using PHP's full object-oriented toolkit.

Core Focus

Designing flexible, extensible systems using PHP's full object-oriented toolkit.

Topics & Subtopics

  1. Interfaces — An interface is a contract — it defines which methods a class must implement without providing any implementation. This enables polymorphism: different classes can be treated interchangeably as long as they implement the same interface.
  2. Abstract Classes — Abstract classes sit between regular classes and interfaces — they can provide concrete method implementations and declare abstract methods that subclasses must implement. A class can only extend one abstract class, unlike interfaces.
  3. Traits — Traits solve the multiple-inheritance problem — they let you package reusable methods and inject them into any class with use. Unlike interfaces, traits provide actual implementation. A class can use multiple traits simultaneously.
  4. Enums (PHP 8.1+) — Enums define a fixed set of named values, replacing the fragile pattern of using string or integer constants. Backed enums associate each case with a string or integer value, making them suitable for database storage and HTTP responses.

Why This Week Matters: Enums (PHP 8.1+) eliminate an entire class of bugs around magic strings/integers used for status codes, roles, and categories. Combined with interfaces and traits, PHP now has the full toolkit needed to write SOLID, testable, framework-quality code.

Code Example

<?php
interface Exportable {
    public function toCSV(): string;
    public function toJSON(): string;
}

class Report implements Exportable {
    public function __construct(private array $data) {}
    public function toCSV(): string  { return implode(",", $this->data); }
    public function toJSON(): string { return json_encode($this->data); }
}

abstract class Shape {
    public function describe(): string { return "I am a " . static::class; }
    abstract public function area(): float;
}
class Circle extends Shape {
    public function __construct(private float $radius) {}
    public function area(): float { return M_PI * $this->radius ** 2; }
}

trait Timestampable {
    private ?string $createdAt = null;
    public function setCreatedAt(): void { $this->createdAt = date("Y-m-d H:i:s"); }
    public function getCreatedAt(): ?string { return $this->createdAt; }
}
class Article {
    use Timestampable;
    public function __construct(public string $title) {}
}

enum OrderStatus: string {
    case Pending   = "pending";
    case Shipped   = "shipped";
    case Delivered = "delivered";
    case Cancelled = "cancelled";
}
$status = OrderStatus::Shipped;
echo $status->value;   // "shipped"
$fromDB = OrderStatus::from("delivered");

Week 11 — File Handling, Error Handling & Exceptions

Reading/writing files, handling runtime errors gracefully, and building a robust exception hierarchy.

Core Focus

Reading/writing files, handling runtime errors gracefully, and building a robust exception hierarchy.

Topics & Subtopics

  1. Reading & Writing Files — PHP's file functions make reading and writing files straightforward. file_get_contents() reads an entire file into a string; file_put_contents() writes a string to a file. For large files, use fopen()/fgets()/fclose() to stream line-by-line.
  2. Try / Catch / Finally & Exceptions — PHP exceptions let you handle errors gracefully without crashing the script. Code that might fail goes in try, your handler goes in catch, and finally runs regardless of whether an exception was thrown — perfect for cleanup code.
  3. Custom Exception Classes — Custom exceptions make your error handling expressive and precise. By extending Exception (or RuntimeException), you create domain-specific exception types that are far more meaningful than a generic Exception.
  4. Working with JSON Files — JSON is the standard format for storing and exchanging structured data. PHP's json_encode() converts arrays/objects to JSON strings, and json_decode() parses them back — making it easy to use JSON files as lightweight data stores.

Why This Week Matters: Exceptions are PHP's idiomatic error-handling mechanism. Custom exception classes carrying domain-specific context (like a shortfall amount or a validation field name) transform debugging from guesswork into precise diagnosis — the hallmark of production-quality PHP.

Code Example

<?php
$path = "data/notes.txt";
file_put_contents($path, "Line 1\nLine 2\nLine 3\n");
$content = file_get_contents($path);
$lines   = file($path, FILE_IGNORE_NEW_LINES | FILE_SKIP_EMPTY_LINES);

function divide(float $a, float $b): float {
    if ($b === 0.0) throw new InvalidArgumentException("Cannot divide by zero.");
    return $a / $b;
}
try {
    echo divide(10, 2);
    echo divide(10, 0);
} catch (InvalidArgumentException $e) {
    echo "Error: " . $e->getMessage();
} finally {
    echo " | Operation complete.";
}

class InsufficientFundsException extends RuntimeException {
    public function __construct(private float $shortfall) {
        parent::__construct("Insufficient funds. Short by: \$$shortfall");
    }
    public function getShortfall(): float { return $this->shortfall; }
}

];
file_put_contents("users.json", json_encode($users, JSON_PRETTY_PRINT));

Week 12 — Databases with PDO & MySQL

Connecting to MySQL databases, performing CRUD operations safely with PDO and prepared statements.

Core Focus

Connecting to MySQL databases, performing CRUD operations safely with PDO and prepared statements.

Topics & Subtopics

  1. Connecting to MySQL with PDO — PDO (PHP Data Objects) is the modern, database-agnostic way to interact with databases in PHP. It supports multiple database drivers and, critically, makes prepared statements easy — the #1 defence against SQL injection.
  2. Prepared Statements & CRUD — Prepared statements separate SQL code from user data, making SQL injection impossible. The :placeholder syntax (named) or ? (positional) binds values safely at execution time — never concatenate user input into SQL strings.
  3. Fetching Data & fetchAll — PDO's fetch() retrieves one row at a time, while fetchAll() returns all matching rows as an array of arrays. Use rowCount() to check how many rows were affected by an UPDATE or DELETE.
  4. Transactions — A database transaction groups multiple SQL statements into an atomic unit — either all succeed or all are rolled back. This is essential for operations that must stay consistent, like transferring money between accounts.

Why This Week Matters: SQL injection is consistently in the OWASP Top 10 vulnerabilities. PDO prepared statements eliminate it completely when used correctly. Transactions ensure data integrity for any multi-step operation — together, these two patterns are non-negotiable for production PHP database work.

Code Example

<?php
$dsn = "mysql:host=localhost;dbname=myapp;charset=utf8mb4";
$pdo = new PDO($dsn, "root", "password", [
    PDO::ATTR_ERRMODE            => PDO::ERRMODE_EXCEPTION,
    PDO::ATTR_DEFAULT_FETCH_MODE => PDO::FETCH_ASSOC,
]);

// INSERT with prepared statement
$stmt = $pdo->prepare("INSERT INTO users (name, email) VALUES (:name, :email)");
$stmt->execute(["name" => "Alice", "email" => "alice@example.com"]);

// SELECT
$stmt = $pdo->prepare("SELECT * FROM users WHERE id = :id");
$stmt->execute(["id" => 1]);
$user = $stmt->fetch();

// fetchAll
$stmt = $pdo->prepare("SELECT id, name, email FROM users WHERE active = 1 ORDER BY name");
$stmt->execute();
$users = $stmt->fetchAll();
foreach ($users as $user) {
    echo "{$user['id']}: {$user['name']} ({$user['email']})<br>";
}

// Transactions
try {
    $pdo->beginTransaction();
    $pdo->prepare("UPDATE accounts SET balance = balance - 100 WHERE id = 1")->execute();
    $pdo->prepare("UPDATE accounts SET balance = balance + 100 WHERE id = 2")->execute();
    $pdo->commit();
} catch (PDOException $e) {
    $pdo->rollBack();
    echo "Transfer failed: " . $e->getMessage();
}

Week 13 — Composer, Namespaces & Modern PHP Tooling

Using Composer for dependency management, organising code with namespaces, and leveraging the PSR standards.

Core Focus

Using Composer for dependency management, organising code with namespaces, and leveraging the PSR standards.

Topics & Subtopics

  1. Composer & composer.json — Composer is PHP's dependency manager — it lets you declare, install, and autoload third-party libraries. Every modern PHP project starts with composer init and manages packages through composer.json and the lockfile composer.lock.
  2. Namespaces & PSR-4 Autoloading — Namespaces prevent class name collisions between your code and third-party libraries. PSR-4 autoloading (configured in composer.json) automatically loads class files based on their namespace — eliminating every require statement.
  3. PHP 8.x Features: Fibers, First-Class Callables & More — PHP 8.0–8.3 introduced a wave of powerful features: Named Arguments, Match Expressions, Nullsafe Operator (?->), Fibers (lightweight concurrency), Union/Intersection types, and readonly properties. These modernise PHP dramatically.
  4. PHP Attributes (Annotations) — PHP 8 Attributes (#[AttributeName]) replace docblock annotations with a first-class, validated syntax. Frameworks like Symfony and Laravel use attributes for routing, validation, and dependency injection configuration.

Why This Week Matters: Composer transformed PHP from "copy random scripts into your project" to a modern, dependency-managed ecosystem. PSR-4 autoloading and namespaces are the foundation every modern framework (Laravel, Symfony, Slim) is built on — there's no going back.

Week 14 — Design Patterns & Architecture

Applying the most important software design patterns used in professional PHP frameworks and applications.

Core Focus

Applying the most important software design patterns used in professional PHP frameworks and applications.

Topics & Subtopics

  1. Repository Pattern — The Repository pattern decouples your business logic from your data storage layer. Your application code talks to a repository interface — the concrete implementation (MySQL, JSON, memory) can be swapped without touching business logic.
  2. Dependency Injection & Service Container — Dependency Injection (DI) means passing a class's dependencies in from the outside rather than creating them internally. A DI Container automates this wiring — it's the heart of frameworks like Laravel and Symfony.
  3. Singleton & Factory Patterns — The Singleton ensures only one instance of a class exists (useful for a DB connection). The Factory pattern centralises object creation, letting you swap out which concrete class is returned based on configuration or context.
  4. Middleware Pattern — Middleware is a chain of handlers that process an HTTP request before it reaches the final controller. Each middleware can inspect, modify, or short-circuit the request — used for auth, logging, rate limiting, and CORS in frameworks like Slim and Laravel.

Why This Week Matters: The Repository pattern and Dependency Injection are the two architectural decisions that separate testable, maintainable PHP from spaghetti code. Every major framework (Laravel, Symfony, Slim) is built on these patterns — understanding them is essential for working in any modern PHP codebase.

Code Example

<?php
// Repository Pattern
interface UserRepository {
    public function findById(int $id): ?array;
    public function save(array $user): void;
}

class MySQLUserRepository implements UserRepository {
    public function __construct(private PDO $pdo) {}
    public function findById(int $id): ?array {
        $s = $this->pdo->prepare("SELECT * FROM users WHERE id = ?");
        $s->execute([$id]);
        return $s->fetch() ?: null;
    }
    public function save(array $user): void { /* ... */ }
}

// Dependency Injection
class OrderService {
    public function __construct(
        private UserRepository  $users,
        private PaymentGateway  $payment,
    ) {}

    public function placeOrder(int $userId, float $amount): bool {
        $user = $this->users->findById($userId);
        return $user ? $this->payment->charge($amount) : false;
    }
}

// Factory Pattern
class PaymentGatewayFactory {
    public static function create(string $provider): PaymentGateway {
        return match($provider) {
            "stripe" => new StripeGateway(),
            "paypal" => new PayPalGateway(),
            default  => throw new \InvalidArgumentException("Unknown: $provider"),
        };
    }
}

// Middleware Pattern
interface Middleware {
    public function handle(array $request, callable $next): array;
}
class AuthMiddleware implements Middleware {
    public function handle(array $request, callable $next): array {
        if (empty($request["token"])) return ["status" => 401, "body" => "Unauthorized"];
        return $next($request);
    }
}

Week 15 — API Development, Security & Performance

Building RESTful JSON APIs, hardening PHP applications against common attacks, and optimizing performance.

Core Focus

Building RESTful JSON APIs, hardening PHP applications against common attacks, and optimizing performance.

Topics & Subtopics

  1. Building a REST API in PHP — A REST API returns JSON responses to HTTP requests. In plain PHP, you parse the request method/path, dispatch to the appropriate handler, and send back a proper Content-Type: application/json header with a status code.
  2. PHP Security Fundamentals — Web security is non-negotiable. The four most critical attack vectors in PHP are: SQL Injection (prevent with PDO prepared statements), XSS (prevent with htmlspecialchars()), CSRF (prevent with tokens), and Session Hijacking (prevent with session_regenerate_id()).
  3. Password Hashing & Secure Authentication — PHP's password_hash() and password_verify() functions implement bcrypt hashing, the industry standard for storing passwords. Never use MD5, SHA1, or plain text — always use these built-in functions.
  4. Caching with APCu & OPcache — OPcache compiles PHP files to bytecode and caches them in memory, eliminating repeated parsing — it's the single biggest performance win for any PHP app. APCu adds a user-level key-value cache for storing computed results.

Why This Week Matters: Security and performance are what separate hobby PHP projects from production-grade applications. password_hash() + PDO prepared statements + OPcache + caching are four lines of effort that prevent every common attack and 10x your app's performance.

Code Example

<?php
// REST API
header("Content-Type: application/json");
header("Access-Control-Allow-Origin: *");

$method = $_SERVER["REQUEST_METHOD"];
$path   = parse_url($_SERVER["REQUEST_URI"], PHP_URL_PATH);

$response = match([$method, $path]) {
    ["GET", "/api/users"]  => ["users" => [["id" => 1, "name" => "Alice"]]],
    default                => ["error" => "Not Found"],
};
echo json_encode($response);

// XSS Prevention
$userInput = "<script>alert('XSS')</script>";
echo htmlspecialchars($userInput, ENT_QUOTES, "UTF-8");

// CSRF Token
if (empty($_SESSION["csrf_token"])) {
    $_SESSION["csrf_token"] = bin2hex(random_bytes(32));
}

// Password Hashing
$password = "my_super_secret_password";
$hash     = password_hash($password, PASSWORD_DEFAULT);
$isValid  = password_verify("my_super_secret_password", $hash);

// APCu Caching
$cacheKey = "all_products";
$products = apcu_fetch($cacheKey, $success);
if (!$success) {
    $products = $pdo->query("SELECT * FROM products")->fetchAll();
    apcu_store($cacheKey, $products, 300); // Cache for 300 seconds
}
echo json_encode($products);
