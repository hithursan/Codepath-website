About This Curriculum

This is a 16-week, phase-based React curriculum designed to take a complete beginner to a professional-level React developer. The roadmap moves from JSX, props, state, and forms through hooks, data fetching, routing, and global state management, into performance optimization, testing, and TypeScript, all the way to advanced patterns, Next.js full-stack development, CI/CD, and three complete capstone projects.

The curriculum is organized into four phases: Phase 1 covers absolute basics and core React concepts, Phase 2 covers hooks, side effects, data fetching, and routing, Phase 3 covers intermediate and advanced core topics, and Phase 4 covers master-level patterns, architecture, and full-stack development.

By Week 16, you will have built three complete, portfolio-ready applications and be confident building production-grade React and Next.js applications from scratch.

Level: Beginner → Master  |  Duration: 16 weeks  |  Stack: React 18+, Vite, Node.js  |  Prerequisites: JavaScript ES6+ basics

Week 1 — Setting Up & Your First React App

Understand what React is, how it works in the browser, and write your very first component.

Core Focus

Understanding what React is, how it works in the browser, and writing your very first component.

Topics & Subtopics

  1. What is React & Why Use It? — React is a JavaScript library built by Meta for creating fast, interactive UIs using reusable components. It uses a Virtual DOM to efficiently update only the parts of the page that change.
  2. Setting Up a React Project with Vite — Vite is a modern, fast build tool that replaces the older Create React App. Running npm create vite@latest scaffolds a ready-to-use React project in seconds with hot module replacement.
  3. JSX — JavaScript + HTML Syntax — JSX is a syntax extension that lets you write HTML-like code directly inside JavaScript files. Babel compiles JSX down to React.createElement() calls before the browser sees it.
  4. Your First Functional Component — A React component is simply a JavaScript function that returns JSX. Components must start with a capital letter so React can tell them apart from regular HTML tags.

Why This Week Matters: Everything in React is a component. Understanding the mental model of "UI as a function of data" from Week 1 makes every subsequent concept — props, state, hooks — feel logical rather than arbitrary.

Code Example

// index.jsx — entry point
import ReactDOM from 'react-dom/client';
import App from './App';
ReactDOM.createRoot(document.getElementById('root')).render(<App />);

// A component is just a function that returns JSX
function WelcomeCard() {
    return (
        <div>
            <h2>My First Component</h2>
            <p>React components are reusable building blocks.</p>
        </div>
    );
}
export default WelcomeCard;

Week 2 — Props, Component Composition & Lists

Pass data into components using props and render dynamic lists from arrays.

Core Focus

Passing data into components using props and rendering dynamic lists.

Topics & Subtopics

  1. Props — Passing Data to Components — Props are how you pass data from a parent component down to a child. They make components dynamic and reusable — like function arguments for your UI.
  2. Destructuring Props — Instead of referencing props.name repeatedly, you can destructure props directly in the function signature for cleaner, more readable code — the standard pattern used by professional React developers.
  3. Rendering Lists with .map() — React uses JavaScript's .map() array method to transform data into JSX elements. Every item rendered in a list must have a unique key prop to help React track items efficiently.
  4. Component Composition & children Prop — The special children prop lets a parent component render whatever is placed between its opening and closing tags, enabling reusable wrapper components.

Why This Week Matters: Props and composition are the fundamental mechanism for building React UIs from small, reusable pieces. Almost every real React application is built by composing many simple components together.

Code Example

// Destructuring props in the parameter list
function ProductCard({ title, price, inStock }) {
    return (
        <div>
            <h3>{title}</h3>
            <p>${price} — {inStock ? "In Stock" : "Sold Out"}</p>
        </div>
    );
}

// Rendering a list with .map() and key prop
const fruits = ["Apple", "Banana", "Cherry"];
function FruitList() {
    return (
        <ul>
            {fruits.map((fruit) => (
                <li key={fruit}>{fruit}</li>
            ))}
        </ul>
    );
}

// children prop for wrapper components
function Card({ children }) {
    return <div style={{ border: "1px solid #ccc", padding: "1rem" }}>{children}</div>;
}

Week 3 — State & Event Handling

Make components interactive using useState and respond to user events.

Core Focus

Making components interactive using useState and responding to user events.

Topics & Subtopics

  1. The useState Hook — useState adds reactive data to a component. When state changes, React automatically re-renders the component with the new value — you never touch the DOM directly.
  2. Handling Events (onClick, onChange) — React handles events using camelCase props like onClick and onChange. You pass a function reference — not a function call — to the event prop.
  3. Conditional Rendering — React uses plain JavaScript for conditional UI: the ternary operator ? : for inline choices, or the && short-circuit operator to render something only when a condition is true.
  4. Updating State Based on Previous State — When new state depends on old state, always use the functional update form of setState. This ensures you always get the latest value and avoids subtle bugs in rapid updates.

Why This Week Matters: State is what makes React applications interactive. Understanding the functional update form and the distinction between props (external data) and state (internal data) is essential before moving to more complex hooks.

Code Example

import { useState } from 'react';

function Counter() {
    const [count, setCount] = useState(0);
    // Always use functional update when new state depends on old state
    const increment = () => setCount((prev) => prev + 1);
    return <button onClick={increment}>Count: {count}</button>;
}

function AuthStatus({ isLoggedIn }) {
    return (
        <div>
            {isLoggedIn ? <p>Welcome back!</p> : <p>Please log in.</p>}
            {isLoggedIn && <button>Log Out</button>}
        </div>
    );
}

Week 4 — Forms, Controlled Components & Lifting State

Build forms the React way and share state between sibling components.

Core Focus

Building forms the React way and sharing state between sibling components.

Topics & Subtopics

  1. Controlled Components & Forms — In React, form inputs are "controlled" by tying their value to a state variable. React is always the single source of truth for the form's data, making validation and submission straightforward.
  2. Handling Form Submission — Attach an onSubmit handler to <form> and call e.preventDefault() to stop the default page-reload behavior. Then you have full control over what happens with the form data.
  3. Lifting State Up — When two sibling components need to share data, move that state up to their nearest common parent. The parent holds the state and passes it down via props — a core React data-flow pattern.
  4. Multi-field Form with a Single State Object — Use a single state object for all form fields instead of separate useState calls. Use the spread operator to merge updates without overwriting other fields.

Why This Week Matters: Forms are in almost every real application — login, signup, checkout, settings. Controlled components and lifting state are the patterns that make React data flow predictable and debuggable.

Code Example

import { useState } from 'react';

// Multi-field form with single state object
function SignupForm() {
    const [form, setForm] = useState({ username: "", password: "" });
    const handleChange = (e) => setForm({ ...form, [e.target.name]: e.target.value });
    const handleSubmit = (e) => { e.preventDefault(); console.log(form); };
    return (
        <form onSubmit={handleSubmit}>
            <input name="username" value={form.username} onChange={handleChange} />
            <input name="password" type="password" value={form.password} onChange={handleChange} />
            <button type="submit">Sign Up</button>
        </form>
    );
}

// Lifting state up
function Parent() {
    const [text, setText] = useState("");
    return (
        <>
            <InputChild onType={setText} />
            <DisplayChild text={text} />
        </>
    );
}

Week 5 — The useEffect Hook & Side Effects

Run code in response to component lifecycle events like mounting and state changes.

Core Focus

Running code in response to component lifecycle events like mounting and state changes.

Topics & Subtopics

  1. What is a Side Effect? — A side effect is anything that reaches outside the React component — fetching data, updating the document title, setting up timers, or subscribing to events. useEffect is the designated place for all side effects.
  2. The Dependency Array — The second argument to useEffect controls when the effect re-runs. Empty array [] means run once on mount; a list means re-run when any listed value changes; no array means run after every render.
  3. Cleanup Functions in useEffect — When your effect sets up something persistent like an event listener or interval, return a cleanup function from useEffect. React calls this when the component unmounts or before the effect runs again.
  4. Fetching Data with useEffect — The most common use of useEffect is fetching data from an API when a component first loads. Always handle loading and error states for a good user experience.

Why This Week Matters: Nearly every real React app fetches data. Understanding the dependency array and cleanup functions prevents two of the most common React bugs: infinite re-render loops and memory leaks from uncleared timers and event listeners.

Code Example

import { useEffect, useState } from 'react';

function Timer() {
    const [seconds, setSeconds] = useState(0);
    useEffect(() => {
        const interval = setInterval(() => setSeconds((s) => s + 1), 1000);
        return () => clearInterval(interval); // Cleanup prevents memory leaks
    }, []); // [] = run once on mount only
    return <p>Elapsed: {seconds}s</p>;
}

function UserFetcher() {
    const [user, setUser] = useState(null);
    useEffect(() => {
        fetch("https://jsonplaceholder.typicode.com/users/1")
            .then((res) => res.json())
            .then((data) => setUser(data));
    }, []);
    return <div>{user ? <p>{user.name}</p> : <p>Loading...</p>}</div>;
}

Week 6 — Advanced Hooks: useRef, useCallback, useMemo

Optimize performance and access DOM nodes directly without triggering re-renders.

Core Focus

Performance optimization and accessing DOM nodes directly without triggering re-renders.

Topics & Subtopics

  1. useRef — Persistent Values & DOM Access — useRef returns a mutable object whose .current property persists across re-renders without causing a re-render when changed. Most commonly used to get a direct reference to a DOM element.
  2. useMemo — Memoizing Expensive Calculations — useMemo caches the result of an expensive computation and only recalculates it when its dependencies change. Use it when a calculation is slow and does not need to run on every render.
  3. useCallback — Memoizing Functions — useCallback returns a memoized version of a callback function that only changes if its dependencies change. It prevents child components from re-rendering unnecessarily when a parent re-renders.
  4. useRef for Tracking Previous Values — Beyond DOM access, useRef can store any value you want to persist between renders. A common pattern is using it to track the previous value of a prop or state variable.

Why This Week Matters: useMemo and useCallback are the most misunderstood hooks — beginners either never use them or use them everywhere unnecessarily. Knowing when each hook is actually needed is a mark of a senior React developer.

Code Example

import { useRef, useMemo, useCallback, useState } from 'react';

function FocusInput() {
    const inputRef = useRef(null);
    return (
        <>
            <input ref={inputRef} placeholder="Click button to focus" />
            <button onClick={() => inputRef.current.focus()}>Focus</button>
        </>
    );
}

function ExpensiveList({ items }) {
    const filtered = useMemo(
        () => items.filter((item) => item.active),
        [items]
    );
    return <ul>{filtered.map((i) => <li key={i.id}>{i.name}</li>)}</ul>;
}

function Parent() {
    const [count, setCount] = useState(0);
    const handleClick = useCallback(() => setCount((c) => c + 1), []);
    return <ChildButton onClick={handleClick} />;
}

Week 7 — Custom Hooks & Context API

Extract reusable logic into custom hooks and share global state without prop drilling.

Core Focus

Extracting reusable logic into custom hooks and sharing global state without prop drilling.

Topics & Subtopics

  1. Building Custom Hooks — A custom hook is a JavaScript function whose name starts with use that calls other hooks internally. They let you extract and reuse stateful logic across multiple components, eliminating code duplication.
  2. The Context API — createContext & useContext — The Context API solves prop drilling — passing props through many layers of components. You create a Context, wrap your tree in a Provider, and any descendant can read the value directly.
  3. Creating a Context Provider Component — The standard pattern wraps createContext and a Provider component in a single file, keeping all context logic in one place and making the component tree clean and readable.
  4. useLocalStorage Custom Hook Pattern — A useLocalStorage custom hook syncs state with the browser's Local Storage automatically — a great example of combining standard hooks with browser APIs into a clean, reusable abstraction.

Why This Week Matters: Custom hooks are the most important tool for code reuse in React. Context is the built-in solution for shared state. Together they eliminate prop drilling and duplicated logic from any size application.

Code Example

import { useState, useEffect, createContext, useContext } from 'react';

// Custom useFetch hook
function useFetch(url) {
    const [data, setData] = useState(null);
    const [loading, setLoading] = useState(true);
    useEffect(() => {
        fetch(url).then((r) => r.json()).then((d) => { setData(d); setLoading(false); });
    }, [url]);
    return { data, loading };
}

// Auth context pattern
const AuthContext = createContext(null);
export function AuthProvider({ children }) {
    const [user, setUser] = useState(null);
    return <AuthContext.Provider value={{ user, setUser }}>{children}</AuthContext.Provider>;
}
export const useAuth = () => useContext(AuthContext);

// useLocalStorage custom hook
function useLocalStorage(key, initial) {
    const [value, setValue] = useState(() => {
        const stored = localStorage.getItem(key);
        return stored ? JSON.parse(stored) : initial;
    });
    const set = (v) => { setValue(v); localStorage.setItem(key, JSON.stringify(v)); };
    return [value, set];
}

Week 8 — React Router & Client-Side Navigation

Build multi-page single-page applications with React Router v6.

Core Focus

Building multi-page single-page applications with React Router v6.

Topics & Subtopics

  1. Setting Up React Router v6 — React Router v6 is the standard library for client-side navigation. You wrap your app in <BrowserRouter>, define routes with <Routes> and <Route>, and React renders the matching component without reloading.
  2. <Link> & <NavLink> for Navigation — Never use <a href> for internal navigation — it causes a full page reload. Use React Router's <Link> instead. <NavLink> automatically adds an active class to the currently active link.
  3. URL Parameters with useParams — React Router lets you define dynamic URL segments using a colon (:paramName). The useParams hook reads those segments in the rendered component, perfect for detail pages.
  4. Programmatic Navigation with useNavigate — The useNavigate hook gives you a function to navigate programmatically — for example, redirecting a user to a dashboard after a successful login.

Why This Week Matters: Almost every real React application has multiple pages. React Router is the universal standard for this, and URL parameters are used in virtually every app that displays individual items or user profiles.

Code Example

import { BrowserRouter, Routes, Route, Link, NavLink, useParams, useNavigate } from 'react-router-dom';

function App() {
    return (
        <BrowserRouter>
            <Routes>
                <Route path="/" element={<Home />} />
                <Route path="/user/:id" element={<UserProfile />} />
            </Routes>
        </BrowserRouter>
    );
}

function UserProfile() {
    const { id } = useParams();
    return <h1>Viewing profile for user #{id}</h1>;
}

function LoginButton() {
    const navigate = useNavigate();
    return <button onClick={() => navigate("/dashboard")}>Log In</button>;
}

Week 9 — Global State Management with Redux Toolkit

Manage complex, application-wide state with Redux Toolkit — the modern standard.

Core Focus

Managing complex, application-wide state with Redux Toolkit (RTK), the modern standard.

Topics & Subtopics

  1. The Redux Data Flow (Store, Slice, Dispatch) — Redux enforces strict unidirectional data flow: the store holds all global state, actions describe what happened, reducers specify how state changes, and dispatch sends actions to the store.
  2. Creating a Slice with createSlice — createSlice automatically generates action creators and action types from your reducer functions, eliminating massive amounts of boilerplate code.
  3. Reading State with useSelector — useSelector is the React-Redux hook for reading data from the store. It subscribes to the store and automatically re-renders the component when the selected state changes.
  4. Async Actions with createAsyncThunk — createAsyncThunk handles async operations like API calls in Redux, automatically dispatching pending, fulfilled, and rejected action types for loading/success/error states.

Why This Week Matters: Once an app has more than a few pieces of shared state, Context alone becomes unwieldy. Redux Toolkit is the industry standard for predictable global state and is used in most large-scale React applications.

Code Example

import { configureStore, createSlice, createAsyncThunk } from '@reduxjs/toolkit';
import { useSelector, useDispatch } from 'react-redux';

const counterSlice = createSlice({
    name: 'counter',
    initialState: { value: 0 },
    reducers: {
        increment: (state) => { state.value += 1; },
        decrement: (state) => { state.value -= 1; },
    },
});
export const { increment, decrement } = counterSlice.actions;

export const store = configureStore({
    reducer: { counter: counterSlice.reducer },
});

function CounterDisplay() {
    const count = useSelector((state) => state.counter.value);
    const dispatch = useDispatch();
    return <button onClick={() => dispatch(increment())}>Count: {count}</button>;
}

export const fetchUser = createAsyncThunk("user/fetch", async (id) => {
    const res = await fetch(`https://jsonplaceholder.typicode.com/users/${id}`);
    return res.json();
});

Week 10 — Performance Optimization & Code Splitting

Make React apps fast at scale with memoization, lazy loading, and profiling.

Core Focus

Making React apps fast at scale with memoization, lazy loading, and profiling.

Topics & Subtopics

  1. React.memo — Preventing Unnecessary Re-renders — React.memo wraps a component and prevents it from re-rendering if its props have not changed. A simple win for components rendered many times in a list.
  2. Lazy Loading with React.lazy & Suspense — React.lazy and Suspense split your app into separate chunks that load on demand. Each page only loads when the user navigates to it, drastically improving initial load time.
  3. Code Splitting by Route — The most impactful place to apply lazy loading is on route components. Wrapping each page in React.lazy ensures users only download the JavaScript for the page they are currently visiting.
  4. Using the React DevTools Profiler — The React DevTools Profiler records your app's render activity, showing which components re-render, how long they take, and why — the essential tool for finding performance bottlenecks.

Why This Week Matters: A slow React app is a bad user experience. Route-based code splitting alone can cut initial JavaScript bundle size by 50-80% for large applications. Profiling before optimizing prevents wasted effort.

Code Example

import { memo, lazy, Suspense, Profiler } from 'react';
import { Routes, Route } from 'react-router-dom';

const LeaderboardItem = memo(function LeaderboardItem({ name, score }) {
    return <li>{name}: {score} pts</li>;
});

const Home      = lazy(() => import('./pages/Home'));
const Dashboard = lazy(() => import('./pages/Dashboard'));

function App() {
    return (
        <Suspense fallback={<div>Loading page...</div>}>
            <Routes>
                <Route path="/" element={<Home />} />
                <Route path="/dash" element={<Dashboard />} />
            </Routes>
        </Suspense>
    );
}

function onRender(id, phase, actualDuration) {
    console.log(`${id} took ${actualDuration.toFixed(2)}ms (${phase})`);
}

Week 11 — Testing React with Vitest & React Testing Library

Write unit and integration tests for React components with industry-standard tools.

Core Focus

Writing unit and integration tests for React components with industry-standard tools.

Topics & Subtopics

  1. Testing Philosophy: RTL Queries — React Testing Library encourages testing components the way a user would interact with them — by querying elements through accessible roles and text, not by implementation details like class names or state variables.
  2. Simulating User Interactions with userEvent — @testing-library/user-event simulates real browser interactions like typing and clicking. This is more accurate than fireEvent because it mirrors actual user behavior.
  3. Testing Async Components (API Calls) — When testing components that fetch data, mock the API call and wait for the async result with RTL's findBy queries, which automatically wait for elements to appear.
  4. Testing Custom Hooks with renderHook — RTL's renderHook utility lets you test custom hooks in isolation without embedding them in a real component — perfect for testing useFetch, useLocalStorage, and other reusable hooks.

Why This Week Matters: Tests prevent regressions — breaking something that used to work. React Testing Library's "test from the user's perspective" philosophy produces tests that give you confidence your app actually works, not just that your implementation details haven't changed.

Code Example

import { render, screen } from '@testing-library/react';
import userEvent from '@testing-library/user-event';
import { renderHook, act } from '@testing-library/react';

test('renders the welcome message', () => {
    render(<WelcomeMessage name="Alice" />);
    expect(screen.getByText("Hello, Alice!")).toBeInTheDocument();
});

test('increments count on button click', async () => {
    render(<Counter />);
    const button = screen.getByRole('button', { name: /increment/i });
    await userEvent.click(button);
    expect(screen.getByText("Count: 1")).toBeInTheDocument();
});

test('useCounter increments correctly', () => {
    const { result } = renderHook(() => useCounter());
    expect(result.current.count).toBe(0);
    act(() => result.current.increment());
    expect(result.current.count).toBe(1);
});

Week 12 — TypeScript with React

Add static types to React components, props, and hooks for safer, more maintainable code.

Core Focus

Adding static types to React components, props, and hooks for safer, more maintainable code.

Topics & Subtopics

  1. Typing Component Props with Interfaces — TypeScript interfaces define the shape of your component's props, providing autocomplete and compile-time errors when you pass the wrong data type — eliminating an entire class of runtime bugs.
  2. Typing useState & useRef — TypeScript can often infer types from initial values, but for complex types like objects or null-initialized values, provide the generic type parameter explicitly to avoid the type becoming too broad.
  3. Typing Event Handlers — React's event system has specific TypeScript types for each kind of event. Using the correct event type like React.ChangeEvent<HTMLInputElement> gives full type safety and autocomplete on the event object.
  4. Generic Components — TypeScript generics let you create components that work with any data type while remaining fully type-safe. The pattern for building reusable list, table, and select components in typed React apps.

Why This Week Matters: TypeScript has become the standard for large React codebases. Typed props, typed event handlers, and generic components catch bugs at compile time that would only surface as runtime errors in production.

Week 13 — Advanced Patterns: Compound Components, Render Props & HOCs

Master advanced React design patterns used in production-level component libraries.

Core Focus

Mastering advanced React design patterns used in production-level component libraries.

Topics & Subtopics

  1. Compound Component Pattern — Lets you build flexible, expressive components that share implicit state internally while giving the consumer full control over layout. Libraries like Radix UI and Reach UI are built on this pattern.
  2. Render Props Pattern — Passing a function as a prop that a component calls to determine what to render. This was the primary way to share stateful logic before hooks and is still used in libraries like Formik and React Final Form.
  3. Higher-Order Components (HOCs) — A function that takes a component and returns a new, enhanced component. HOCs add cross-cutting concerns like auth checks, logging, or analytics without modifying the original component.
  4. The useReducer Hook for Complex State — The built-in React alternative to useState for complex state logic. It works like Redux — you dispatch actions and a pure reducer computes the next state — making state transitions predictable and testable.

Why This Week Matters: These patterns appear throughout major React component libraries. Understanding Compound Components is essential for using (and building) headless UI libraries like Radix UI. useReducer is the go-to for state machines and complex form logic.

Code Example

// Compound Component Pattern
const AccordionContext = React.createContext();
function Accordion({ children }) {
    const [open, setOpen] = React.useState(null);
    return <AccordionContext.Provider value={{ open, setOpen }}>{children}</AccordionContext.Provider>;
}
Accordion.Item = function Item({ id, title, children }) {
    const { open, setOpen } = React.useContext(AccordionContext);
    return <div><button onClick={() => setOpen(id)}>{title}</button>{open === id && children}</div>;
};

// useReducer for complex state
function reducer(state, action) {
    switch (action.type) {
        case "increment": return { count: state.count + 1 };
        case "reset":     return { count: 0 };
        default: throw new Error("Unknown action");
    }
}
function Counter() {
    const [state, dispatch] = React.useReducer(reducer, { count: 0 });
    return <button onClick={() => dispatch({ type: "increment" })}>Count: {state.count}</button>;
}

Week 14 — Full-Stack Integration: Next.js, APIs & Authentication

Move beyond client-only React into full-stack development with Next.js App Router.

Core Focus

Moving beyond client-only React into full-stack development with Next.js App Router.

Topics & Subtopics

  1. Next.js App Router & Server Components — Next.js 14+ introduces React Server Components (RSCs) — components that run only on the server and can directly access databases and APIs without any client-side JavaScript.
  2. Client vs Server Components — All components are Server Components by default in the App Router. You opt into client-side rendering with the "use client" directive — and only when you need interactivity, browser APIs, or React hooks.
  3. API Routes in Next.js — Next.js lets you build backend API endpoints inside the same project using Route Handlers, eliminating the need for a separate Express server for many use cases.
  4. Authentication with NextAuth.js — NextAuth.js (Auth.js v5) provides drop-in authentication for Next.js apps, supporting OAuth providers (GitHub, Google), email/password, and database sessions.

Why This Week Matters: React Server Components fundamentally change what is possible in React. Data fetching becomes as simple as async/await with zero client-side JavaScript, making apps faster and simpler than anything previously possible.

Code Example

// app/users/page.tsx — Server Component (runs on server only)
async function UsersPage() {
    const users = await fetch("https://jsonplaceholder.typicode.com/users").then(r => r.json());
    return <ul>{users.map((u) => <li key={u.id}>{u.name}</li>)}</ul>;
}

// Client Component — opt in with "use client"
"use client";
import { useState } from 'react';
function LikeButton({ initialCount }) {
    const [likes, setLikes] = useState(initialCount);
    return <button onClick={() => setLikes(l => l + 1)}>❤️ {likes}</button>;
}

// app/api/users/route.ts — API Route Handler
import { NextResponse } from 'next/server';
export async function GET() {
    const users = [{ id: 1, name: "Alice" }, { id: 2, name: "Bob" }];
    return NextResponse.json(users);
}

Week 15 — Real-World Architecture, Monorepos & CI/CD

Structure large React codebases, manage complexity, and automate deployments.

Core Focus

Structuring large React codebases, managing complexity, and automating deployments.

Topics & Subtopics

  1. Feature-Based Folder Structure — Organizing files by type (/components, /hooks) breaks down as apps grow. The industry standard for large apps is organizing by feature/domain, keeping all related files co-located and reducing cross-feature coupling.
  2. Absolute Imports & Path Aliases — Configuring path aliases turns unreadable relative imports into clean, absolute ones like '@/components/Button'. Configured in tsconfig.json and vite.config.ts.
  3. Environment Variables & .env Files — React apps need API keys and configuration that differs between development and production. Vite exposes env variables prefixed with VITE_ to your client-side code via import.meta.env.
  4. GitHub Actions CI/CD Pipeline — A CI/CD pipeline automatically runs tests and deploys your app on every push. This GitHub Actions workflow runs tests on pull requests and deploys on every push to main.

Why This Week Matters: A well-structured codebase scales. A poorly structured one becomes unmaintainable. CI/CD pipelines catch broken builds before they reach users. These practices separate hobby projects from production-grade software.
