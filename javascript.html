About This Curriculum

This is a 24-week, phase-based JavaScript curriculum designed to take a complete beginner to a professional-level JavaScript developer. The roadmap moves from core syntax, runtime, and types through control flow, functions, data structures, OOP, asynchronous programming, and Node.js, all the way to generators, design patterns, performance optimization, and three complete capstone projects.

The curriculum is organized into four phases: Phase 1 covers absolute basics and JavaScript's dynamic type system, Phase 2 covers control flow, functions, and modern data structures, Phase 3 covers object-oriented design, error handling, async programming, and the Node.js ecosystem, and Phase 4 takes you into generators, advanced async patterns, design patterns, performance, and production-level projects.

By Week 24, you will have built three complete, portfolio-ready applications and be confident writing modern JavaScript for both the browser and Node.js environments.

Level: Beginner → Master  |  Duration: 24 weeks  |  Study Time: ~5–8 hours/week  |  Environments: Browser + Node.js

Phase 1 — Absolute Basics

Weeks 1–5: Build a rock-solid foundation in JavaScript syntax, the runtime, and data types.

Week 1 — JavaScript Setup & Your First Script

Understand where JavaScript runs and write your very first program.

Core Focus

Understanding where JavaScript runs and writing your first program.

Topics & Subtopics

  1. The JavaScript Runtime (Browser vs Node.js) — JavaScript runs inside a browser engine like V8 in Chrome, or standalone via Node.js on a server. Both share the core language but differ in available built-ins like window or process.
  2. Linking Scripts & Node Execution — In the browser, JS is loaded via a <script> tag; in Node, you run a .js file with the node command. Understanding both contexts is essential before diving into the language itself.
  3. console.log() & Developer Tools — console.log() prints values to the console and is the most basic debugging tool in JavaScript. Browser DevTools and the Node terminal both display this output.
  4. Comments & Code Structure — Single-line (//) and multi-line (/* */) comments document code without being executed. JavaScript files are processed top-to-bottom unless functions or modules alter that flow.

Why This Week Matters: Before writing any logic, you need to understand where JavaScript runs, how it is executed, and how to see output. The browser/Node distinction shapes everything from available APIs to how you debug problems.

Code Example

// hello.js - Your first JavaScript program
console.log("Hello, JavaScript World!");

/* Multi-line comment:
   This script runs top-to-bottom in Node or the browser. */
let today = "Setup Day";
console.log("Today is:", today);

🛠 Practical Task

Install Node.js, write a hello.js file that prints your name and a welcome message, then run it using node hello.js from the terminal.

Week 2 — Variables & Primitive Data Types

Store data using JavaScript's dynamic and flexible type system.

Core Focus

Storing data using JavaScript's dynamic type system.

Topics & Subtopics

  1. let, const, and var — let and const are modern, block-scoped declarations, while var is the older function-scoped style to avoid in new code. const prevents reassignment, making intent clearer.
  2. Primitive Types — JavaScript has primitives including number, string, boolean, null, undefined, and bigint. Unlike Java, JS does not require explicit type declarations.
  3. Dynamic Typing — A variable's type can change at runtime because JavaScript does not enforce types at declaration. This flexibility requires extra discipline to avoid type-related bugs.
  4. typeof Operator — typeof returns a string describing a value's current type, useful for debugging and runtime validation.

Why This Week Matters: JavaScript's dynamic type system is both its greatest strength and a common source of bugs. Understanding how types work from the beginning saves countless hours of debugging later.

Code Example

let age = 25;               // number
const name = "Alice";       // string, cannot be reassigned
let isActive = true;        // boolean
let nothing = null;         // intentional empty value
console.log(typeof age, typeof name, typeof isActive);

🛠 Practical Task

Create a script that declares variables for a product (name, price, quantity, inStock) and logs a formatted summary using console.log().

Week 3 — Operators & Type Coercion

Manipulate data and understand JavaScript's flexible type conversions.

Core Focus

Manipulating data and understanding JavaScript's flexible type conversions.

Topics & Subtopics

  1. Arithmetic Operators — +, -, *, /, %, and ** perform mathematical operations. JavaScript numbers are always floating-point internally, even for whole numbers.
  2. Comparison Operators: == vs === — == compares values after type coercion, while === compares both value and type without conversion. Using === is always the recommended default.
  3. Logical Operators & Truthy/Falsy Values — &&, ||, and ! combine boolean logic, and JS treats values like 0, "", and null as falsy. Understanding truthiness is critical for correct conditionals.
  4. Type Coercion — JavaScript automatically converts types in many operations. Explicit conversion with Number() or String() avoids relying on these implicit rules.

Why This Week Matters: JavaScript's type coercion is one of its most misunderstood features. Knowing exactly when and how types are converted prevents entire categories of bugs.

Code Example

console.log(5 + "5");       // "55" (coercion: number to string)
console.log(5 === "5");     // false (strict: different types)
console.log(5 == "5");      // true (loose: coerced first)
console.log(Boolean(0));    // false (falsy value)
console.log(Number("42"));  // 42 (explicit conversion)

🛠 Practical Task

Write a script that takes a temperature in Celsius (as a string) and converts it to Fahrenheit, demonstrating explicit type conversion with Number().

Week 4 — Strings & Template Literals

Work with and transform text data using JavaScript's powerful string tools.

Core Focus

Working with and transforming text data.

Topics & Subtopics

  1. String Methods — Methods like .length, .toUpperCase(), .slice(), and .includes() let you inspect and transform text, used constantly in real-world data processing.
  2. Template Literals — Backtick-delimited strings allow embedded expressions via ${} and multi-line text without concatenation. They are the modern standard for building dynamic strings.
  3. String Immutability — Strings in JavaScript cannot be changed in place. Methods always return new strings rather than modifying the original.
  4. String Searching & Splitting — Methods like .indexOf(), .split(), and .replace() locate or break apart text. These are foundational for parsing and formatting user input.

Why This Week Matters: String manipulation is one of the most common tasks in any JavaScript program. Template literals alone transform how readable and maintainable dynamic text output can be.

Code Example

const name = "javascript";
const upper = name.toUpperCase();
const greeting = `Hello, ${name}! You have ${5} new messages.`;
console.log(greeting);
console.log(name.split("").reverse().join(""));

Week 5 — Arrays & Basic Objects

Store ordered and structured collections of related data.

Core Focus

Storing collections of related data using arrays and object literals.

Topics & Subtopics

  1. Array Declaration & Indexing — Arrays hold ordered collections of values accessed via zero-based indices. They are dynamically resizable and can hold mixed types.
  2. Common Array Methods — Methods like .push(), .pop(), .length, and .indexOf() manipulate and inspect array contents. Mastering these prepares you for higher-order array methods later.
  3. Object Literals — Objects store key-value pairs using curly-brace syntax, representing structured real-world entities. Dot or bracket notation accesses properties.
  4. Iterating Arrays with for and for...of — A standard for loop provides full index control, while for...of simplifies iteration when only values are needed.

Why This Week Matters: Arrays and objects are the two most important data containers in JavaScript. Nearly every program you write will use both heavily, and they are the building blocks of more advanced data structures.

Code Example

const scores = [90, 85, 77];
scores.push(100);
for (const score of scores) {
    console.log(score);
}
const person = { name: "Alice", age: 30 };
console.log(person.name, person["age"]);

Phase 2 — Control Flow & Logic

Weeks 6–10: Learn to direct program execution and organize code into reusable, modular units.

Week 6 — Conditional Statements

Make decisions in code using JavaScript's conditional constructs.

Core Focus

Making decisions in code.

Topics & Subtopics

  1. if, else if, else — These statements branch execution based on a condition evaluated top to bottom. Only the first truthy branch executes its block.
  2. switch Statements — switch compares one value against multiple cases, serving as a cleaner alternative to long if-else chains. Each case typically ends with break.
  3. Ternary Operator — The condition ? valueIfTrue : valueIfFalse syntax compresses simple if-else logic into a single expression, commonly used for concise variable assignments.
  4. Optional Chaining & Nullish Coalescing — ?. safely accesses nested properties without throwing on null/undefined, while ?? provides a fallback only when a value is null or undefined.

Why This Week Matters: Decision-making is the foundation of all program logic. Optional chaining and nullish coalescing are modern JavaScript features that make defensive coding far cleaner and more readable.

Code Example

const age = 20;
const status = age >= 18 ? "Adult" : "Minor";
console.log(status);
const user = { profile: null };
console.log(user.profile?.bio ?? "No bio set");

🛠 Practical Task

Write a grading script that converts a numeric score (0–100) into a letter grade using if-else, then rewrite the same logic using a switch statement.

Week 7 — Loops & Iteration

Repeat actions efficiently over data using JavaScript's loop constructs.

Core Focus

Repeating actions efficiently over data.

Topics & Subtopics

  1. for Loop — The classic loop combines initialization, condition, and increment for precise counting iterations. It remains the most flexible loop for index-based logic.
  2. while and do...while Loops — while checks its condition before each pass, while do...while always runs once first. Choose based on whether the first execution should be unconditional.
  3. break and continue — break exits a loop entirely, while continue skips to the next iteration, providing finer control beyond the basic condition.
  4. for...in vs for...of — for...in iterates over an object's enumerable keys, while for...of iterates over values in iterables like arrays. Confusing these is a common beginner mistake.

Why This Week Matters: Loops are how programs process collections and automate repetition. The distinction between for...in and for...of alone prevents many subtle bugs when working with objects and arrays.

Code Example

for (let i = 1; i <= 5; i++) {
    if (i === 3) continue;
    console.log(i);
}
const obj = { a: 1, b: 2 };
for (const key in obj) {
    console.log(key, obj[key]);
}

🛠 Practical Task

Write a script that prints a multiplication table (1–10) for a number set in a variable, then extend it to print a full 10x10 grid using nested loops.

Week 8 — Functions Deep Dive

Write reusable, organized blocks of logic using all of JavaScript's function forms.

Core Focus

Writing reusable, organized blocks of logic.

Topics & Subtopics

  1. Function Declarations & Expressions — Functions can be declared with the function keyword or assigned as expressions. Declarations are hoisted, while expressions are not, affecting when they can be called.
  2. Arrow Functions — Arrow functions offer concise syntax and do not bind their own this, inheriting it from the surrounding scope. They are the modern default for short, anonymous functions.
  3. Default & Rest Parameters — Default parameters provide fallback values when arguments are omitted. Rest parameters (...args) gather any number of extra arguments into an array.
  4. Higher-Order Functions — Functions that accept or return other functions are central to JavaScript's functional style and underlie array methods like map and filter.

Why This Week Matters: Functions are the fundamental unit of reusability in JavaScript. Arrow functions and higher-order functions are used everywhere in modern JS, from event handlers to React components.

Code Example

function add(a, b = 0) {
    return a + b;
}
const multiply = (a, b) => a * b;
function sumAll(...nums) {
    return nums.reduce((total, n) => total + n, 0);
}
console.log(add(5), multiply(2, 3), sumAll(1, 2, 3));

🛠 Practical Task

Write a calculate arrow function that accepts an operator string and two numbers, supporting add, subtract, multiply, and divide operations.

Week 9 — Arrays Deep Dive: map, filter, reduce

Process collections declaratively instead of with manual loops.

Core Focus

Processing collections declaratively instead of with manual loops.

Topics & Subtopics

  1. .map() — Transforms every array element into a new array based on a callback, without mutating the original. It is the go-to method for applying a transformation across an entire list.
  2. .filter() — Builds a new array containing only elements that pass a given test function, replacing manual loops that conditionally push to a results array.
  3. .reduce() — Collapses an array into a single accumulated value via a callback and initial value. It is the most powerful but often the trickiest array method to master.
  4. Method Chaining — Array methods can be chained together like .filter().map().reduce() to build readable, declarative data pipelines favored in modern JavaScript.

Why This Week Matters: These three methods are the backbone of modern JavaScript data processing. Once you internalize them, you will rarely need to write a manual loop for array transformations again.

Code Example

const nums = [1, 2, 3, 4, 5, 6];
const result = nums
    .filter(n => n % 2 === 0)
    .map(n => n * n)
    .reduce((sum, n) => sum + n);
console.log(result); // 56

🛠 Practical Task

Given an array of product objects with name and price properties, use .filter() and .map() to get names of products priced under $50, then use .reduce() to sum all prices.

Week 10 — Maps, Sets & Modules

Use specialized data structures and organize code across files with ES Modules.

Core Focus

Specialized data structures and code organization across files.

Topics & Subtopics

  1. Map — Stores key-value pairs where keys can be any type, maintains insertion order, and offers cleaner iteration than plain objects for dynamic key sets.
  2. Set — Stores only unique values, automatically discarding duplicates. Ideal for membership checks and deduplicating arrays.
  3. ES Modules (import/export) — Modules split code across files, explicitly exporting and importing only what is needed. This replaces older global-script patterns with clean dependency management.
  4. Destructuring — Array and object destructuring extract multiple values into variables in one concise statement, dramatically reducing boilerplate when working with structured data.

Why This Week Matters: ES Modules are the standard for organizing modern JavaScript projects. Map and Set solve real data problems that plain objects and arrays handle awkwardly.

Code Example

// mathUtils.js
export const square = (n) => n * n;

// main.js
import { square } from "./mathUtils.js";
const uniqueNums = new Set([1, 2, 2, 3]);
const { name, age } = { name: "Alice", age: 30 };
console.log(square(4), uniqueNums, name, age);

🛠 Practical Task

Split a set of string utility functions into a separate module file, import them into a main script, and use a Set to deduplicate a list of sample words.

Phase 3 — Intermediate & Advanced Core

Weeks 11–17: Master OOP, error handling, asynchronous programming, and the Node.js ecosystem.

Week 11 — Objects & Object-Oriented Basics

Model real-world entities with objects and understand JavaScript's prototype system.

Core Focus

Modeling real-world entities with objects and prototypes.

Topics & Subtopics

  1. Object Creation Patterns — Objects can be created via literals, constructor functions, or Object.create(), each with different use cases. These patterns clarify how JavaScript's flexible object model works.
  2. The Prototype Chain — Every JavaScript object has an internal link to a prototype it inherits from. This chain is the mechanism behind all inheritance in JavaScript.
  3. this Keyword — this refers to the context an object method is called on, which can change depending on how a function is invoked. Arrow functions and regular functions handle this differently.
  4. Object.keys(), values(), entries() — These methods extract an object's keys, values, or key-value pairs as arrays, bridging objects into the array method ecosystem.

Why This Week Matters: Understanding the prototype chain and how this works is what separates JavaScript developers who "use" OOP from those who truly understand it. This knowledge is also tested in almost every technical interview.

Code Example

const car = {
    model: "Tesla",
    year: 2024,
    describe() {
        return `${this.model} (${this.year})`;
    }
};
console.log(car.describe());
console.log(Object.keys(car));

🛠 Practical Task

Create a book object with title, author, and price properties plus a displayInfo() method, then use Object.entries() to log each property and its value.

Week 12 — ES6 Classes & Encapsulation

Use modern class syntax to structure object-oriented JavaScript code.

Core Focus

Using modern class syntax to structure object-oriented code.

Topics & Subtopics

  1. Class Declarations & Constructors — The class keyword provides clean syntax over prototype-based inheritance. The constructor() method initializes new instances.
  2. Instance Methods & Properties — Methods inside a class body become part of each instance's prototype, while constructor-assigned properties belong to each individual object.
  3. Private Fields (#field) — The # prefix marks class fields and methods as truly private, inaccessible from outside the class — giving JavaScript proper encapsulation.
  4. Getters & Setters — get and set keywords define methods that behave like properties, allowing validation logic to run transparently on read and write access.

Why This Week Matters: ES6 classes and private fields are now standard in modern JavaScript codebases. Private fields in particular solve a long-standing problem with encapsulation in JavaScript OOP.

Code Example

class BankAccount {
    #balance = 0;
    deposit(amount) {
        if (amount > 0) this.#balance += amount;
    }
    get balance() { return this.#balance; }
}
const acc = new BankAccount();
acc.deposit(100);
console.log(acc.balance); // 100

Refactor your book object from Week 11 into a Book class with private fields, a constructor, and a setter for price that rejects negative values.

Week 13 — Inheritance & Polymorphism

Build class hierarchies and write flexible, reusable behavior across related types.

Core Focus

Building class hierarchies and flexible, reusable behavior.

Topics & Subtopics

  1. extends & Subclasses — A class can inherit from another using extends, gaining the parent's methods and properties. This promotes code reuse across related object types.
  2. super Keyword — super() calls the parent constructor, and super.method() calls an overridden parent method. Essential when a subclass needs to extend rather than fully replace parent behavior.
  3. Method Overriding — A subclass can redefine an inherited method, and JavaScript will use the subclass version at runtime — enabling polymorphic behavior.
  4. instanceof & Type Checking — instanceof checks whether an object was created from a particular class or its subclasses. Useful for runtime type-checking in polymorphic code.

Why This Week Matters: Inheritance and polymorphism allow writing flexible code that works across many object types without knowing the exact type in advance — a concept used throughout modern JavaScript frameworks.

Code Example

class Animal {
    speak() { return "Some sound"; }
}
class Dog extends Animal {
    speak() { return "Woof!"; }
}
const pet = new Dog();
console.log(pet.speak());          // "Woof!"
console.log(pet instanceof Animal); // true

🛠 Practical Task

Create a Shape base class with an area() method, then build Circle and Rectangle subclasses that override it. Loop through an array of shapes logging each area.

Week 14 — Error Handling & Defensive Coding

Write resilient code that handles failures gracefully instead of crashing.

Core Focus

Writing resilient code that handles failures gracefully.

Topics & Subtopics

  1. try...catch...finally — Risky code goes in try, errors are caught in catch, and finally always runs. This isolates failures from crashing the rest of the program.
  2. The Error Object & Custom Errors — JavaScript's Error object carries a message and stack trace, and can be extended to create custom error types like ValidationError.
  3. throw Statement — throw manually raises an error, immediately halting normal execution until caught. It is how your own code signals that something has gone wrong.
  4. Validating Input Defensively — Checking arguments and throwing early for invalid input prevents bugs from silently propagating deeper. This "fail fast" approach is a hallmark of professional code.

Why This Week Matters: Unhandled errors crash applications and expose users to bad experiences. Proper error handling, especially in async code (Weeks 15–16), is what separates production-ready code from prototype code.

Code Example

class InsufficientFundsError extends Error {
    constructor(msg) {
        super(msg);
        this.name = "InsufficientFundsError";
    }
}
function withdraw(balance, amount) {
    if (amount > balance) throw new InsufficientFundsError("Not enough funds");
    return balance - amount;
}
try { withdraw(50, 100); }
catch (e) { console.log(e.name, "-", e.message); }

🛠 Practical Task

Write a withdraw function that throws a custom InsufficientFundsError when overdrawn, and a calling script that handles it gracefully with a friendly console message.

Week 15 — Asynchronous JavaScript: Callbacks & Promises

Handle operations that take time without blocking program execution.

Core Focus

Handling operations that take time without blocking execution.

Topics & Subtopics

  1. The Event Loop — JavaScript runs on a single thread but uses an event loop to handle async tasks like timers and I/O without blocking. Understanding this model explains why JS code does not "wait" by default.
  2. Callbacks & Callback Hell — A callback is a function passed to execute once an async operation finishes. Nesting many callbacks creates hard-to-read "callback hell," the pain point that motivated Promises.
  3. Promises — A Promise represents a value available now, later, or never, with .then()/.catch() handling success and failure. Promises provide a cleaner, chainable alternative to nested callbacks.
  4. Promise.all() & Promise.race() — Promise.all() waits for all promises to resolve, while Promise.race() resolves as soon as the first one finishes. Both coordinate multiple concurrent async operations.

Why This Week Matters: Almost everything in real-world JavaScript is asynchronous — API calls, database queries, file reads, timers. The event loop and Promises are the foundation you must understand before modern async/await syntax makes complete sense.

Code Example

function fetchData() {
    return new Promise((resolve, reject) => {
        setTimeout(() => resolve("Data loaded!"), 1000);
    });
}
fetchData()
    .then(data => console.log(data))
    .catch(err => console.error(err));

🛠 Practical Task

Write three functions that each return a Promise resolving after a different delay (simulating API calls), then use Promise.all() to log all results once every one has completed.

Week 16 — async/await & Working with APIs

Write clean asynchronous code and fetch real data from REST APIs.

Core Focus

Writing clean asynchronous code and fetching real data.

Topics & Subtopics

  1. async Functions — Marking a function async makes it always return a Promise and allows the use of await inside it — the modern standard for readable async code.
  2. await Keyword — await pauses execution within an async function until a Promise resolves, without blocking the rest of the application.
  3. fetch() API — fetch() makes HTTP requests to servers or REST APIs, returning a Promise that resolves to a response object. It is the standard built-in tool for network communication.
  4. Error Handling in Async Code — Wrapping await calls in try...catch properly handles network failures. Forgetting this is a common source of unhandled promise rejection bugs.

Why This Week Matters: async/await is the standard way professionals write asynchronous JavaScript today. Combining it with fetch() to call real APIs unlocks a massive range of possible applications.

Code Example

async function getUser(id) {
    try {
        const response = await fetch(`https://api.example.com/users/${id}`);
        const data = await response.json();
        return data;
    } catch (error) {
        console.error("Fetch failed:", error.message);
    }
}

🛠 Practical Task

Write an async function that fetches data from a public test API like JSONPlaceholder, logs the result, and properly catches and logs any network errors.

Week 17 — The Node.js Ecosystem: File System & npm

Work with files and third-party packages outside the browser using Node.js.

Core Focus

Working with files and third-party packages outside the browser.

Topics & Subtopics

  1. Node's File System (fs) Module — The built-in fs module reads and writes files on disk in both synchronous and Promise-based forms, enabling JavaScript to interact with the local file system.
  2. npm & package.json — npm is Node's package manager, and package.json declares a project's metadata and dependencies. Nearly every real-world JS project uses this ecosystem.
  3. CommonJS vs ES Modules in Node — Node historically used require()/module.exports, while modern Node supports import/export ES Modules. Knowing both is necessary when reading varied codebases.
  4. Environment Variables & .env — Environment variables store configuration like API keys outside source code. This is standard practice for keeping secrets out of version control.

Why This Week Matters: Almost every real JavaScript project beyond simple scripts runs in Node and uses npm packages. Understanding the file system and package ecosystem is essential for backend development, CLI tools, and build pipelines.

Code Example

import { readFile, writeFile } from "fs/promises";

async function saveNote(text) {
    await writeFile("notes.txt", text);
    const content = await readFile("notes.txt", "utf-8");
    console.log("Saved content:", content);
}
saveNote("Learning Node.js file system!");

Phase 4 — Master Level Concepts

Weeks 18–24: Tackle generators, advanced async patterns, design patterns, performance, and build three complete portfolio projects.

Week 18 — Generators, Iterators & Closures

Master advanced control-flow and functional patterns unique to JavaScript.

Core Focus

Mastering advanced control-flow and functional patterns unique to JavaScript.

Topics & Subtopics

  1. Closures — A closure forms when an inner function retains access to its outer function's variables even after the outer function has returned. Closures power private state, memoization, and module encapsulation.
  2. Generator Functions (function*) — Generators use function* and yield to pause and resume execution, producing values lazily over time. They are the foundation for custom iterators and certain async patterns.
  3. The Iterator Protocol — An object becomes iterable by implementing [Symbol.iterator] returning an object with a .next() method. This is what makes for...of work on arrays, strings, Maps, and custom objects.
  4. Lazy Evaluation — Generators compute values only when requested rather than all at once, making them memory-efficient for large or infinite sequences.

Why This Week Matters: Generators and closures are what JavaScript developers point to as truly "advanced" features. They underlie many library patterns, async iteration, and sophisticated state management techniques.

Code Example

function makeCounter() {
    let count = 0;
    return () => ++count;
}
function* idGenerator() {
    let id = 1;
    while (true) yield id++;
}
const gen = idGenerator();
console.log(gen.next().value, gen.next().value); // 1 2

🛠 Practical Task

Write a generator function that lazily yields the Fibonacci sequence, and a closure-based function that creates an independent counter for each separate call.

Week 19 — Advanced Asynchronous Patterns & Concurrency

Coordinate complex asynchronous workflows like a senior engineer.

Core Focus

Coordinating complex asynchronous workflows like a senior engineer.

Topics & Subtopics

  1. Microtasks vs Macrotasks — Promises resolve in the microtask queue, which runs before macrotasks like setTimeout callbacks. This explains subtle ordering behavior in mixed sync/async code.
  2. Promise.allSettled() & Promise.any() — allSettled() waits for every promise regardless of failure, while any() resolves as soon as one succeeds. Both give finer control than all()/race().
  3. Async Iterators (for await...of) — Async generators produce promises that resolve over time, consumed cleanly with for await...of. Essential for streaming data like paginated API results.
  4. Web Workers / Worker Threads — Since JS is single-threaded, true parallel computation requires Web Workers in the browser or worker_threads in Node. These communicate via message-passing rather than shared memory.

Why This Week Matters: Real applications deal with partial failures, streaming data, and CPU-intensive tasks. These advanced async patterns are the tools senior JavaScript engineers reach for when basic Promises are not enough.

Code Example

async function fetchAll(urls) {
    const results = await Promise.allSettled(urls.map(u => fetch(u)));
    results.forEach(r => console.log(r.status));
}
async function* streamNumbers() {
    for (let i = 1; i <= 3; i++) {
        await new Promise(r => setTimeout(r, 100));
        yield i;
    }
}

🛠 Practical Task

Simulate 5 API calls with varying success/failure using Promise.allSettled() and log a summary, then write an async generator that streams 5 numbers with a delay between each.

Week 20 — Design Patterns & Functional Programming

Write maintainable, idiomatic JavaScript at a professional level.

Core Focus

Writing maintainable, idiomatic JavaScript at a professional level.

Topics & Subtopics

  1. Module Pattern & Singleton — Using closures or ES modules to expose only a controlled public interface mimics the Singleton pattern, ensuring shared, centralized state and clean APIs.
  2. Factory Functions — A factory function creates and returns objects without class or new, offering a flexible alternative for object creation popular in functional-leaning codebases.
  3. Currying & Partial Application — Currying transforms a multi-argument function into a sequence of single-argument functions, enabling reusable, pre-configured function variants via closures.
  4. Pure Functions & Immutability — A pure function always produces the same output for the same input with no side effects. Favoring immutable data via spread syntax makes code more predictable and testable.

Why This Week Matters: These design patterns appear throughout major JavaScript libraries and frameworks. Currying is used in libraries like Ramda and Redux, factory functions power many testing utilities, and pure functions are central to React's design philosophy.

Code Example

const curriedAdd = (a) => (b) => (c) => a + b + c;
console.log(curriedAdd(1)(2)(3)); // 6

const addItem = (cart, item) => [...cart, item];
const cart1 = ["apple"];
const cart2 = addItem(cart1, "banana");
console.log(cart1, cart2); // original unchanged

Week 21 — Performance, Memory & Browser Internals

Understand what happens under the hood for optimized, professional JavaScript.

Core Focus

Understanding what happens under the hood for optimized code.

Topics & Subtopics

  1. The Call Stack & Stack Overflow — JavaScript tracks function calls in a call stack, and excessive recursion without a base case causes a RangeError. Understanding the stack clarifies recursive behavior and error traces.
  2. Garbage Collection in JS — JavaScript engines automatically reclaim memory from unreachable objects using mark-and-sweep. Memory leaks occur when references are unintentionally kept alive, such as forgotten event listeners.
  3. Debouncing & Throttling — Debouncing delays execution until after a pause in repeated calls, while throttling limits execution to a fixed rate. Both are essential for handling frequent events like scrolling or typing.
  4. Browser Rendering Basics (DOM & Reflow) — Frequent direct DOM manipulation triggers costly reflows and repaints. Batching DOM updates and minimizing layout-affecting changes keeps UIs fast and responsive.

Why This Week Matters: Senior JavaScript developers understand why their code is fast or slow. Debouncing and throttling are asked about in nearly every frontend interview, and understanding the garbage collector prevents production memory leaks.

Code Example

function debounce(fn, delay) {
    let timer;
    return (...args) => {
        clearTimeout(timer);
        timer = setTimeout(() => fn(...args), delay);
    };
}
const logSearch = debounce((term) => console.log("Searching:", term), 300);
logSearch("a");
logSearch("ap");
logSearch("app"); // only "app" logs after 300ms

Code Example

function createStore(initialState) {
    let state = initialState;
    const listeners = [];
    return {
        getState: () => state,
        subscribe: (fn) => listeners.push(fn),
        setState: (newState) => {
            state = { ...state, ...newState };
            listeners.forEach(fn => fn(state));
        }
    };
}
