About This Roadmap

Data Science is the discipline of extracting meaning, insight, and predictions from data using a combination of programming, statistics, and domain knowledge. This roadmap takes a complete beginner to a confident, portfolio-ready data scientist in 15 weeks of full-time study (~35–40 hours/week).

The path runs: Python Foundations → NumPy & pandas → Statistics & Math → Data Cleaning → Visualization → EDA → Machine Learning → Advanced ML → Deep Learning → SQL & Deployment → Capstone.

By Week 15 you will have built approximately 14 mini-projects and one complete, deployed data science project — presented as an interactive web application on Streamlit Cloud.

Week 1 — Python Fundamentals for Data Science

Write real Python programs using variables, functions, collections, and control flow — with examples drawn from data science contexts throughout.

Topics & Subtopics

  1. Python environment setup and first notebook — running Python in Jupyter Notebook cells, Google Colab, and .py scripts; print(), type(), comments; Markdown cells in Jupyter.
  2. Variables and data types — int, float, str, bool; type conversion; f-strings: f"The mean is {mean:.2f}".
  3. Collections for data science — lists (index, slice, .append(), .sort()), tuples (coordinate pairs, column name tuples), dictionaries ({"feature": value}), sets (unique values), list comprehensions.
  4. Control flow — if/elif/else; for loop; while loop; break, continue.
  5. Functions — def, parameters, return values, default parameters, lambda functions, built-in functions: len(), sum(), min(), max(), sorted(), zip(), enumerate(), map(), filter().
  6. File I/O basics — open(); reading .txt and .csv files line by line; context manager: with open(...) as f:.
  7. Python OOP basics (light intro) — classes and objects; __init__(), self, methods — enough to understand scikit-learn's fit()/predict() pattern.

Why This Week Matters: Every data science tool (NumPy, pandas, scikit-learn) is a Python library. Without Python fluency, you can't read documentation, understand errors, or modify examples to solve your specific problems. The list/dict/function skills this week are used in every subsequent week.

Common Mistakes

  • Confusing = (assignment) with == (comparison).
  • Modifying a list while iterating over it.
  • Using mutable default arguments in functions (def f(lst=[]) — the list is shared across calls).
  • Not documenting Jupyter notebooks — running cells out of order.

How to Avoid Them

  • Read comparisons aloud: "if x equals-equals 5."
  • Iterate over a copy: for item in list.copy(): if modification is needed.
  • Use None as a default argument: def f(lst=None): if lst is None: lst = [].
  • Run Jupyter notebooks from top to bottom (Kernel → Restart & Run All) regularly.

Week 2 — NumPy & pandas: The Data Science Foundation

Master NumPy for fast numerical computing and pandas for real-world tabular data manipulation — the two most important data science libraries.

Topics & Subtopics

  1. NumPy — creating arrays (np.array(), np.zeros(), np.arange(), np.linspace(), np.random.rand()); shapes/dtypes; indexing/slicing; boolean indexing; broadcasting; ufuncs (np.sum(), np.mean(), np.std(), np.percentile()); reshaping; np.where(), np.unique(); linear algebra intro.
  2. pandas — Series & DataFrame — creating from dict/list/CSV; pd.read_csv(), pd.read_excel(), pd.read_json().
  3. Inspecting data — .head(), .tail(), .shape, .info(), .describe(), .dtypes, .columns.
  4. Selecting data — df['col'], df[['col1','col2']], df.loc[] (label-based), df.iloc[] (integer-based); boolean filtering with & and |.
  5. Transforming data — adding/removing columns; sorting; groupby().agg(); value_counts().
  6. Handling missing data — .isnull(), .notnull(), .fillna(), .dropna().
  7. String operations — df['col'].str.lower(), .str.contains(), .str.split().
  8. Merging and dates — pd.merge(), pd.concat(), pd.to_datetime(), .to_csv(), .to_excel().

Why This Week Matters: NumPy and pandas are the absolute core of Python data science. 95% of real data science work begins with loading a dataset into a pandas DataFrame and manipulating it. Every other library (scikit-learn, Matplotlib, Seaborn) works with NumPy arrays or pandas DataFrames as inputs.

🛠 Mini Project

Common Mistakes

  • Using df['col'] == None instead of df['col'].isnull() — the == None check doesn't work with pandas NaN.
  • Chaining operations without inplace=True or assignment — changes are not saved.
  • Using .iloc and .loc interchangeably — they behave differently on non-integer indexes.
  • Forgetting that boolean filtering with multiple conditions requires parentheses: (df['a'] > 0) & (df['b'] < 10).

How to Avoid Them

  • Missing values: always use .isnull() / .notnull(), never == None or == np.nan.
  • All pandas operations return a new DataFrame unless inplace=True — always assign: df = df.dropna().
  • .loc['label'] for label-based; .iloc[5] for position-based.
  • Multiple conditions always need parentheses AND use &/| operators, never and/or.

Week 3 — Statistics & Mathematics for Data Science

Build the statistical and mathematical intuition needed to understand and correctly apply data science algorithms.

Topics & Subtopics

  1. Descriptive statistics — mean, median, mode (when each is appropriate); variance, standard deviation, IQR, range; percentiles and quantiles; skewness and kurtosis; covariance and correlation.
  2. Probability fundamentals — sample space, events, probability; conditional probability P(A|B); Bayes' theorem (critical for Naive Bayes); independence vs dependence.
  3. Probability distributions — Bernoulli, Binomial, Poisson; Uniform, Normal (Gaussian); the 68-95-99.7 rule; Z-score; Central Limit Theorem (CLT).
  4. Hypothesis testing — null hypothesis (H₀) vs alternative hypothesis (H₁); p-value; significance level (α = 0.05); t-test (one-sample, two-sample, paired); chi-squared test; confidence intervals.
  5. Linear algebra basics (for ML) — vectors and dot products; matrices (multiplication, transpose, inverse); eigenvalues and eigenvectors (conceptual — for PCA in Week 10).
  6. Calculus intuition (for ML) — derivatives (slope of a function); gradient (derivative of a multi-variable function); gradient descent — the optimization algorithm behind most ML and all deep learning.
  7. Computing statistics with Python — scipy.stats: ttest_ind(), chi2_contingency(), pearsonr(); visualizing distributions with Matplotlib and Seaborn.

Why This Week Matters: Understanding statistics is what separates a data scientist from someone who runs code without understanding it. Misinterpreting a p-value, not knowing when correlation implies causation, or applying a model that assumes normality to non-normal data are common errors with real consequences. The intuition for gradient descent (this week) directly explains why neural networks train the way they do (Week 13).

Daily Study Plan

Day Focus
Day 1 Descriptive stats — compute mean/median/mode/std/IQR on the Titanic dataset; interpret each.
Day 2 Skewness + correlation matrix on a real dataset; visualize with histograms and scatter plots.
Day 3 Probability basics — Bayes' theorem; simulate coin flips and dice rolls with numpy.random.
Day 4 Normal distribution + Z-score + CLT — simulate 1000 sample means and verify normality.
Day 5 Hypothesis testing — t-test (does survival rate differ by sex?), chi-squared (are class and survival independent?).
Day 6 Linear algebra with NumPy — vector dot products, matrix multiplication; connect to ML context.
Day 7 Gradient descent simulation — code a gradient descent on a simple quadratic function; visualize convergence + mini project + GitHub commit.

Practice Tasks

  • Test whether the average salary differs significantly between two departments using a t-test.
  • Calculate the correlation matrix for a dataset and identify the 3 most correlated feature pairs.
  • Simulate rolling a die 10,000 times and verify the Central Limit Theorem on the sample means.

🛠 Mini Project

A Statistical Analysis Report notebook — loads a real dataset (e.g., the gapminder dataset), computes all descriptive statistics with interpretations, tests 3 hypotheses using the appropriate test, reports p-values with correct interpretation, shows a correlation matrix heatmap, and simulates the Central Limit Theorem on one variable — with thorough Markdown documentation — pushed to GitHub.

Common Mistakes

  • Confusing correlation with causation — two variables moving together does not mean one causes the other.
  • Interpreting p-value as "probability the null hypothesis is true" — it is not.
  • Applying a t-test when data is not approximately normal and sample sizes are small.
  • Using mean for skewed data (salary, house prices) instead of median.

How to Avoid Them

  • Always plot your data before computing statistics — outliers and skewness are visible in a histogram.
  • p-value = "probability of observing this result if H₀ is true" — never "probability H₀ is false."
  • For small samples with non-normal data: use non-parametric tests (Mann-Whitney U, Kruskal-Wallis).
  • Report median and IQR for skewed variables; mean and std for symmetric, normally distributed ones.

  • Khan Academy Statistics (khanacademy.org) — free, interactive
  • scipy.stats documentation (docs.scipy.org)

Phase 2 — Data Analysis & Visualization

Weeks 4 through 6 transform raw, messy data into actionable insight. You will learn to clean real-world datasets, create compelling visualizations with Matplotlib, Seaborn, and Plotly, and conduct a complete structured Exploratory Data Analysis — the process every data scientist performs before training any model.

Week 4 — Data Cleaning & Preprocessing

Transform raw, messy, real-world data into clean, analysis-ready data — handling missing values, outliers, duplicates, encoding, and scaling.

Topics & Subtopics

  1. Understanding dirty data — types of problems: missing values, duplicates, inconsistent formats, incorrect data types, outliers, impossible values; data quality assessment.
  2. Handling missing values — .isnull().sum(), .isnull().mean(); strategies: deletion (dropna()), mean/median/mode imputation, forward/backward fill, SimpleImputer from scikit-learn.
  3. Handling duplicates — .duplicated(), .drop_duplicates().
  4. Data type correction — pd.to_numeric(), pd.to_datetime(), .astype().
  5. Encoding categorical variables — Label Encoding (LabelEncoder); One-Hot Encoding (pd.get_dummies() / OneHotEncoder); Ordinal Encoding; the dummy variable trap.
  6. Outlier detection and handling — Z-score method (±3 std); IQR method (Q1 - 1.5*IQR, Q3 + 1.5*IQR); box plots; remove, cap (winsorization), or log transform.
  7. Feature scaling — MinMaxScaler (0–1 range); StandardScaler (mean=0, std=1); when scaling matters (k-NN, SVM, neural networks) vs when it doesn't (Decision Trees).
  8. scikit-learn Pipeline intro — Pipeline for chaining preprocessing steps; ColumnTransformer for different transformations per column.

Why This Week Matters: Real-world data is always messy. Data scientists spend 60–80% of their time cleaning data. A model trained on uncleaned data produces unreliable results. The preprocessing skills taught this week are applied in every machine learning project from Week 7 onward.

Daily Study Plan

Day Focus
Day 1 Data profiling — load a messy dataset; assess missing values, dtypes, duplicates, ranges.
Day 2 Missing value handling — compare deletion vs imputation on a regression target.
Day 3 Categorical encoding — label + one-hot on the same column; compare the resulting DataFrames.
Day 4 Outlier detection — Z-score + IQR on salary/price columns; box plot visualization.
Day 5 Feature scaling — apply MinMaxScaler and StandardScaler; observe effect on data distribution.
Day 6 Pipeline + ColumnTransformer — combine all preprocessing steps for numerical and categorical columns.
Day 7 End-to-end cleaning of a new dataset + mini project + GitHub commit.

Practice Tasks

  • Clean the Ames Housing dataset completely: handle all 81 features with missing values and encode all categorical columns.
  • Compare model performance (use a simple LinearRegression) on the raw vs cleaned dataset.
  • Build a reusable clean_dataset(df) function that handles the most common issues.

🛠 Mini Project

A Data Cleaning Pipeline notebook — loads a real messy dataset (Titanic or Ames Housing), builds a full Pipeline + ColumnTransformer that: imputes missing numerics with median, imputes missing categoricals with mode, one-hot encodes categoricals, standardizes numerics — saves the cleaned output to CSV, documents every decision with Markdown justification — pushed to GitHub.

Common Mistakes

  • Imputing with the mean of the entire dataset instead of the training set mean (causes data leakage).
  • One-hot encoding without dropping one category (dummy variable trap — multicollinearity).
  • Scaling the target variable (only scale features, not the target).
  • Not encoding categorical columns before feeding to scikit-learn (most estimators don't accept strings).

How to Avoid Them

  • Fit scalers and imputers only on training data: scaler.fit(X_train) then scaler.transform(X_test).
  • Use pd.get_dummies(df, drop_first=True) or OneHotEncoder(drop='first').
  • Only scale X (features), never y (target) for most algorithms.

Week 6 — Exploratory Data Analysis (EDA)

Conduct a complete, structured Exploratory Data Analysis from start to finish — combining all Phase 1 and Phase 2 skills to systematically understand a dataset before modelling.

Topics & Subtopics

  1. EDA framework — the 6-step process: understand the problem → understand the data → assess quality → explore univariate → explore bivariate/multivariate → summarize findings; EDA is question-driven, not tool-driven.
  2. Univariate analysis — numerical: histogram, box plot, mean/median/std/skewness, outlier detection; categorical: value counts, bar chart, proportion table.
  3. Bivariate analysis — numerical vs numerical: scatter plot, correlation coefficient; categorical vs numerical: box plot by category, bar chart of means; categorical vs categorical: pd.crosstab(), stacked bar chart, chi-squared test.
  4. Multivariate analysis — pair plots with hue; correlation matrix heatmap; scatter plots with color and size encoding; 3D scatter plots with Plotly.
  5. Feature relationships and business insights — identifying the most predictive features for the target; identifying multicollinearity.
  6. Profiling tools — ydata-profiling (automatic EDA reports); sweetviz (comparative EDA); when to use auto-profiling vs manual EDA.
  7. Communicating EDA findings — structured EDA report in Jupyter; key findings → supporting evidence → implications for modelling.

Why This Week Matters: EDA is what data scientists do before training any model. The insights from EDA determine: which features to include, which to engineer, which algorithms are appropriate, and what data quality issues need fixing. A thorough EDA prevents countless modelling mistakes downstream.

Daily Study Plan

Day Focus
Day 1 Choose a dataset from Kaggle; define 10 specific questions you want to answer; write them in Markdown.
Day 2 Univariate analysis — all numerical + all categorical columns with visualizations and written observations.
Day 3 Bivariate: numerical vs numerical — scatter plots, correlation; find the top correlated features.
Day 4 Bivariate: categorical vs numerical — box plots; categorical vs categorical — cross-tabs + chi-squared.
Day 5 Multivariate — pair plot, correlation heatmap, 3-variable scatter.
Day 6 Auto-profiling with ydata-profiling; compare findings to your manual EDA.
Day 7 Write full EDA summary + implications for modelling + mini project + GitHub commit.

🛠 Mini Project

A Complete EDA Report notebook — conducts a structured EDA on a real Kaggle dataset (student choice), following all 6 EDA phases, with 10+ visualizations, 3 statistical tests, a correlation heatmap, an auto-profiling report, a written summary of key findings, and a "next steps for modelling" section — this notebook should be portfolio-quality: clean, well-documented, interpretive — pushed to GitHub.

Common Mistakes

  • Running EDA without questions — producing plots that don't answer anything.
  • Analyzing all 50 columns of a dataset equally — focus on the target variable and the most relevant features.
  • Only using one visualization type (e.g., only histograms).
  • Not writing interpretive text — a plot without interpretation is just an image.

How to Avoid Them

  • Write your research questions first — every chart should answer one of them.
  • Start with .corr()['target'].sort_values() — immediately ranks features by correlation with the target.
  • Use at least 4 different visualization types.
  • Every chart gets a 1–3 sentence interpretation below it.

Week 7 — Machine Learning Fundamentals & Linear Regression

Understand the core concepts of machine learning and implement your first predictive model — Linear Regression — using scikit-learn's fit/predict pattern.

Topics & Subtopics

  1. Machine learning fundamentals — types of ML (supervised/unsupervised/reinforcement); the ML workflow; training/validation/test sets; train_test_split(); bias-variance tradeoff; underfitting vs overfitting.
  2. scikit-learn API pattern — consistent fit()/predict()/score() interface across all algorithms.
  3. Simple Linear Regression — y = β₀ + β₁x; Ordinary Least Squares (OLS); interpreting coefficients; assumptions (linearity, independence, homoscedasticity, normality of residuals).
  4. Multiple Linear Regression — multiple predictors; feature matrix X shape; model.coef_, model.intercept_.
  5. Regression evaluation metrics — MAE, MSE, RMSE, R² (R-squared), Adjusted R²; from sklearn.metrics import mean_squared_error, r2_score.
  6. Residual analysis — plotting residuals vs fitted values; checking normality of residuals; detecting patterns.
  7. Polynomial Regression — PolynomialFeatures(degree=2); fitting a curved relationship with a linear model.

Why This Week Matters: Linear Regression is the foundation of ML — understanding it deeply (especially bias-variance tradeoff and residual analysis) makes every subsequent algorithm easier to understand. The scikit-learn API pattern you learn here (fit/predict/score) is the same for every algorithm through Week 13.

Week 8 — Classification: Logistic Regression, KNN & Decision Trees

Solve classification problems — predicting a category rather than a number — using three foundational algorithms and classification-specific evaluation metrics.

Topics & Subtopics

  1. Classification concepts — binary vs multi-class; decision boundary; class imbalance.
  2. Classification evaluation metrics — confusion matrix (TP, TN, FP, FN); accuracy, precision, recall (sensitivity), F1-score; ROC curve and AUC; classification_report(), confusion_matrix(), roc_auc_score(); when to prioritize precision vs recall.
  3. Logistic Regression — sigmoid function; decision threshold (default 0.5); model.predict_proba(); interpreting coefficients in log-odds terms.
  4. K-Nearest Neighbors (KNN) — majority vote among K nearest neighbors; distance metrics (Euclidean, Manhattan); choosing K with cross-validation; why KNN requires feature scaling; curse of dimensionality.
  5. Decision Trees — recursive binary splitting; Gini impurity vs entropy; max_depth, min_samples_split; tree.plot_tree(); dt.feature_importances_; no scaling required.
  6. Handling class imbalance — SMOTE (imbalanced-learn); undersampling; class_weight='balanced'; use AUC-ROC not accuracy for imbalanced problems.

Why This Week Matters: Classification is the most common ML task — spam detection, churn prediction, disease diagnosis, image recognition, and credit risk all frame the problem as "which class does this belong to?" The confusion matrix, precision/recall, and AUC introduced this week are used to evaluate every classifier you'll train.

Week 9 — Advanced Supervised Learning: SVM, Naive Bayes & Cross-Validation

Learn two more powerful classifiers (SVM and Naive Bayes), master cross-validation for reliable model evaluation, and apply hyperparameter tuning.

Topics & Subtopics

  1. Support Vector Machines (SVM) — maximum-margin hyperplane; support vectors; soft-margin SVM with C parameter; kernel trick; kernels: linear, rbf, poly; always requires feature scaling.
  2. Naive Bayes — Bayes' theorem applied to classification; naive independence assumption; Gaussian NB (continuous features); Multinomial NB (text/word counts); Bernoulli NB (binary features); best use case: text classification.
  3. Cross-Validation — why a single train/test split is insufficient; K-Fold CV: train on K-1 folds, test on 1; cross_val_score(estimator, X, y, cv=5, scoring='f1'); Stratified K-Fold for imbalanced classes; reporting: mean ± standard deviation of CV scores.
  4. Hyperparameter Tuning — hyperparameters vs parameters; Grid Search: GridSearchCV(estimator, param_grid, cv=5); Random Search: RandomizedSearchCV; fitting on train, evaluating on test.
  5. Model selection best practices — the final test set is used ONCE; learning curves; validation curves.

Why This Week Matters: Real ML projects require reliable model evaluation — a single train/test split can be misleading. Cross-validation gives you a confidence interval on your model's performance. GridSearchCV is the standard way to find the best hyperparameters. SVM and Naive Bayes add powerful tools for text and high-dimensional data.

Common Mistakes

  • Using the test set during hyperparameter tuning — makes the test set a "training" set.
  • Reporting CV scores without standard deviation — "F1 = 0.87" is less useful than "F1 = 0.87 ± 0.03."
  • Forgetting to scale for SVM — kernel computations are meaningless on unscaled features.
  • Using Grid Search with a very fine grid on too many parameters — exponentially slow.

How to Avoid Them

  • The test set is used ONCE — after all GridSearch and tuning is done.
  • Always report: f"{scores.mean():.3f} ± {scores.std():.3f}".
  • SVM always goes in a Pipeline after StandardScaler.
  • Start with a coarse grid (e.g., C: [0.1, 1, 10]), then refine around the best value.

Week 10 — Unsupervised Learning: Clustering, PCA & Anomaly Detection

Discover patterns in data without labels using clustering algorithms, reduce data dimensionality with PCA, and detect anomalies.

Topics & Subtopics

  1. K-Means Clustering — algorithm: initialize centroids → assign → recompute → repeat; KMeans(n_clusters=k); Elbow method (inertia); Silhouette score; limitations: spherical clusters, sensitive to outliers.
  2. Hierarchical Clustering — agglomerative (bottom-up) vs divisive; dendrogram; linkage methods (Ward, complete, average); scipy.cluster.hierarchy.dendrogram().
  3. DBSCAN — density-based; no need to specify K; finds arbitrarily-shaped clusters; marks noise as -1; eps and min_samples parameters.
  4. Cluster evaluation — Silhouette score; Davies-Bouldin index; visual inspection via PCA projection.
  5. Principal Component Analysis (PCA) — finds orthogonal axes of maximum variance; PCA(n_components=2).fit_transform(X); explained variance ratio; scree plot; uses: visualization, preprocessing, reducing overfitting.
  6. t-SNE and UMAP (intro) — non-linear dimensionality reduction for visualization; from sklearn.manifold import TSNE.
  7. Anomaly Detection (intro) — IsolationForest(contamination=0.05); Local Outlier Factor (LOF); Z-score based detection; use cases: fraud detection, network intrusion.

Why This Week Matters: Many real problems have no labels — you don't know what categories exist in advance. Customer segmentation, document grouping, and anomaly detection are inherently unsupervised tasks. PCA is used as a preprocessing step in many supervised ML pipelines to reduce dimensionality and improve performance.

Common Mistakes

  • Decorate any loading function with @st.cache_data or @st.cache_resource.
  • Add input validation in the Streamlit app before calling model.predict().
