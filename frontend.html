Web Frontend Development

Complete 12-Week Beginner-to-Job-Ready Roadmap — HTML, CSS, JavaScript & React

Intro W1 W2 W3 W4 W5 W6 W7 W8 W9 W10 W11 W12

About This Roadmap

This roadmap takes a complete beginner to a job-ready, portfolio-equipped frontend developer in 12 weeks of full-time study. The path runs: HTML → CSS → Responsive Design → Git/GitHub → JavaScript → DOM & Browser APIs → Modern JavaScript (ES6+) → React → React Ecosystem → Deployment, Performance & Accessibility → Capstone Project.

Every week builds directly on the previous one. By Week 12 you will have 1 capstone project and 8–10 smaller projects to show in a portfolio, plus the conceptual depth to answer technical interview questions confidently.

Level: Beginner  |  Duration: 12 weeks  |  Study mode: Full-time (~35–40 hrs/week)  |  Framework: React

Week 1 — HTML Foundations

Understand how webpages are structured and build a complete static multi-section page using only semantic HTML.

Topics & Subtopics

  1. How the web works — client/server, browsers, HTTP basics (conceptual).
  2. HTML document structure — <!DOCTYPE>, <html>, <head>, <body>, metadata tags.
  3. Text elements — headings (h1–h6), paragraphs, span, div, line breaks, horizontal rules.
  4. Lists — ordered, unordered, description lists.
  5. Links & navigation — <a>, relative vs absolute paths, anchor links.
  6. Images & media — <img>, alt text, <video>, <audio>.
  7. Tables — <table>, <tr>, <td>, <th>, <thead>, <tbody>.
  8. Forms — <form>, <input> types, <textarea>, <select>, <button>, <label>.
  9. Semantic HTML5 — <header>, <nav>, <main>, <section>, <article>, <aside>, <footer>.

Why This Week Matters: HTML is the skeleton of every website. Every CSS rule and every JavaScript interaction targets an HTML element. Skipping or rushing semantic structure now creates messy, hard-to-style, inaccessible pages later.

Daily Study Plan

Day Focus
Day 1 How the web works (conceptual) + document structure + text elements. Build a "My Bio" page.
Day 2 Lists + links + navigation. Add a nav menu and list of hobbies to your bio page.
Day 3 Images, video, audio embedding. Add a profile picture and a sample video.
Day 4 Tables — build a sample timetable/marksheet table.
Day 5 Forms — build a contact form with text, email, dropdown, radio, checkbox, textarea, submit.
Day 6 Semantic HTML5 tags — restructure all previous pages using header/nav/main/section/footer.
Day 7 Review + mini project + push first code to GitHub.

Practice Tasks

  • Recreate the structure (not styling) of a simple Wikipedia article page using only HTML.
  • Build a multi-page "personal portfolio skeleton" (Home, About, Contact — linked via <a> tags).
  • Build a recipe page with an image, ingredient list (ul), and steps (ol).

🛠 Mini Project

A 3-page unstyled personal portfolio (Home, About, Contact) with semantic structure, a navigation bar linking all 3 pages, an image, and a working contact form (no backend needed yet).

Common Mistakes

  • Using <div> and <span> for everything instead of semantic tags.
  • Forgetting alt attributes on images.
  • Nesting block elements inside inline elements incorrectly.
  • Not closing tags properly.

How to Avoid Them

  • Always ask "is there a more meaningful tag for this?" before reaching for <div>.
  • Make alt text a non-negotiable habit from day one.
  • Use VS Code's auto-formatting (Prettier) to catch unclosed/misnested tags.

Free Resources

  • MDN Web Docs — HTML Basics (developer.mozilla.org)
  • freeCodeCamp — "Responsive Web Design" HTML section
  • W3Schools HTML Tutorial (for quick syntax lookup)

Estimated Study Hours: 35–38 hours

Week 2 — CSS Foundations

Style any HTML page confidently — colors, fonts, spacing, borders, and basic layout.

Topics & Subtopics

  1. CSS syntax & ways to add CSS — inline, internal, external stylesheets.
  2. Selectors — element, class, ID, grouping, descendant, pseudo-classes (:hover, :first-child), pseudo-elements (::before).
  3. The Box Model — content, padding, border, margin; box-sizing: border-box.
  4. Colors & backgrounds — HEX, RGB, RGBA, HSL; background-color, background-image, gradients.
  5. Typography — font-family, font-size, font-weight, line-height, Google Fonts.
  6. Display & positioning basics — display: block/inline/inline-block/none, position: static/relative/absolute/fixed/sticky.
  7. CSS specificity & the cascade — how conflicting rules are resolved.
  8. Units — px, %, em, rem, vh, vw.

Why This Week Matters: CSS turns a plain document into a designed website. The Box Model and specificity are the two concepts beginners misunderstand most — mastering them now prevents weeks of frustration.

Daily Study Plan

Day Focus
Day 1 CSS syntax, selectors, linking stylesheets. Style your bio page's text and headings.
Day 2 Box Model deep-dive (padding/border/margin/box-sizing). Style spacing on your nav and sections.
Day 3 Colors, backgrounds, gradients. Add a color theme to your portfolio.
Day 4 Typography — fonts, sizes, line-height. Import a Google Font and apply a type scale.
Day 5 Display property + position property (all 5 values practiced individually).
Day 6 Specificity & cascade rules + units (px vs rem vs %). Refactor CSS to use rem for fonts.
Day 7 Review + mini project + commit/push to GitHub.

Practice Tasks

  • Build a styled pricing card (image, title, price, feature list, button) using only Box Model + positioning.
  • Recreate a simple navbar with a logo on the left and 3 links on the right using display properties.
  • Style a quote/testimonial block with a colored left border and background tint.

🛠 Mini Project

Take your Week 1 portfolio and fully style it: consistent color palette, custom typography, styled navigation bar, styled buttons, and styled form — using only Box Model + display + position.

Common Mistakes

  • Forgetting box-sizing: border-box, causing unexpected element widths.
  • Overusing position: absolute to "force" layouts.
  • Relying only on px units, making the page un-scalable.
  • Writing overly specific selectors that become hard to override.

How to Avoid Them

  • Add * { box-sizing: border-box; } as a habitual reset at the top of every stylesheet.
  • Treat position: absolute/fixed as a last resort, not a default tool.
  • Use rem for font sizes and % or rem for spacing where it makes sense.

Free Resources

  • MDN Web Docs — CSS Basics & Box Model
  • CSS-Tricks — "The Box Model" article
  • freeCodeCamp — CSS section of Responsive Web Design
  • Josh W Comeau's free blog post "An Interactive Guide to CSS"

Estimated Study Hours: 35–38 hours

Week 3 — Flexbox & Layout Mastery

Build any modern one-dimensional layout confidently using Flexbox.

Topics & Subtopics

  1. Flex container properties — display: flex, flex-direction, flex-wrap, justify-content, align-items, align-content, gap.
  2. Flex item properties — flex-grow, flex-shrink, flex-basis, flex shorthand, align-self, order.
  3. Common Flexbox patterns — centering content, equal-height columns, sticky footer, navbar layouts, card grids.
  4. CSS variables (custom properties) — :root, var(), theming.
  5. Transitions & simple animations — transition, transform, :hover effects.

Why This Week Matters: Flexbox is the single most-used layout tool in real-world frontend work. Nearly every navbar, card layout, and button group in production websites uses Flexbox.

Daily Study Plan

Day Focus
Day 1 Flex container properties — flex-direction, justify-content, align-items. Play Flexbox Froggy.
Day 2 Flex-wrap, gap, align-content. Build a responsive card grid.
Day 3 Flex item properties — grow/shrink/basis. Build a 3-column equal-height layout.
Day 4 Common patterns — navbar (logo left, links right), sticky footer.
Day 5 CSS variables — define a color/spacing theme using :root and var().
Day 6 Transitions & hover effects — animate buttons, cards, links.
Day 7 Review + mini project + GitHub commit.

Practice Tasks

  • Build a pricing page with 3 equal-height pricing cards using Flexbox.
  • Build a navbar with a logo, center links, and a right-aligned button.
  • Build an image gallery that wraps responsively using flex-wrap.

🛠 Mini Project

A fully responsive "Restaurant Landing Page" with a Flexbox navbar, hero section, 3-column feature section, hover animations, and CSS-variable-based color theme.

Common Mistakes

  • Confusing justify-content (main axis) with align-items (cross axis).
  • Forgetting that flex-direction: column flips which axis is "main."
  • Overusing flex-grow without understanding flex-basis.

Free Resources

  • Flexbox Froggy (flexboxfroggy.com) — free interactive game
  • CSS-Tricks — "A Complete Guide to Flexbox"
  • MDN Web Docs — Flexbox guide
  • Kevin Powell (YouTube) — Flexbox tutorials

Estimated Study Hours: 35–38 hours

Week 4 — CSS Grid & Responsive Design

Build complex two-dimensional layouts and make any page fully responsive.

Topics & Subtopics

  1. CSS Grid container properties — display: grid, grid-template-columns/rows, gap, grid-template-areas.
  2. CSS Grid item properties — grid-column, grid-row, spanning cells.
  3. Grid vs Flexbox — when to use which.
  4. Responsive design principles — mobile-first vs desktop-first.
  5. Media queries — @media, common breakpoints.
  6. Responsive units & images — max-width: 100%, fluid typography with clamp().

Why This Week Matters: A site that isn't responsive is considered broken by modern standards. Grid is standard for any non-trivial page layout.

Daily Study Plan

Day Focus
Day 1 Grid container basics — grid-template-columns/rows, gap. Build a 3x3 grid.
Day 2 Grid item placement — spanning rows/columns. Build a magazine-style gallery.
Day 3 grid-template-areas — build a full page layout using named areas.
Day 4 Media queries — breakpoints for mobile/tablet/desktop. Convert Week 3 page to responsive.
Day 5 Mobile-first refactor of an earlier project.
Day 6 Responsive images and fluid typography with clamp().
Day 7 Review + mini project + GitHub commit.

🛠 Mini Project

A fully responsive personal blog homepage with CSS Grid layout (header, sidebar, main article grid, footer), at least 3 breakpoints, and fluid typography.

Common Mistakes

  • Designing desktop-first and "squeezing" down for mobile.
  • Using too many arbitrary breakpoints.
  • Forgetting max-width: 100% on images.

Free Resources

  • CSS Grid Garden (cssgridgarden.com)
  • CSS-Tricks — "A Complete Guide to Grid"
  • MDN Web Docs — Responsive Design & Media Queries
  • web.dev — "Learn Responsive Design" (Google)

Estimated Study Hours: 38–40 hours

Week 5 — Git, GitHub, Tailwind CSS & Deployment

Track code with Git, collaborate on GitHub, style with Tailwind, and deploy live.

Topics & Subtopics

  1. Git fundamentals — init, add, commit, status, log, .gitignore.
  2. GitHub basics — creating repos, push/pull, README files, GitHub Pages.
  3. Branching basics — branch, checkout/switch, merge.
  4. Tailwind CSS fundamentals — setup, utility classes, responsive prefixes.
  5. Deployment — GitHub Pages, Netlify, or Vercel.
  6. Browser DevTools deep-dive — Elements, Console, Network tabs.

Why This Week Matters: Git/GitHub is non-negotiable for every employer. Deployment turns projects into shareable live links. Tailwind is among the most in-demand styling approaches.

Daily Study Plan

Day Focus
Day 1 Git fundamentals — init, add, commit, status, log, .gitignore.
Day 2 GitHub — create repos, push projects, write README files.
Day 3 Branching — create a branch, make changes, merge back.
Day 4 Tailwind CSS setup + utility classes.
Day 5 Rebuild Week 3 restaurant page using Tailwind CSS only.
Day 6 Deployment — deploy 2–3 projects via GitHub Pages and Netlify/Vercel.
Day 7 DevTools deep-dive + review + final commits/push.

🛠 Mini Project

Deploy a personal portfolio "v1" live on Netlify or GitHub Pages, with code on GitHub, a clean README, and a working live link.

Common Mistakes

  • Committing huge unrelated changes in one commit.
  • Never writing README files.
  • Forgetting to push after committing.
  • Avoiding branches due to merge conflict fear.

Free Resources

  • Git & GitHub for Beginners (freeCodeCamp YouTube)
  • GitHub Docs — "Hello World" guide
  • Tailwind CSS official docs (tailwindcss.com/docs)
  • Netlify Docs / Vercel Docs

Estimated Study Hours: 35–38 hours

Week 6 — JavaScript Fundamentals

Understand core programming logic — variables, data types, operators, conditionals, loops, and functions.

Topics & Subtopics

  1. Variables & data types — let, const, var; string, number, boolean, null, undefined.
  2. Operators — arithmetic, comparison (== vs ===), logical, ternary.
  3. Conditionals — if/else, else if, switch.
  4. Loops — for, while, do...while, break/continue.
  5. Functions — declarations, expressions, arrow functions, scope.
  6. Arrays basics — creating, indexing, push, pop, iteration.
  7. Objects basics — key-value pairs, dot vs bracket notation.

Why This Week Matters: This is the single most important transition — from "describing/styling pages" to "programming logic." Rushing this week is the #1 cause of beginners feeling lost in React later.

Daily Study Plan

Day Focus
Day 1 Variables, data types, typeof. Declare and log 10 different variable types.
Day 2 Operators. Solve 10 small operator-based problems.
Day 3 Conditionals. Solve: grade calculator, even/odd checker, leap year checker.
Day 4 Loops. Solve: multiplication table, sum 1–100, FizzBuzz.
Day 5 Functions. Refactor Day 3–4 solutions into reusable functions.
Day 6 Arrays basics. Solve: find max/min, count vowels, reverse array.
Day 7 Objects basics + review + mini project + GitHub commit.

🛠 Mini Project

A console-based "Number Guessing Game" — random number, prompts for guesses, "too high/too low" feedback, attempt tracking.

Common Mistakes

  • Using var instead of let/const.
  • Confusing = with ==/===.
  • Infinite loops from forgetting to update loop variable.
  • Not understanding function scope.

How to Avoid Them

  • Default to const; use let only when needed; avoid var.
  • Always use === (strict equality).
  • Double-check loop increment logic before running.

Free Resources

  • freeCodeCamp — JavaScript Algorithms and Data Structures
  • MDN Web Docs — JavaScript First Steps
  • javascript.info — free JS tutorial
  • The Odin Project — JavaScript Foundations

Estimated Study Hours: 40–42 hours

Week 7 — DOM Manipulation, Events & Arrays/Objects Deep Dive

Make any static page interactive — clicks, dynamic content, form validation using JavaScript.

Topics & Subtopics

  1. Selecting DOM elements — getElementById, querySelector, querySelectorAll.
  2. Manipulating the DOM — textContent, innerHTML, classList, creating/appending elements.
  3. Events — addEventListener, click, submit, keydown, input, event object.
  4. Event bubbling & delegation.
  5. Form handling & validation — preventDefault(), reading values, validation logic.
  6. Array methods deep dive — map, filter, reduce, forEach, find, includes.
  7. Object methods & destructuring.

Why This Week Matters: This is where JavaScript visibly transforms pages. map/filter/reduce are used constantly in React, so mastering them now makes React dramatically easier.

Daily Study Plan

Day Focus
Day 1 DOM selection methods. Select and log elements from your portfolio page.
Day 2 DOM manipulation. Build a dark/light mode toggle button.
Day 3 Events — click, input, keydown. Build a basic image slider.
Day 4 Forms — preventDefault, validation. Add validation to your contact form.
Day 5 Array methods — map, filter, reduce, forEach. Solve 8–10 problems.
Day 6 Object methods + destructuring. Refactor earlier code.
Day 7 Review + mini project + GitHub commit.

🛠 Mini Project

A "To-Do List App" (HTML/CSS/JS) with: add task, delete task, mark complete, filter (all/active/completed), task count — deployed live.

Common Mistakes

  • Using innerHTML carelessly, risking security issues.
  • Forgetting event.preventDefault() on forms.
  • Mutating arrays directly instead of using map/filter.

Free Resources

  • MDN Web Docs — DOM manipulation & Events
  • javascript.info — "Document," "Events" chapters
  • freeCodeCamp — DOM manipulation projects
  • JS Array Methods cheat sheet (MDN)

Estimated Study Hours: 40–42 hours

Week 8 — Modern JavaScript (ES6+), Async JS & Intro to React

Master modern syntax, fetch real API data, and take the first steps into React.

Topics & Subtopics

  1. ES6+ syntax — template literals, spread/rest, default parameters, modules.
  2. Asynchronous JavaScript — Promises, async/await.
  3. Fetch API — fetch(), responses, try/catch.
  4. JSON — JSON.parse/JSON.stringify, API data.
  5. Why frameworks exist — limitations of vanilla DOM at scale.
  6. React introduction — Virtual DOM, components, JSX basics.
  7. Setting up React — Vite, folder structure.

Why This Week Matters: Async JS and Fetch are how real websites get real data. Understanding why React exists makes it feel natural rather than arbitrary.

Daily Study Plan

Day Focus
Day 1 ES6+ syntax — template literals, spread/rest, modules.
Day 2 Promises + .then()/.catch().
Day 3 async/await + try/catch.
Day 4 Fetch API + JSON — fetch from a public API and display data.
Day 5 Build a small API project (joke/quote generator).
Day 6 React intro — Virtual DOM, JSX. Set up first Vite + React project.
Day 7 Review + mini project + GitHub commit.

🛠 Mini Project

An API-powered mini app (Quote Generator / Joke Generator / Random Dog Image Viewer) with fetch/async/await, a "New" button, loading state, and error handling — deployed live.

Common Mistakes

  • Forgetting await and using Promise object as data.
  • Not handling fetch errors at all.
  • Confusing JSX with HTML (className vs class).

Free Resources

  • javascript.info — "Promises, async/await"
  • MDN Web Docs — Using Fetch
  • React official docs — "Quick Start" (react.dev)
  • JSONPlaceholder / free public API directory

Estimated Study Hours: 40–42 hours

Week 9 — React Fundamentals

Build a multi-component React application using JSX, props, and state.

Topics & Subtopics

  1. Components — function components, naming, composition.
  2. JSX in depth — expressions, conditional rendering, lists with .map() + key.
  3. Props — passing data, defaults, children prop.
  4. State — useState, immutability, state vs props.
  5. Event handling — onClick, onChange, onSubmit, passing functions as props.
  6. Conditional rendering — ternary, &&, early returns.
  7. Lifting state up — sharing state via common parent.

Why This Week Matters: Components, props, and state are the conceptual core of React. Once these "click," everything else becomes much easier. Most beginner confusion traces back to a shaky grasp of these three.

Daily Study Plan

Day Focus
Day 1 Components + composition. Build Header, Card, Footer components.
Day 2 JSX — expressions, lists with .map() + key.
Day 3 Props — passing data, defaults, children.
Day 4 State — useState. Build counter + favorite toggle.
Day 5 Event handling + conditional rendering. Build show/hide toggle, tab switcher.
Day 6 Lifting state up — parent managing shared state for siblings.
Day 7 Review + mini project + GitHub commit.

🛠 Mini Project

A React "Movie/Book Search & Favorites" app — item list, search/filter input, mark favorites, state via useState — deployed live.

Common Mistakes

  • Mutating state directly instead of using setter function.
  • Missing or misusing key prop in lists.
  • Confusing props vs state.

How to Avoid Them

  • Always use setX(newValue), never mutate directly.
  • Use stable unique IDs as key, not array index.
  • "Does this component own or receive it?" — owns = state, receives = props.

Free Resources

  • React official docs — "Learn React" (react.dev/learn)
  • freeCodeCamp — React section
  • Scrimba — Learn React (free modules)

Estimated Study Hours: 42–45 hours

Week 10 — React Hooks, Routing & Real API Integration

Build a realistic multi-page React app that fetches live data and manages side effects.

Topics & Subtopics

  1. useEffect — side effects, dependency array, cleanup.
  2. Fetching API data in React — useState + useEffect + fetch, loading/error states.
  3. React Router — BrowserRouter, Routes, Route, Link, dynamic routes, useParams.
  4. Forms in React — controlled components, multi-field forms, validation.
  5. Custom hooks (intro) — extracting reusable logic.
  6. Component organization — folder structure.

Why This Week Matters: useEffect and routing turn a single component into a real multi-page application with live data. These are the exact skills tested in entry-level React interviews.

Daily Study Plan

Day Focus
Day 1 useEffect — dependency array, cleanup. 3 isolated examples.
Day 2 Fetching API data with loading/error states. Build "User List" component.
Day 3 React Router — multi-page nav with Link/NavLink.
Day 4 Dynamic routes with useParams — details page (/item/:id).
Day 5 Forms — controlled inputs, multi-field form with validation.
Day 6 Custom hooks — extract useFetch and reuse it.
Day 7 Review + mini project + GitHub commit.

🛠 Mini Project

A "Recipe/Product Explorer" React app — API list, search bar, detail pages via /item/:id, loading/error states, clean navigation — deployed on Vercel/Netlify.

Common Mistakes

  • Wrong useEffect dependency array causing infinite loops.
  • Not handling loading/error states.
  • Forgetting BrowserRouter wrapper at top level.

Free Resources

  • React official docs — useEffect, Custom Hooks (react.dev)
  • React Router docs (reactrouter.com)
  • freeCodeCamp — React Router tutorials

Estimated Study Hours: 42–45 hours

Week 11 — Deployment, Performance, Accessibility & Testing

Ship a production-quality, accessible, performant app with basic automated tests.

Topics & Subtopics

  1. Production deployment — npm run build, Vercel/Netlify, environment variables.
  2. Web performance — image optimization, lazy loading, Lighthouse audits.
  3. Accessibility (a11y) — ARIA, keyboard nav, color contrast, alt text.
  4. Basic SEO — meta tags, title tags, Open Graph.
  5. Intro to testing — concepts + a first test with React Testing Library / Vitest.
  6. Browser compatibility — caniuse.com, polyfills (conceptual).

Why This Week Matters: A project that only works on your laptop isn't finished. This week professionalizes everything and separates "tutorial-level" work from a presentable portfolio.

Daily Study Plan

Day Focus
Day 1 Production build & deployment for React. Deploy Week 10 project.
Day 2 Performance — Lighthouse audit, fix 2–3 issues.
Day 3 Accessibility — ARIA, keyboard nav testing, fix issues.
Day 4 Color contrast + alt text audit across all projects.
Day 5 SEO — meta tags, titles, Open Graph tags.
Day 6 Testing — Vitest/RTL setup, write 2–3 simple tests.
Day 7 Apply fixes across all projects + final commits.

🛠 Mini Project

Polished "v2" of Week 10 app: improved Lighthouse score, accessibility fixes, SEO meta tags, 2+ automated tests — redeployed live.

Common Mistakes

  • Treating deployment as "done" without auditing.
  • Adding alt="" everywhere lazily.
  • Ignoring keyboard navigation.
  • Skipping testing entirely.

Free Resources

  • web.dev — Performance, Accessibility, SEO (Google)
  • WebAIM — Accessibility checklist & contrast checker
  • React Testing Library docs
  • Vitest docs

Estimated Study Hours: 38–40 hours

Week 12 — Capstone Project, Portfolio & Interview Prep

Build, polish, and ship a capstone; assemble a portfolio; prepare for interviews/viva.

Topics & Subtopics

  1. Project planning — scope, wireframing, component breakdown.
  2. Capstone build — applying Weeks 1–11 in one project.
  3. Portfolio website — listing all projects with live links.
  4. GitHub profile polish — pinned repos, strong READMEs.
  5. Interview/viva prep — concept review, mock Q&A.

Why This Week Matters: This converts 11 weeks of learning into tangible proof of skill. A finished capstone and polished portfolio are what evaluators and recruiters actually look at.

Daily Study Plan

Day Focus
Day 1 Plan capstone — features, wireframe, list APIs needed.
Day 2 Build core structure and React component skeleton.
Day 3 Implement state, interactivity, API integration.
Day 4 Implement routing and remaining features.
Day 5 Polish — responsive, accessibility, Lighthouse, SEO.
Day 6 Deploy capstone; build/update portfolio site.
Day 7 Interview prep — explain projects out loud, review concepts.

Final Capstone: "EventHub" — Event Discovery & Booking Interface

A complete frontend app demonstrating every roadmap skill:

  1. Home/Browse — event list from API/JSON in responsive Grid/Flexbox cards.
  2. Search & Filter — by name/category with controlled inputs.
  3. Event Details — dynamic route (/event/:id).
  4. "My Saved Events" — save/unsave with useState/useEffect.
  5. Responsive, accessible, performant — passes Lighthouse, semantic HTML.

Tech: HTML5, CSS3, Tailwind (optional), JavaScript ES6+, React, React Router, Fetch API, Git/GitHub, Vercel/Netlify.

Deliverables: Live link, GitHub repo with README, project report.

Common Mistakes

  • Over-scoping the capstone (too many features).
  • Skipping wireframing and coding immediately.
  • Neglecting the portfolio site itself.

How to Avoid Them

  • 3–5 core features maximum, fully polished.
  • Always wireframe before coding.
  • Portfolio site is equally important — it's seen first.

Free Resources

  • Excalidraw / Figma (free) — wireframing
  • Pramp / interviewing.io — mock interviews
  • web.dev — project checklists

Estimated Study Hours: 42–45 hours
